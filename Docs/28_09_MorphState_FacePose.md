# 28.09 — Морфы (blendshapes) и FacePose

## Что мы делаем?

Закрываем «анимационный» блок Фазы 28 — **морфами** (они же blendshapes, shape keys). Если скелет двигает кости, то морфы двигают **сами вершины меша**: открыть рот, прикрыть глаза, надуть щёки, сделать гримасу. Это вторая большая ось анимации, нужная для лица и мимики.

## Что такое морф

Морф (blendshape) — это **отклонение позиции каждой вершины меша** от базовой формы:

```
base  + morph_smile * w_smile
      + morph_blink * w_blink
      + morph_o     * w_o
      = итоговая поза вершин
```

`w_*` — веса от 0 до 1 (иногда расширяют до отрицательных и >1 для гипербол). При `w = 0` морф не влияет, при `w = 1` — действует полностью. Веса можно смешивать **аддитивно** между разными морфами — поэтому ими хорошо ловится мимика.

## Где морфы хранятся

Морфы — часть **`.vmdl`**. Художник в Blender/Maya/3ds Max делает blendshape’ы → экспортирует в `.fbx` со shape keys → s&box импортирует их в `.vmdl`. Список доступных морфов виден в Asset Browser → Model Inspector.

У Citizen-головы уже есть готовый набор морфов: `mouth_open`, `mouth_smile`, `eyes_blink_L`, `eyes_blink_R`, `brow_up_L`, `brow_down_R`, `cheek_puff` и т.д.

## API: чтение и запись веса морфа

`SkinnedModelRenderer` (или `ModelRenderer` с морфами) даёт доступ к морфам по имени:

```csharp
// поставить вес
renderer.Morphs.Set( "mouth_smile", 0.7f );

// прочитать
float w = renderer.Morphs.Get( "mouth_smile" );

// сбросить все
renderer.Morphs.Reset();
```

> Точные имена методов могут меняться между версиями s&box (`Morphs.Set` / `SetMorph` — встречалось по-разному). Главное — **доступ через `SkinnedModelRenderer`** и работа в строках-именах.

## MorphState: морф как часть префаба

Компонент **`MorphState`** (Фаза [12.05](12_05_MorphState.md)) описывает «один заранее настроенный набор морфов» как ассет/префаб. Удобно для пропов: «закрытая бочка», «вмятина», «открытая дверца» — всё одной моделью с тремя пресетами.

Во время игры компонент `MorphState`:

1. Хранит словарь `name -> weight`.
2. На `OnEnabled` (или по событию) **прокатывает** все веса в свой `SkinnedModelRenderer`.
3. Сетевая синхронизация — через `[Sync]` на словаре (см. соответствующий док).

Это противоположность «процедурного» подхода: тут морфы статичны и заранее настроены художником.

## FacePoseEditor: морфы для Citizen-лица

[21_02 — FacePoseEditor](21_02_FacePoseEditor.md) — это **редактор внутри игры/инструмента**, который показывает превью головы Citizen и позволяет крутить ползунки веса морфа. Используется в Sandbox для:

- Редактора игрока: настроить выражение лица аватара.
- Сохранения «паспорта» лица в `PlayerData` (через сериализацию весов морфов).

## Анимация морфов: lip-sync и эмоции

Морфы анимируются двумя способами:

### 1. Из AnimGraph

Граф умеет выводить значения морфов как часть итоговой позы. Тогда у клипа лицевой анимации (`.vanim` для головы) ты не двигаешь кости челюсти, а двигаешь **вес морфа** `mouth_open` по времени. Граф сам подмешает их в `Renderer.Morphs`.

### 2. Из C# напрямую

Для динамики (lip-sync под голос, реакция на урон) — пишешь сам:

```csharp
protected override void OnUpdate()
{
    var voiceLevel = SandboxVoice.Local.OutputLevel;  // 0..1
    Head.Morphs.Set( "mouth_open", voiceLevel );
}
```

В Sandbox `SandboxVoice` уже отдаёт уровень громкости — можно цеплять к открытию рта говорящего игрока.

## Морфы и сеть

Веса морфов **не синхронизируются автоматически** между клиентами — их слишком много и они меняются часто. Принципы:

- **Косметика** (моргание, lip-sync) — считается **локально на каждом клиенте**, не сеть.
- **Долгоживущее состояние** (постоянное выражение лица из FacePose) — синхронизируется через `[Sync]` на словаре весов или через `MorphState`/PlayerData.

## Подводные камни

- **Имя морфа должно совпадать** один-в-один с именем shape key из FBX. Регистр важен.
- **Морфы есть не у всех моделей.** Если у модели нет blendshape’ов — `Morphs.Set` молча ничего не сделает.
- **Морф + кости** на одной части лица могут «драться» (например, морф челюсти + кость челюсти). Договаривайся с художником, кто отвечает за что.
- **На производительность** морфы влияют умеренно: в кадре сжатие весов делается один раз на инстанс. Но если у тебя 200 говорящих NPC — лучше lip-sync апдейтить раз в 2 кадра.

## Подсказки на практике

- **Lip-sync** под `SandboxVoice.OutputLevel` — самое простое «оживление» головы.
- **Моргание** — таймер на 2–6 секунд, потом 0.15 сек `eyes_blink = 1` и обратно.
- **Урон в лицо** — короткая гримаса (`mouth_grimace = 1` на 0.3 сек, потом `LerpTo(0)`).
- **FacePose** запоминай в `PlayerData` (см. [03.06](03_06_PlayerData.md)) — лицо переживёт реконнект.
