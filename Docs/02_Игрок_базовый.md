# Этап 2: Игрок — первый компонент

## Цель этого этапа

Создать базового игрока, который может ходить по карте, прыгать и смотреть по сторонам. После этого этапа ты сможешь зайти в игру и управлять персонажем.

## Как устроен игрок в s&box

В s&box игрок — это **GameObject**, к которому прикреплены **компоненты**. Каждый компонент отвечает за одну функцию:

```
GameObject "Player"
├── PlayerController   ← ВСТРОЕННЫЙ в движок (ходьба, прыжки, физика)
├── Player             ← НАШ код (здоровье, смерть)
├── PlayerData         ← НАШ код (счёт, имя, статистика)
├── PlayerInventory    ← НАШ код (инвентарь, оружие)
├── PlayerFlashlight   ← НАШ код (фонарик)
├── PlayerFallDamage   ← НАШ код (урон от падения)
└── ... и другие
```

**Ключевой момент**: `PlayerController` — это компонент **самого движка** s&box. Он уже умеет ходить, прыгать, приседать, управлять камерой. Нам не нужно писать физику передвижения с нуля!

**В движке** (`sbox-public`): `PlayerController` — это класс из пространства `Sandbox.Movement`. Он обрабатывает ввод (`Input.Down("Forward")`) и рассчитывает движение с учётом физики (гравитация, столкновения со стенами, лестницы).

## Шаг 2.1: Создаём интерфейс событий игрока

Прежде чем создавать игрока, нам нужен способ, чтобы компоненты «общались» друг с другом. Для этого создадим **интерфейс событий**.

Создай файл `Code/Player/PlayerEvent.cs`:

```csharp
/// <summary>
/// Вызывается только на GameObject'е игрока для его действий и событий.
/// Любой компонент на объекте игрока может реализовать этот интерфейс,
/// чтобы реагировать на события.
/// </summary>
public interface IPlayerEvent : ISceneEvent<IPlayerEvent>
{
    // Вызывается когда игрок заспавнился
    void OnSpawned() { }

    // Структура с параметрами урона
    public struct DamageParams
    {
        public float Damage { get; set; }       // Количество урона
        public GameObject Attacker { get; set; }  // Кто атаковал
        public GameObject Weapon { get; set; }    // Каким оружием
        public TagSet Tags { get; set; }          // Теги урона (headshot, explosion...)
        public Vector3 Position { get; set; }     // Где попала пуля
        public Vector3 Origin { get; set; }       // Откуда стреляли
    }
    void OnDamage( DamageParams args ) { }

    // Структура с параметрами смерти
    public struct DiedParams
    {
        public GameObject Attacker { get; set; }  // Кто убил
    }
    void OnDied( DiedParams args ) { }

    void OnJump() { }                                    // Прыгнул
    void OnLand( float distance, Vector3 velocity ) { }  // Приземлился
    void OnSuicide() { }                                 // Суицид
    void OnPickup( BaseCarryable item ) { }              // Подобрал предмет
    void OnCameraMove( ref Angles angles ) { }           // Камера поворачивается
    void OnCameraSetup( CameraComponent camera ) { }     // Настройка камеры
    void OnCameraPostSetup( CameraComponent camera ) { } // После настройки камеры
}

/// <summary>
/// Глобальные события локального игрока — рассылаются по всей сцене.
/// Используется для UI и эффектов, которые реагируют на действия игрока.
/// </summary>
public interface ILocalPlayerEvent : ISceneEvent<ILocalPlayerEvent>
{
    void OnJump() { }
    void OnLand( float distance, Vector3 velocity ) { }
    void OnPickup( BaseCarryable weapon ) { }
}
```

### Как работают события в движке:

**ISceneEvent<T>** — это механизм движка s&box для отправки событий. Когда вызывается:

```csharp
IPlayerEvent.PostToGameObject( GameObject, x => x.OnDied( args ) );
```

Движок находит **все компоненты** на этом GameObject, которые реализуют `IPlayerEvent`, и вызывает у них метод `OnDied`. Это как рассылка уведомлений.

> 💡 Методы в интерфейсе имеют **пустую реализацию по умолчанию** (`{ }`). Это значит, что компонент может реализовать только те события, которые ему нужны, а остальные игнорировать.

> ⚠️ **Заметь**: На этом шаге код **не скомпилируется**, потому что мы ссылаемся на `BaseCarryable`, которого ещё нет. Это нормально — мы создадим его позже. Пока можешь закомментировать строку `void OnPickup(BaseCarryable item) { }` если хочешь проверить компиляцию.

## Шаг 2.2: Создаём данные игрока (PlayerData)

Этот компонент хранит **постоянную** информацию об игроке, которая переживает смерть: имя, счёт убийств, Steam ID.

Создай файл `Code/Player/PlayerData.cs`:

```csharp
/// <summary>
/// Хранит постоянные данные игрока: убийства, смерти, имя.
/// Этот компонент живёт отдельно от Player — он НЕ уничтожается при смерти.
/// </summary>
public sealed partial class PlayerData : Component
{
    /// <summary>
    /// Уникальный ID игрока — совпадает с ID соединения (Connection).
    /// </summary>
    [Property] public Guid PlayerId { get; set; }
    
    /// <summary>
    /// Steam ID игрока — 64-битное число из Steam.
    /// </summary>
    [Property] public long SteamId { get; set; } = -1L;
    
    /// <summary>
    /// Отображаемое имя игрока.
    /// </summary>
    [Property] public string DisplayName { get; set; }

    // Синхронизируемые по сети значения
    [Sync] public int Kills { get; set; }
    [Sync] public int Deaths { get; set; }
    [Sync] public bool IsGodMode { get; set; }

    /// <summary>
    /// Находит соединение (Connection) этого игрока.
    /// </summary>
    public Connection Connection => Connection.Find( PlayerId );

    /// <summary>
    /// Это я? (проверка на локального игрока)
    /// </summary>
    public bool IsMe => PlayerId == Connection.Local.Id;

    /// <summary>
    /// Пинг игрока в миллисекундах.
    /// </summary>
    public float Ping => Connection?.Ping ?? 0;

    /// <summary>
    /// Все PlayerData в текущей сцене.
    /// </summary>
    public static IEnumerable<PlayerData> All => Game.ActiveScene.GetAll<PlayerData>();

    /// <summary>
    /// Найти PlayerData по соединению.
    /// </summary>
    public static PlayerData For( Connection connection ) 
        => connection == null ? default : For( connection.Id );

    /// <summary>
    /// Найти PlayerData по ID игрока.
    /// </summary>
    public static PlayerData For( Guid playerId )
    {
        return All.FirstOrDefault( x => x.PlayerId == playerId );
    }

    // --- Система респавна (на стороне хоста) ---
    
    private bool _needsRespawn;
    private RealTimeSince _timeSinceDied;

    /// <summary>
    /// Вызывается на хосте когда игрок умирает.
    /// Запускает обратный отсчёт респавна.
    /// </summary>
    public void MarkForRespawn()
    {
        _needsRespawn = true;
        _timeSinceDied = 0;
    }

    /// <summary>
    /// Запрос на респавн. Вызывается владельцем (Rpc.Host + OwnerOnly).
    /// </summary>
    [Rpc.Host( NetFlags.OwnerOnly | NetFlags.Reliable )]
    public void RequestRespawn()
    {
        _needsRespawn = false;

        // Удаляем оставшегося наблюдателя (призрака)
        foreach ( var observer in Scene.GetAllComponents<PlayerObserver>()
            .Where( x => x.Network.Owner?.Id == PlayerId ).ToArray() )
        {
            observer.GameObject.Destroy();
        }

        // Просим GameManager заспавнить игрока
        GameManager.Current?.SpawnPlayer( this );
    }

    protected override void OnUpdate()
    {
        // Только хост управляет автоматическим респавном
        if ( !Networking.IsHost ) return;
        if ( !_needsRespawn ) return;
        if ( _timeSinceDied < 4f ) return; // Ждём 4 секунды

        RequestRespawn();
    }
}
```

### Почему PlayerData отделён от Player?

Когда игрок **умирает**, его GameObject с компонентом `Player` **уничтожается**. Но мы не хотим потерять счёт убийств и статистику! Поэтому `PlayerData` живёт на **отдельном** GameObject, который существует всю игру.

```
Сцена
├── PlayerData "Игрок 1" ← живёт всю игру
│   └── Kills: 5, Deaths: 2
├── Player "Игрок 1"     ← уничтожается при смерти
│   └── Health: 100
└── ...
```

### Как работает респавн:

```
1. Игрок умирает → Player.Kill() уничтожает GameObject
2. PlayerData.MarkForRespawn() запускает таймер
3. Через 4 секунды → PlayerData.RequestRespawn()
4. GameManager.SpawnPlayer() создаёт нового Player
```

### Про [Sync] и [Rpc]:

- `[Sync] public int Kills` — когда хост меняет `Kills`, все клиенты автоматически получают новое значение
- `[Rpc.Host( NetFlags.OwnerOnly )]` — этот метод может вызвать **только владелец** этого объекта, и он выполнится на **хосте**

## Шаг 2.3: Создаём основной компонент Player

Сейчас мы создадим **упрощённую версию** Player — только базовые вещи. Позже мы будем его расширять.

Создай файл `Code/Player/Player.cs`:

```csharp
/// <summary>
/// Основной компонент игрока. Хранит здоровье и обрабатывает урон/смерть.
/// </summary>
public sealed partial class Player : Component, PlayerController.IEvents
{
    /// <summary>
    /// Находит локального игрока (того, кем ты управляешь).
    /// </summary>
    public static Player FindLocalPlayer() 
        => Game.ActiveScene.GetAll<Player>().FirstOrDefault( x => x.IsLocalPlayer );

    /// <summary>
    /// Ссылка на PlayerController — встроенный компонент движка.
    /// [RequireComponent] означает: "этот компонент ОБЯЗАТЕЛЕН на том же GameObject".
    /// </summary>
    [RequireComponent] public PlayerController Controller { get; set; }
    
    /// <summary>
    /// Ссылка на GameObject тела (3D модель персонажа).
    /// </summary>
    [Property] public GameObject Body { get; set; }

    /// <summary>
    /// Здоровье. [Sync] — синхронизируется по сети от хоста.
    /// </summary>
    [Property, Range( 0, 100 ), Sync( SyncFlags.FromHost )] 
    public float Health { get; set; } = 100;
    
    [Property, Range( 0, 100 ), Sync( SyncFlags.FromHost )] 
    public float MaxHealth { get; set; } = 100;

    /// <summary>
    /// Ссылка на данные игрока (отдельный компонент).
    /// </summary>
    [Sync( SyncFlags.FromHost )] public PlayerData PlayerData { get; set; }

    /// <summary>
    /// Позиция и направление глаз (откуда игрок смотрит).
    /// Берётся из PlayerController.
    /// </summary>
    public Transform EyeTransform
    {
        get
        {
            if ( !Controller.IsValid() ) return default;
            return Controller.EyeTransform;
        }
    }

    /// <summary>
    /// Это локальный игрок? (не прокси-копия другого игрока)
    /// </summary>
    public bool IsLocalPlayer => !IsProxy;
    
    public Guid PlayerId => PlayerData.IsValid() ? PlayerData.PlayerId : Guid.Empty;
    public long SteamId => PlayerData.IsValid() ? PlayerData.SteamId : 0;
    public string DisplayName => PlayerData.IsValid() ? PlayerData.DisplayName : "Unknown";

    /// <summary>
    /// Вызывается движком каждый кадр перед обработкой ввода.
    /// Это из интерфейса PlayerController.IEvents.
    /// </summary>
    void PlayerController.IEvents.PreInput()
    {
        OnControl();
    }

    /// <summary>
    /// Обработка ввода игрока каждый кадр.
    /// </summary>
    private void OnControl()
    {
        // Разрешаем движение
        Controller.UseInputControls = true;
    }

    /// <summary>
    /// Когда игрок приземлился (из PlayerController.IEvents).
    /// </summary>
    void PlayerController.IEvents.OnLanded( float distance, Vector3 impactVelocity )
    {
        IPlayerEvent.PostToGameObject( GameObject, x => x.OnLand( distance, impactVelocity ) );
    }

    /// <summary>
    /// Когда игрок прыгнул (из PlayerController.IEvents).
    /// </summary>
    void PlayerController.IEvents.OnJumped()
    {
        IPlayerEvent.PostToGameObject( GameObject, x => x.OnJump() );
    }
}
```

### Разбор ключевых моментов:

#### `[RequireComponent] public PlayerController Controller`

Атрибут `[RequireComponent]` говорит движку: «На этом GameObject **обязательно** должен быть компонент `PlayerController`». Если его нет — движок добавит автоматически.

`PlayerController` — это **встроенный** компонент движка, который:
- Обрабатывает клавиши WASD → движение
- Обрабатывает мышь → поворот камеры
- Считает гравитацию
- Проверяет столкновения со стенами
- Управляет прыжком и приседанием

#### `PlayerController.IEvents`

Это **интерфейс обратных вызовов** из `PlayerController`. Когда происходит что-то важное (прыжок, приземление, ввод) — контроллер вызывает методы этого интерфейса. Наш `Player` реализует его и может реагировать.

#### `IsProxy`

В мультиплеере на твоём компьютере существуют **копии** других игроков. Они называются **прокси** (proxy). Ты не управляешь ими — они просто отображают позицию других игроков.

```
На ТВОЁМ компьютере:
├── Player "Ты"     ← IsProxy = false (ты им управляешь)
├── Player "Саша"   ← IsProxy = true  (копия, управляет Саша на своём компе)
└── Player "Маша"   ← IsProxy = true  (копия)
```

#### `[Sync( SyncFlags.FromHost )]`

Означает: «Это значение может менять **только хост** (сервер), и оно автоматически отправляется всем клиентам». Это защита от читов — клиент не может сам себе поставить 9999 здоровья.

## Шаг 2.4: Создание префаба (шаблона) игрока

Префаб создаётся в **редакторе s&box**, а не в коде. Вот что нужно сделать:

1. В редакторе нажми **File → New Prefab**
2. Назови его `player` и сохрани в `Assets/prefabs/engine/player.prefab`
3. Добавь компоненты:
   - **PlayerController** (встроенный — отвечает за ходьбу)
   - **Player** (наш компонент)
   - **SkinnedModelRenderer** (для отображения 3D модели)
   - **Dresser** (встроенный — одежда персонажа)

### Настройки PlayerController в префабе:

| Свойство | Значение | Зачем |
|----------|---------|-------|
| Gravity | 800 | Сила гравитации |
| WalkSpeed | 200 | Скорость ходьбы |
| RunSpeed | 320 | Скорость бега (Shift) |
| CrouchSpeed | 80 | Скорость в приседе |
| JumpStrength | 300 | Сила прыжка |
| StepHeight | 18 | Высота ступеньки, через которую можно перешагнуть |
| EyeHeight | 64 | Высота глаз от земли |

### Сетевые настройки:

В разделе **Network** префаба:
- **NetworkMode**: `Object` — это корневой сетевой объект
- **OwnerTransfer**: `Takeover` — владелец может быть передан

## Шаг 2.5: Создание точки спавна

Чтобы игрок появлялся на карте, нужна **точка спавна** — место, где он материализуется.

Создай файл `Code/Map/MapPlayerSpawner.cs`:

```csharp
/// <summary>
/// Отмечает точку спавна игрока на карте.
/// Просто поставь этот компонент на пустой GameObject в сцене.
/// Позиция этого объекта = место появления игрока.
/// </summary>
public class MapPlayerSpawner : Component
{
    // Этот компонент не содержит логики!
    // Он просто "метка" на карте.
    // GameManager найдёт все MapPlayerSpawner в сцене
    // и случайно выберет одну для спавна.
}
```

> 💡 **Зачем пустой компонент?** Это паттерн «маркер» (marker). GameManager будет искать все объекты с компонентом `MapPlayerSpawner` в сцене. Их позиции — это возможные точки спавна. Карты могут содержать свои точки, или ты можешь добавить их в сцену руками.

## ✅ Проверка

На этом этапе ты **ещё не сможешь** полностью проверить результат в игре, потому что нет GameManager, который создаёт игрока при подключении. Но ты можешь:

1. Убедиться, что код **компилируется** без ошибок
2. Увидеть компонент `Player` в списке компонентов редактора
3. Создать префаб и добавить на него компоненты
