# Этап 4: Здоровье и урон

## Цель этого этапа

Добавить полноценную систему здоровья и урона:
- Игрок может получать урон
- Броня поглощает часть урона  
- При смерти создаётся тряпичная кукла (ragdoll)
- Через 4 секунды игрок возрождается

## Как работает урон в движке s&box

В s&box есть встроенный интерфейс `Component.IDamageable`. Любой компонент, который его реализует, может получать урон:

```csharp
public interface IDamageable
{
    void OnDamage( in DamageInfo damage );
}
```

**DamageInfo** — структура из движка, содержащая:
- `Damage` — количество урона
- `Attacker` — кто атаковал (GameObject)
- `Weapon` — каким оружием
- `Position` — куда попал
- `Origin` — откуда стреляли
- `Tags` — теги урона (headshot, explosion, crush...)

Когда пуля попадает в объект, движок ищет на нём компонент с `IDamageable` и вызывает `OnDamage()`.

## Шаг 4.1: Теги урона

Создай файл `Code/Player/DamageTags.cs`:

```csharp
/// <summary>
/// Константы для тегов урона.
/// Теги помогают определить ТИП урона, чтобы по-разному реагировать.
/// </summary>
public static class DamageTags
{
    public const string Headshot = "headshot";     // Попадание в голову
    public const string Crush = "crush";           // Раздавлен
    public const string Explosion = "explosion";   // Взрыв
    public const string GibAlways = "gibalways";   // Всегда разрывает на части
    public const string Shock = "shock";           // Электрошок
}
```

### Как используются теги:

```
Пуля попала в голову → Tags содержит "headshot" → урон × 2
Взрыв динамита → Tags содержит "explosion" → тело разлетается на части
Обычная пуля → нет специальных тегов → обычный урон
```

    /// </summary>

## Шаг 4.3: Расширяем Player — добавляем урон и смерть

Теперь обновим наш `Player.cs`, добавив реализацию `IDamageable`:

Обнови файл `Code/Player/Player.cs` — добавь `Component.IDamageable` и `IKillSource` к списку интерфейсов:

```csharp
public sealed partial class Player : Component, 
    Component.IDamageable,         // ← НОВОЕ: может получать урон
    IKillSource,                   // ← НОВОЕ: может быть источником убийства
    PlayerController.IEvents
{
    // ... всё что было раньше ...

    // --- НОВОЕ: Броня ---
    [Property, Range( 0, 100 ), Sync( SyncFlags.FromHost )] 
    public float Armour { get; set; } = 0;
    
    [Property, Range( 0, 100 ), Sync( SyncFlags.FromHost )] 
    public float MaxArmour { get; set; } = 100;

    // --- НОВОЕ: IKillSource ---
    string IKillSource.DisplayName => DisplayName;
    long IKillSource.SteamId => SteamId;
    
    void IKillSource.OnKill( GameObject victim )
    {
        PlayerData.Kills++;
    }

    // --- НОВОЕ: Обработка урона ---
    
    /// <summary>
    /// Вызывается движком когда этот игрок получает урон.
    /// Весь расчёт происходит на ХОСТЕ.
    /// </summary>
    public void OnDamage( in DamageInfo dmg )
    {
        // Мёртвые не получают урон
        if ( Health < 1 ) return;
        
        // God Mode — урон не проходит
        if ( !PlayerData.IsValid() ) return;
        if ( PlayerData.IsGodMode ) return;

        var damage = dmg.Damage;
        
        // Выстрел в голову — двойной урон
        if ( dmg.Tags.Contains( DamageTags.Headshot ) )
            damage *= 2;

        // Броня поглощает урон
        if ( Armour > 0 )
        {
            float remainingDamage = damage - Armour;
            Armour = Math.Max( 0, Armour - damage );
            damage = Math.Max( 0, remainingDamage );
        }

        // Применяем урон
        Health -= damage;

        // Оповещаем все компоненты об уроне
        NotifyOnDamage( new IPlayerEvent.DamageParams()
        {
            Damage = damage,
            Attacker = dmg.Attacker,
            Weapon = dmg.Weapon,
            Tags = dmg.Tags,
            Position = dmg.Position,
            Origin = dmg.Origin,
        });

        // Если ещё жив — выходим
        if ( Health >= 1 ) return;

        // Умер!
        Health = 0;
        GameManager.Current.OnDeath( this, dmg );
        Kill( dmg );
    }

    /// <summary>
    /// Рассылает событие урона всем клиентам (для эффектов, звуков).
    /// </summary>
    [Rpc.Broadcast( NetFlags.HostOnly | NetFlags.Reliable )]
    private void NotifyOnDamage( IPlayerEvent.DamageParams args )
    {
        IPlayerEvent.PostToGameObject( GameObject, x => x.OnDamage( args ) );
    }

    // --- НОВОЕ: Смерть ---

    /// <summary>
    /// Вызывается на хосте когда игрок умирает.
    /// </summary>
    void Kill( in DamageInfo d )
    {
        // Оповещаем всех о смерти
        NotifyDeath( new IPlayerEvent.DiedParams() 
        { 
            Attacker = d.Attacker 
        });

        // Создаём тряпичную куклу (ragdoll)
        CreateRagdoll();

        // Помечаем для респавна и удаляем игрока
        PlayerData?.MarkForRespawn();
        GameObject.Destroy();
    }

    /// <summary>
    /// Рассылает событие смерти всем клиентам.
    /// </summary>
    [Rpc.Broadcast( NetFlags.HostOnly | NetFlags.Reliable )]
    void NotifyDeath( IPlayerEvent.DiedParams args )
    {
        IPlayerEvent.PostToGameObject( GameObject, x => x.OnDied( args ) );
    }

    // --- НОВОЕ: Тряпичная кукла ---

    /// <summary>
    /// Создаёт ragdoll (тряпичную куклу) на месте смерти.
    /// Выполняется у ВСЕХ клиентов (Rpc.Broadcast).
    /// </summary>
    [Rpc.Broadcast( NetFlags.HostOnly | NetFlags.Reliable )]
    void CreateRagdoll()
    {
        if ( !Controller.Renderer.IsValid() )
            return;

        // Создаём новый объект для трупа
        var go = new GameObject( true, "Ragdoll" );
        go.Tags.Add( "ragdoll" );
        go.WorldTransform = WorldTransform;

        // Копируем модель игрока
        var mainBody = go.Components.Create<SkinnedModelRenderer>();
        mainBody.CopyFrom( Controller.Renderer );
        mainBody.UseAnimGraph = false; // Отключаем анимации

        // Добавляем физику (тело падает и катится)
        var physics = go.Components.Create<ModelPhysics>();
        physics.Model = mainBody.Model;
        physics.Renderer = mainBody;
        physics.CopyBonesFrom( Controller.Renderer, true );
    }
}
```

### Как работает расчёт урона — пошагово:

```
1. Пуля попадает в игрока
   ↓
2. Движок вызывает OnDamage( DamageInfo )
   - damage = 30, Tags = ["headshot"]
   ↓
3. Проверка: Health > 0? GodMode? → продолжаем
   ↓
4. Headshot? → damage = 30 × 2 = 60
   ↓
5. Броня (50)? → Armour = max(0, 50-60) = 0
                  damage = max(0, 60-50) = 10
   ↓
6. Health = 100 - 10 = 90 → жив!
   
--- или если урон больше ---

6. Health = 30 - 60 = -30 → МЁРТВ!
   ↓
7. Kill() → CreateRagdoll() + Destroy()
   ↓
8. PlayerData.MarkForRespawn() → через 4 секунды → SpawnPlayer()
```

### Что такое Ragdoll (тряпичная кукла)?

Когда игрок умирает, его 3D модель превращается в **физический объект** — тряпичную куклу. Каждая кость тела становится отдельным физическим элементом (голова, руки, ноги), связанным суставами. Это создаёт реалистичное падение тела.

```
Живой игрок:                    Ragdoll:
├── Анимация: бег               ├── Физика: голова (rigidbody)
├── Управление: WASD            ├── Физика: торс (rigidbody)
└── Столкновения: один collider  ├── Физика: левая рука
                                 ├── Физика: правая рука
                                 └── Физика: ноги
                                 (всё связано суставами)
```

**В движке**: `ModelPhysics` — компонент, который создаёт физические тела из скелета модели. `CopyBonesFrom()` копирует текущую позу персонажа, чтобы тело начало падать из той позы, в которой игрок был.

## Шаг 4.4: Расширяем GameManager — обработка смерти

Добавь в `GameManager.cs` метод обработки смерти:

```csharp
// Добавь этот метод в класс GameManager

/// <summary>
/// Вызывается на хосте когда игрок погибает.
/// Обновляет статистику убийцы и жертвы.
/// </summary>
public void OnDeath( Player player, DamageInfo dmg )
{
    Assert.True( Networking.IsHost );

    // Находим источник убийства
    var source = dmg.Attacker?.GetComponent<IKillSource>();
    if ( source == null ) return;

    // Проверяем — это суицид?
    var isSuicide = source is Player p && p == player;

    // Если не суицид — записываем фраг убийце
    if ( !isSuicide )
        source.OnKill( player.GameObject );

    // Записываем смерть жертве
    player.PlayerData.Deaths++;
}
```

## Шаг 4.5: Урон от падения

Создай файл `Code/Player/PlayerFallDamage.cs`:

```csharp
/// <summary>
/// Наносит урон игроку при падении с высоты.
/// Реагирует на событие OnLand (приземление).
/// </summary>
public class PlayerFallDamage : Component, IPlayerEvent
{
    /// <summary>
    /// Высота, при которой урон = максимальный (смертельный).
    /// Единица = примерно 1 дюйм в игровом мире.
    /// 1536 единиц ≈ 39 метров.
    /// </summary>
    [Property] public float FatalHeight { get; set; } = 1536;

    /// <summary>
    /// Минимальная высота, с которой начинается урон.
    /// </summary>
    [Property] public float SafeHeight { get; set; } = 256;

    /// <summary>
    /// Событие приземления — считаем урон от высоты.
    /// </summary>
    void IPlayerEvent.OnLand( float distance, Vector3 velocity )
    {
        if ( !Networking.IsHost ) return;

        // Если высота меньше безопасной — урона нет
        if ( distance < SafeHeight ) return;

        // Рассчитываем урон: от 0 до 100 в зависимости от высоты
        var fraction = (distance - SafeHeight) / (FatalHeight - SafeHeight);
        var damage = fraction * 100f;

        // Наносим урон
        var player = GetComponent<Player>();
        if ( player.IsValid() )
        {
            var dmg = new DamageInfo( damage, GameObject, GameObject );
            player.OnDamage( in dmg );
        }
    }
}
```

### Как работает урон от падения:

```
Высота падения:  0     256    768     1536    2000
                 |------|-------|--------|--------|
Урон:            0      0     ~33      100     100+
                 безопасно  больно   смертельно
```

Формула: `урон = (высота - безопасная) / (смертельная - безопасная) × 100`

Например, падение с 768 единиц: `(768 - 256) / (1536 - 256) × 100 = 40` урона.

## Шаг 4.6: Фонарик игрока

Пока мы добавляем компоненты игрока, давай сразу добавим фонарик — он простой и полезный.

Создай файл `Code/Player/PlayerFlashlight.cs`:

```csharp
/// <summary>
/// Фонарик игрока. Включается/выключается клавишей F.
/// </summary>
public class PlayerFlashlight : Component, IPlayerEvent
{
    [Property] public SpotLight Light { get; set; }

    /// <summary>
    /// Включен ли фонарик. Синхронизируется по сети.
    /// </summary>
    [Sync] public bool IsOn { get; set; }

    void IPlayerEvent.OnSpawned()
    {
        IsOn = false;
    }

    protected override void OnUpdate()
    {
        // Только локальный игрок может включать/выключать
        if ( IsProxy ) return;

        if ( Input.Pressed( "Flashlight" ) )
        {
            Toggle();
        }

        // Обновляем состояние света
        if ( Light.IsValid() )
        {
            Light.Enabled = IsOn;
        }
    }

    void Toggle()
    {
        IsOn = !IsOn;
        
        // Играем звук щелчка
        Sound.Play( IsOn ? "flashlight-on" : "flashlight-off", WorldPosition );
    }
}
```

### Как работает синхронизация фонарика:

```
Игрок 1 нажимает F:
1. IsOn = true (у Игрока 1)
2. [Sync] отправляет IsOn = true всем
3. У Игрока 2 обновляется IsOn = true
4. Light.Enabled = true → свет видно всем
```
