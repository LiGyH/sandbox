# 26.02 — Трассировка: Scene.Trace и луч (Ray)

## Что мы делаем?

«Трассировка» (trace, raycast) — это **виртуальный выстрел невидимым лучом** из точки A в точку B, чтобы спросить движок: *«во что и где этот луч попадёт?»*. Это самая часто используемая операция в шутерах и сэндбоксах: пули, лазерные указатели, тулган, выделение объекта мышью, проверка «вижу ли я цель».

## API: `Scene.Trace`

В s&box трассировка строится через **builder-паттерн**: ты последовательно добавляешь параметры и в конце вызываешь `Run()`.

```csharp
SceneTraceResult tr = Scene.Trace
    .Ray( startPos, endPos )
    .Run();

if ( tr.Hit )
{
    Log.Info( $"Попали в {tr.GameObject} на {tr.EndPosition}" );
}
```

`Scene.Trace.Ray( from, to )` — луч из точки `from` в точку `to`. Альтернатива — задать **направление + длина**:

```csharp
var tr = Scene.Trace
    .Ray( eyePos, eyeForward * 8192 )
    .Run();
```

## Что лежит в `SceneTraceResult`?

| Поле | Что значит |
|---|---|
| `bool Hit` | попали ли вообще (если `false` — остальные поля пустые) |
| `Vector3 StartPosition` | откуда начинался луч |
| `Vector3 EndPosition` | где луч остановился (точка попадания, или `endPos`, если ничего не задели) |
| `Vector3 Normal` | нормаль поверхности в точке удара |
| `float Distance` | пройденное расстояние от старта |
| `float Fraction` | `Distance / общая_длина_луча` (0..1) |
| `GameObject GameObject` | в какой объект попали |
| `Component Component` | в какой компонент (например, `Collider`) |
| `Surface Surface` | какой физический материал (см. [26.01](26_01_Physics_Bodies.md)) |
| `int Hitbox` | индекс хитбокса (если попали в часть модели) |
| `Vector3 HitPosition` | синоним `EndPosition`, удобнее семантически |

## Простейший случай: «куда смотрит игрок?»

```csharp
var ray = new Ray( Camera.WorldPosition, Camera.WorldRotation.Forward );
var tr  = Scene.Trace.Ray( ray, 4096f ).Run();

if ( tr.Hit )
{
    DebugOverlay.Sphere( tr.HitPosition, 4f, Color.Red, duration: 1f );
}
```

`new Ray( origin, direction )` + перегрузка `.Ray( ray, distance )` — это наиболее читаемый способ для прицеливания.

## Игнорировать самого себя

Большинство «выстрелов» начинаются изнутри владельца — иначе луч сразу попадёт в его собственный коллайдер. Используй `IgnoreGameObjectHierarchy` или `WithoutTags`:

```csharp
var tr = Scene.Trace
    .Ray( muzzle, muzzle + forward * 4096f )
    .IgnoreGameObjectHierarchy( Player.GameObject )
    .Run();
```

> Иерархия = сам `Player.GameObject` **и все его потомки**. Это правильный способ «не задеть себя», потому что у игрока обычно много дочерних объектов (модель, оружие, viewmodel).

## Использовать настройки матрицы столкновений

Помнишь [00.05 — Настройки столкновений](00_05_Настройки_столкновений.md)? Туда ты задавал, что тег `bullet` сталкивается с `world` и `solid`, но не с `noclip`. Чтобы трассировка использовала именно эти правила:

```csharp
var tr = Scene.Trace
    .Ray( from, to )
    .WithCollisionRules( "bullet" )   // вести себя как пуля
    .Run();
```

Это удобнее, чем вручную перечислять теги в `WithTag`/`WithoutTag` — все правила задаются централизованно в одном месте.

## Хитбоксы моделей

По умолчанию трассировка попадает в **физическую форму** модели (PhysShape). Если нужно различать «выстрел в голову» и «выстрел в ногу», включи хитбоксы:

```csharp
var tr = Scene.Trace
    .Ray( from, to )
    .UseHitboxes( true )
    .Run();

if ( tr.Hit && tr.Hitbox >= 0 )
{
    var tags = tr.Hitbox.Tags;          // например, "head", "limb"
    if ( tags.Has( "head" ) )
        damage *= 2.5f;
}
```

## Производительность

| Действие | Стоимость |
|---|---|
| 1 трассировка/кадр | копеечная, не думай |
| 100 трассировок/кадр | заметно, оптимизируй |
| 10000 трассировок/кадр | плохо — пересмотри алгоритм |

Если делаешь много лучей подряд, **сначала отсекай дешёвыми проверками** (расстояние, скалярное произведение направлений), а потом уже трассируй.

## Что важно запомнить

- `Scene.Trace.Ray(...).Run()` — основная операция «прощупать мир».
- Результат — `SceneTraceResult` с `Hit`, `HitPosition`, `Normal`, `GameObject`, `Surface`.
- Не забывай `.IgnoreGameObjectHierarchy(self)` — иначе попадёшь в себя.
- `.WithCollisionRules("bullet")` использует матрицу из настроек проекта.
- `.UseHitboxes( true )` нужен для хедшотов/попаданий по конечностям.

## Что дальше?

Луч — это бесконечно тонкая линия. В [26.03](26_03_Tracing_Shapes.md) посмотрим **объёмные** трассировки (сфера, куб, развёртка) — для проверки «помещается ли проп», «не зажат ли игрок в стене», «есть ли свободное место для спавна».
