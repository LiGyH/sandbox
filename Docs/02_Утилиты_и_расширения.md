## Цель этого этапа

Добавить **базовые утилитные классы**, от которых зависят практически все остальные системы игры: оружие, инструменты, камера, HUD, управление, эффекты. Это «невидимый фундамент» — после этого этапа визуально ничего не изменится, но без этих файлов ни один следующий этап не скомпилируется.

Мы создадим:

- **Конвейер камеры** (CameraSetup + CameraNoise)
- **Визуальные эффекты**: кровь, тряска окружения, трассировщик пуль

- **Запись демо** (DemoRecording)
- **Интерфейс убийцы** (IKillSource) для ленты убийств
- **Систему управления** (ControlSystem, ClientInput, IPlayerControllable)

Каждый файл приведён в точности как в оригинальном проекте Sandbox от Facepunch.

## Шаг 2.1: Extension-методы — `Code/Utility/Extensions.cs`

### Что это и зачем

**Extension-методы** (методы расширения) — это механизм C#, позволяющий «добавить» метод к существующему типу, не изменяя его исходный код. Мы добавляем к типу `Vector3` метод `WithAimCone()`, который применяет случайный разброс к направлению — так работает **конус разброса оружия** (aim cone / weapon spread).

Когда игрок стреляет, пуля летит не идеально прямо, а с небольшим случайным отклонением. Чем больше `degrees` — тем шире конус, тем менее точное оружие.

### Полный код

Создай файл `Code/Utility/Extensions.cs`:

```csharp
public static class Extensions
{
	public static Vector3 WithAimCone( this Vector3 direction, float degrees )
	{
		var angle = Rotation.LookAt( direction );
		angle *= new Angles( Game.Random.Float( -degrees / 2.0f, degrees / 2.0f ), Game.Random.Float( -degrees / 2.0f, degrees / 2.0f ), 0 );
		return angle.Forward;
	}

	public static Vector3 WithAimCone( this Vector3 direction, float horizontalDegrees, float verticalDegrees )
	{
		var angle = Rotation.LookAt( direction );
		angle *= new Angles( Game.Random.Float( -verticalDegrees / 2.0f, verticalDegrees / 2.0f ), Game.Random.Float( -horizontalDegrees / 2.0f, horizontalDegrees / 2.0f ), 0 );
		return angle.Forward;
	}
}
```

### Подробный разбор

#### Extension-методы в C#

Чтобы создать extension-метод, нужно:
1. Класс должен быть `static`
2. Метод должен быть `static`
3. Первый параметр помечается ключевым словом `this`

```csharp
// Объявление:
public static Vector3 WithAimCone( this Vector3 direction, float degrees )

// Использование — вызывается КАК БУДТО это метод Vector3:
var spreadDirection = aimDirection.WithAimCone( 5.0f );

// Что эквивалентно:
var spreadDirection = Extensions.WithAimCone( aimDirection, 5.0f );
```

#### Как работает `Rotation.LookAt`

`Rotation.LookAt( direction )` — создаёт вращение (кватернион), которое «смотрит» в указанном направлении. Если `direction = Vector3.Forward`, вращение будет нулевым (без поворота). Если `direction` указывает вправо — вращение повернёт на 90° по yaw.

#### Алгоритм разброса

1. Берём направление выстрела `direction`
2. Создаём из него вращение: `Rotation.LookAt( direction )`
3. Добавляем случайное отклонение по pitch (вверх-вниз) и yaw (влево-вправо)
4. `Game.Random.Float( -degrees/2, degrees/2 )` — случайное число в диапазоне ±половина конуса
5. `angle.Forward` — извлекаем новое направление из модифицированного вращения

#### Две перегрузки

- **Одинаковый разброс** (`float degrees`) — конус одинаковый по горизонтали и вертикали. Подходит для большинства оружия.
- **Разный разброс** (`float horizontalDegrees, float verticalDegrees`) — позволяет задать разный конус по осям. Например, дробовик может быть шире по горизонтали.

#### Какие системы используют

- Система стрельбы оружия (`BaseWeapon`, `PistolWeapon`, `SMGWeapon` и т.д.)
- Любой код, которому нужен случайный разброс направления

## Шаг 2.2: Масштабирование HUD — `Code/Utility/HUD.cs`

### Что это и зачем

Когда ты рисуешь интерфейс (HUD), размеры элементов задаются в пикселях. Но на мониторе 1920×1080 и на мониторе 3840×2160 один и тот же элемент в 100 пикселей будет выглядеть совершенно по-разному — на 4K он будет вдвое меньше.

Класс `Hud` решает эту проблему: он вычисляет **коэффициент масштабирования** относительно эталонного разрешения 1080p.

### Полный код

Создай файл `Code/Utility/HUD.cs`:

```csharp
﻿namespace Sandbox;

internal static class Hud
{
	public static float Scale => Screen.Height / 1080.0f;
}
```

### Подробный разбор

| Элемент | Объяснение |
|---------|-----------|
| `namespace Sandbox` | Помещаем класс в пространство имён `Sandbox` — стандартное для кода игры в s&box |
| `internal static class Hud` | `internal` — доступен только внутри нашего проекта (не виден другим сборкам). `static` — нельзя создать экземпляр, все члены статические |
| `Screen.Height` | Свойство движка s&box, возвращающее высоту экрана в пикселях |
| `/ 1080.0f` | Делим на эталонное разрешение. На 1080p `Scale = 1.0`, на 1440p `Scale ≈ 1.33`, на 4K `Scale = 2.0` |

#### Как используется

```csharp
// В коде UI:
var fontSize = 16 * Hud.Scale;        // 16px на 1080p, 32px на 4K
var padding = 10 * Hud.Scale;         // 10px на 1080p, 20px на 4K
var crosshairSize = 4 * Hud.Scale;    // Размер перекрестия
```

Это простейшая, но критически важная утилита — без неё интерфейс будет нечитаемым на высоких разрешениях или гигантским на низких.

## Шаг 2.3: Локальное хранилище данных — `Code/Utility/LocalData.cs`

### Что это и зачем

`LocalData` — это простая обёртка для сохранения данных на диск игрока. Данные хранятся в виде JSON-файлов в специальной папке `FileSystem.Data`, которую s&box выделяет каждому проекту.

Типичные применения:
- Сохранение пользовательских настроек (чувствительность мыши, яркость)
- Кеширование данных (списки избранных серверов)
- Любые данные, которые нужно сохранить между сессиями

### Полный код

Создай файл `Code/Utility/LocalData.cs`:

```csharp

/// <summary>
/// Quick data folder file storage, good for saving local data
/// </summary>
public static class LocalData
{
	/// <summary>
	/// Serialize <paramref name="value"/> and write it to <c>{key}.json</c> under <see cref="FileSystem.Data"/>.
	/// The directory hierarchy is created automatically.
	/// </summary>
	public static void Set<T>( string key, T value )
	{
		var path = KeyToPath( key );
		var dir = System.IO.Path.GetDirectoryName( path );

		if ( !string.IsNullOrEmpty( dir ) && !FileSystem.Data.DirectoryExists( dir ) )
			FileSystem.Data.CreateDirectory( dir );

		FileSystem.Data.WriteJson( path, value );
	}

	/// <summary>
	/// Read and deserialize the value stored at <paramref name="key"/>.
	/// Returns <paramref name="fallback"/> if the file doesn't exist or deserialization fails.
	/// </summary>
	public static T Get<T>( string key, T fallback = default )
	{
		var path = KeyToPath( key );

		if ( !FileSystem.Data.FileExists( path ) )
			return fallback;

		try
		{
			return FileSystem.Data.ReadJson<T>( path );
		}
		catch ( Exception ex )
		{
			Log.Warning( ex, $"[LocalData] Failed to read '{path}'" );
			return fallback;
		}
	}

	/// <summary>
	/// Returns true if a value has been stored at <paramref name="key"/>.
	/// </summary>
	public static bool Has( string key ) => FileSystem.Data.FileExists( KeyToPath( key ) );

	/// <summary>
	/// Delete the value stored at <paramref name="key"/>. No-op if it doesn't exist.
	/// </summary>
	public static void Delete( string key )
	{
		var path = KeyToPath( key );
		if ( FileSystem.Data.FileExists( path ) )
			FileSystem.Data.DeleteFile( path );
	}

	static string KeyToPath( string key ) => $"{key}.json";
}
```

### Подробный разбор

#### FileSystem.Data — что это?

В s&box каждый проект получает свою изолированную папку для данных. `FileSystem.Data` — это виртуальная файловая система, привязанная к этой папке. Ты не можешь случайно записать файл куда попало — только в выделенную директорию.

```
%appdata%/sbox/data/facepunch.sandbox/    ← это FileSystem.Data
├── settings.json                          ← LocalData.Set("settings", obj)
├── favorites.json                         ← LocalData.Set("favorites", list)
└── profiles/
    └── player1.json                       ← LocalData.Set("profiles/player1", obj)
```

#### Обобщённые методы (Generics)

```csharp
public static void Set<T>( string key, T value )
```

`<T>` — это **параметр типа**. Метод работает с любым типом данных:

```csharp
LocalData.Set( "volume", 0.8f );                    // T = float
LocalData.Set( "name", "Player1" );                  // T = string
LocalData.Set( "favorites", new List<string>() );    // T = List<string>
LocalData.Set( "config", new MyConfig() );            // T = MyConfig
```

Компилятор сам определяет `T` по переданному значению. Это называется **вывод типа** (type inference).

#### Автоматическое создание директорий

```csharp
var dir = System.IO.Path.GetDirectoryName( path );
if ( !string.IsNullOrEmpty( dir ) && !FileSystem.Data.DirectoryExists( dir ) )
    FileSystem.Data.CreateDirectory( dir );
```

Если ключ содержит `/` (например, `"profiles/player1"`), путь к файлу будет `profiles/player1.json`. Метод автоматически создаёт папку `profiles/`, если её нет.

#### Безопасное чтение с fallback

```csharp
public static T Get<T>( string key, T fallback = default )
```

- `fallback = default` — значение по умолчанию: `null` для классов, `0` для чисел, `false` для `bool`
- Если файл не существует — возвращается `fallback`
- Если JSON повреждён — ловим исключение, логируем и возвращаем `fallback`
- Это **никогда не крашит игру** — безопасный подход

#### KeyToPath — простейшая конвертация

```csharp
static string KeyToPath( string key ) => $"{key}.json";
// "settings" → "settings.json"
// "profiles/player1" → "profiles/player1.json"
```

#### Какие системы используют

- Сохранение настроек игры
- Система инвентаря (персистентные данные)
- Любая система, которой нужно сохранить данные между сессиями

## Шаг 2.4: Система «кук» для настроек — `Code/Utility/CookieSource.cs`

### Что это и зачем

В Sandbox у каждого инструмента (Tool Gun) есть настройки: цвет шарика, материал верёвки, сила пружины и т.д. Эти настройки должны **сохраняться между сессиями** — чтобы при следующем заходе игрок не настраивал всё заново.

`CookieSource` — это система, которая автоматически сохраняет и загружает свойства объекта в **Game.Cookies** (хранилище «кук» движка s&box). Она использует **рефлексию** — способность программы анализировать собственную структуру во время выполнения.

### Полный код

Создай файл `Code/Utility/CookieSource.cs`:

```csharp
﻿using System.Text.Json;
using System.Text.Json.Serialization;

/// <summary>
/// We want to save the properties of this instance as cookies.
/// </summary>
public interface ICookieSource
{
	/// <summary>
	/// Prefix to put before the names of any cookies, without a trailing period.
	/// For example: <c>"tool.balloon"</c>
	/// </summary>
	string CookiePrefix { get; }
}

/// <summary>
/// Extension methods for <see cref="ICookieSource"/>.
/// </summary>
public static class CookieSourceExtensions
{
	private static bool IsCookieProperty( PropertyDescription property )
	{
		// TODO: maybe we want to support static properties too?

		if ( property.IsStatic ) return false;
		if ( !property.HasAttribute<PropertyAttribute>() ) return false;
		if ( property.HasAttribute<JsonIgnoreAttribute>() ) return false;
		if ( !property.CanWrite || !property.CanRead ) return false;

		return true;
	}

	private static string GetCookieName( PropertyDescription property, string prefix )
	{
		var name = property.GetCustomAttribute<JsonPropertyNameAttribute>()?.Name ?? property.Name.ToLowerInvariant();

		return !string.IsNullOrEmpty( prefix ) ? $"{prefix}.{name}" : name;
	}

	extension( ICookieSource source )
	{
		private IEnumerable<(string CookieName, PropertyDescription Property)> GetCookieProperties()
		{
			var typeDesc = TypeLibrary.GetType( source.GetType() );
			if ( typeDesc is null ) return [];

			var prefix = source.CookiePrefix;

			return typeDesc.Properties.Where( IsCookieProperty )
				.Select( x => (GetCookieName( x, prefix ), x) );
		}

		/// <summary>
		/// Saves any instance properties with a <see cref="PropertyAttribute"/>, excluding any
		/// with <see cref="JsonIgnoreAttribute"/>, into <see cref="Game.Cookies"/>.
		/// </summary>
		public void SaveCookies()
		{
			foreach ( var (cookieName, property) in source.GetCookieProperties() )
			{
				try
				{
					var cookieValue = property.GetValue( source );

					Game.Cookies.SetString( cookieName, JsonSerializer.Serialize( cookieValue, property.PropertyType ) );
				}
				catch ( Exception ex )
				{
					Log.Warning( ex, $"Exception while saving cookie \"{cookieName}\"." );
				}
			}
		}

		/// <summary>
		/// Loads any instance properties with a <see cref="PropertyAttribute"/>, excluding any
		/// with <see cref="JsonIgnoreAttribute"/>, from <see cref="Game.Cookies"/>.
		/// </summary>
		public void LoadCookies()
		{
			foreach ( var (cookieName, property) in source.GetCookieProperties() )
			{
				if ( !Game.Cookies.TryGetString( cookieName, out var jsonString ) ) continue;

				try
				{
					var cookieValue = JsonSerializer.Deserialize( jsonString, property.PropertyType );

					property.SetValue( source, cookieValue );
				}
				catch ( Exception ex )
				{
					Log.Warning( ex, $"Exception while loading cookie \"{cookieName}\"." );
				}
			}
		}
	}
}
```

### Подробный разбор

#### Интерфейс ICookieSource

```csharp
public interface ICookieSource
{
    string CookiePrefix { get; }
}
```

Любой класс, который хочет сохранять свои свойства, реализует этот интерфейс и указывает **префикс**. Например, для инструмента «Шарик»:

```csharp
public class BalloonTool : BaseTool, ICookieSource
{
    public string CookiePrefix => "tool.balloon";

    [Property] public Color BalloonColor { get; set; } = Color.Red;
    [Property] public float BalloonSize { get; set; } = 1.0f;
}
```

Куки сохранятся как:
- `tool.balloon.ballooncolor` → `"{"r":1,"g":0,"b":0,"a":1}"`
- `tool.balloon.balloonsize` → `"1.0"`

#### Game.Cookies — хранилище движка

`Game.Cookies` — это встроенное в s&box хранилище ключ-значение. Оно:
- Сохраняется автоматически между сессиями
- Привязано к проекту (разные игры — разные куки)
- Доступно только на клиенте (локальные данные)

#### TypeLibrary — рефлексия в s&box

s&box имеет собственную систему рефлексии `TypeLibrary`, которая быстрее стандартной .NET рефлексии:

```csharp
var typeDesc = TypeLibrary.GetType( source.GetType() );
// typeDesc.Properties — список всех свойств типа
// property.HasAttribute<PropertyAttribute>() — есть ли атрибут [Property]?
// property.GetValue( source ) — получить значение свойства
// property.SetValue( source, value ) — установить значение свойства
```

#### Фильтрация свойств

Не все свойства сохраняются. `IsCookieProperty` проверяет:

| Условие | Зачем |
|---------|-------|
| `property.IsStatic == false` | Статические свойства не принадлежат экземпляру |
| `property.HasAttribute<PropertyAttribute>()` | Только свойства с `[Property]` — это осознанный выбор автора |
| `!property.HasAttribute<JsonIgnoreAttribute>()` | `[JsonIgnore]` = «не сохранять это» |
| `property.CanWrite && property.CanRead` | Должны быть и getter, и setter |

#### Новый синтаксис extension в C# — `extension( ICookieSource source )`

Это **новейший синтаксис C#** (extensions). Вместо классических extension-методов с `this`, в s&box используется блок `extension(...)`:

```csharp
extension( ICookieSource source )
{
    public void SaveCookies() { ... }
    public void LoadCookies() { ... }
}
```

Это позволяет группировать extension-методы для одного типа и даже определять приватные вспомогательные методы внутри блока.

#### Сериализация через System.Text.Json

```csharp
JsonSerializer.Serialize( cookieValue, property.PropertyType )
JsonSerializer.Deserialize( jsonString, property.PropertyType )
```

Значения свойств конвертируются в JSON-строки и обратно. Это позволяет сохранять любые типы: числа, строки, цвета, перечисления, вложенные объекты.

#### Какие системы используют

- Все инструменты Tool Gun (Balloon, Rope, Spring, Weld и т.д.)
- Любой компонент, который хочет сохранять свои настройки между сессиями

## Шаг 2.5: Режимы сортировки пакетов — `Code/Utility/PackageSortMode.cs`

### Что это и зачем

В s&box есть встроенный магазин пакетов (asset store), где можно скачивать модели, карты, инструменты. Пакеты можно сортировать: по популярности, по новизне, по трендам, случайно. Этот enum + extension-метод задают типизированный способ работы с сортировкой.

### Полный код

Создай файл `Code/Utility/PackageSortMode.cs`:

```csharp
﻿/// <summary>
/// A typed sort mode for finding packages.
/// </summary>
public enum PackageSortMode
{
	Popular,
	Newest,
	Trending,
	Random
}

public static class PackageSortModeExtensions
{
	/// <summary>
	/// Returns the API supported string equivalent of this sort mode.
	/// </summary>
	public static string ToIdentifier( this PackageSortMode sortMode )
	{
		return sortMode switch
		{
			PackageSortMode.Popular => "popular",
			PackageSortMode.Newest => "newest",
			PackageSortMode.Trending => "trending",
			PackageSortMode.Random => "random",
			_ => "popular"
		};
	}
}
```

### Подробный разбор

#### Enum (перечисление)

```csharp
public enum PackageSortMode
{
    Popular,    // = 0
    Newest,     // = 1
    Trending,   // = 2
    Random      // = 3
}
```

Enum — это тип данных с фиксированным набором значений. Внутри это целые числа (0, 1, 2, 3), но в коде ты работаешь с читаемыми именами.

#### Switch-выражение (C# 8+)

```csharp
return sortMode switch
{
    PackageSortMode.Popular => "popular",
    _ => "popular"   // _ = все остальные случаи (default)
};
```

Это **switch-выражение** — компактная форма `switch`, которая возвращает значение. Каждая ветка — `паттерн => результат`. `_` — wildcard, ловит всё, что не совпало.

#### Зачем отдельный метод?

API s&box принимает строки (`"popular"`, `"newest"`), а в коде удобнее работать с enum-ами (автодополнение, проверка на этапе компиляции). `ToIdentifier()` — мост между ними.

## Шаг 2.6: Конвейер камеры — `Code/Utility/CameraSetup.cs`

### Что это и зачем

Камера в игре — это не просто «глаза игрока». К ней применяется множество эффектов: тряска от взрыва, отдача оружия, покачивание при ходьбе, шум. Все они должны работать **в определённом порядке**.

`CameraSetup` создаёт **конвейер** (pipeline) с тремя этапами:
1. **PreSetup** — эффекты ДО вью-модели (модели оружия в руках)
2. **Setup** — позиционирование вью-модели
3. **PostSetup** — эффекты ПОСЛЕ вью-модели (включая её)

### Полный код

Создай файл `Code/Utility/CameraSetup.cs`:

```csharp

/// <summary>
/// Creates a bunch of callbacks, allowing finer control over applying camera effects
/// </summary>
public sealed class CameraSetup : Component
{
	protected override void OnPreRender()
	{
		var cc = GetComponent<CameraComponent>();
		if ( cc is null ) return;

		ICameraSetup.Post( x => x.PreSetup( cc ) );
		ICameraSetup.Post( x => x.Setup( cc ) );
		ICameraSetup.Post( x => x.PostSetup( cc ) );
	}
}


public interface ICameraSetup : ISceneEvent<ICameraSetup>
{
	// Effects before viewmodel
	public void PreSetup( CameraComponent cc ) { }

	// Place viewmodel
	public void Setup( CameraComponent cc ) { }

	// Effects including viewmodel
	public void PostSetup( CameraComponent cc ) { }
}
```

### Подробный разбор

#### Component — базовый класс компонентов s&box

В s&box архитектура основана на **компонентах** (как в Unity). `Component` — базовый класс для всех компонентов. Каждый компонент прикрепляется к `GameObject`.

#### OnPreRender — когда вызывается?

```
Каждый кадр (frame):
  OnUpdate()        ← Логика
  OnPreRender()     ← Подготовка к рендерингу (здесь!)
  [Рендеринг GPU]   ← Камера рисует сцену
```

`OnPreRender()` вызывается **непосредственно перед отрисовкой кадра**. Это идеальный момент для настройки камеры, потому что:
- Вся логика уже выполнена (позиция игрока обновлена)
- Камера ещё не начала рисовать (можно менять её параметры)

#### ISceneEvent<T> — система событий сцены

```csharp
public interface ICameraSetup : ISceneEvent<ICameraSetup>
```

`ISceneEvent<T>` — это паттерн s&box для **глобальных событий в сцене**. Любой компонент, реализующий `ICameraSetup`, автоматически подписывается на события камеры.

```csharp
ICameraSetup.Post( x => x.PreSetup( cc ) );
```

`Post()` — вызывает метод `PreSetup` на **всех** компонентах в сцене, которые реализуют `ICameraSetup`. Это как широковещательное (broadcast) сообщение.

#### Интерфейс с реализацией по умолчанию

```csharp
public void PreSetup( CameraComponent cc ) { }
```

Методы имеют **пустую реализацию по умолчанию** (C# 8+). Это значит, что класс может реализовать только те методы, которые ему нужны:

```csharp
// Этот класс обрабатывает только PostSetup:
public class MyShake : Component, ICameraSetup
{
    void ICameraSetup.PostSetup( CameraComponent cc )
    {
        cc.WorldRotation *= someShake;
    }
    // PreSetup и Setup — используют пустую реализацию по умолчанию
}
```

#### Какие системы используют

- `CameraNoiseSystem` (тряска, отдача, удары)
- `EnvironmentShake` (тряска от взрывов)
- Система вью-моделей оружия
- Любой компонент, влияющий на камеру

## Шаг 2.7: Эффекты крови — `Code/Utility/Effects.cs`

### Что это и зачем

Когда пуля попадает в игрока, на ближайшей стене должен остаться **кровавый декал** (наклейка). Класс `Effects` управляет спавном таких декалей.

### Полный код

Создай файл `Code/Utility/Effects.cs`:

```csharp
namespace Sandbox;

public class Effects( Scene scene ) : GameObjectSystem<Effects>( scene )
{
	public List<Material> BloodDecalMaterials { get; set; } =
	[
		Cloud.Material( "jase.bloodsplatter08" ),
		Cloud.Material( "jase.bloodsplatter07" ),
		Cloud.Material( "jase.bloodsplatter06" ),
		Cloud.Material( "jase.bloodsplatter05" ),
		Cloud.Material( "jase.bloodsplatter04" )
	];

	public void SpawnBlood( Vector3 hitPosition, Vector3 direction, float damage = 50.0f )
	{
		const float BloodEjectDistance = 256.0f;
		var tr = Game.ActiveScene.Trace.Ray( new Ray( hitPosition, -direction ), BloodEjectDistance )
			.WithoutTags( "player" )
			.Run();

		if ( !tr.Hit ) return;

		var material = Random.Shared.FromList( BloodDecalMaterials );
		if ( !material.IsValid() ) return;

		var gameObject = Game.ActiveScene.CreateObject();
		gameObject.Name = "Blood splatter";
		gameObject.WorldPosition = tr.HitPosition + tr.Normal;
		gameObject.WorldRotation = Rotation.LookAt( -tr.Normal );
		gameObject.WorldRotation *= Rotation.FromAxis( Vector3.Forward, Game.Random.Float( 0, 360 ) );
	}
}
```

### Подробный разбор

#### GameObjectSystem<T> — синглтон-система

```csharp
public class Effects( Scene scene ) : GameObjectSystem<Effects>( scene )
```

`GameObjectSystem<T>` — это **система**, которая существует в единственном экземпляре на сцену. В отличие от компонентов (которые привязаны к GameObject), системы — глобальные. Доступ: `Effects.Current.SpawnBlood(...)`.

Обрати внимание на синтаксис: `Effects( Scene scene )` — это **primary constructor** (C# 12). Параметр `scene` передаётся в конструктор базового класса.

#### Cloud.Material — загрузка ресурсов из облака

```csharp
Cloud.Material( "jase.bloodsplatter08" )
```

s&box имеет **облачную систему ресурсов**. Вместо того чтобы включать текстуры крови в проект, мы загружаем их по идентификатору пакета. `jase.bloodsplatter08` — это пакет пользователя `jase`, содержащий текстуру крови.

#### Алгоритм спавна крови

1. Из точки попадания (`hitPosition`) пускаем луч **в обратном направлении** (`-direction`) — ищем стену за спиной жертвы
2. `WithoutTags("player")` — игнорируем других игроков
3. Если луч попал в стену — создаём объект-декал
4. Позиция: точка попадания + смещение на 1 единицу по нормали (чтобы декал не «утонул» в стене)
5. Поворот: декал смотрит от стены + случайный поворот 0–360° (чтобы декали не были одинаковыми)

#### Какие системы используют

- Система здоровья (при получении урона)
- Система оружия (при попадании)

## Шаг 2.8: Тряска окружения — `Code/Utility/EnvironmentShake.cs`

### Что это и зачем

`EnvironmentShake` — это компонент, который можно разместить на любом объекте сцены (взрыв, землетрясение, механизм). Он трясёт камеру игрока тем сильнее, чем ближе игрок к источнику. Тряска затухает с расстоянием и (опционально) со временем.

### Полный код

Создай файл `Code/Utility/EnvironmentShake.cs`:

```csharp

[Alias( "EnvShake" )]
public sealed class EnvironmentShake : Component, Component.ITemporaryEffect, ICameraSetup
{
	[Property] public float Rate { get; set; } = 10.0f;
	[Property] public Angles Scale { get; set; } = new Angles( 100, 100, 0 );
	[Property] public float MaxDistance { get; set; } = 1024;
	[Property] public Curve DistanceCurve { get; set; } = new Curve( new( 0, 1 ), new( 1, 0 ) );
	[FeatureEnabled( "Timed" ), Property] public bool TimeLimited { get; set; } = false;
	[Feature( "Timed" ), Property] public float TimeLength { get; set; } = 5;
	[Feature( "Timed" ), Property] public Curve TimeCurve { get; set; } = new Curve( new( 0, 1 ), new( 1, 0 ) );
	[Sync] public TimeUntil EndTime { get; set; }

	public bool IsActive => !TimeLimited || EndTime > 0;

	protected override void OnEnabled()
	{
		EndTime = TimeLength;
	}

	void ICameraSetup.PostSetup( CameraComponent camera )
	{
		var distance = camera.WorldPosition.Distance( WorldPosition );
		var distanceDelta = (distance / MaxDistance).Clamp( 0, 1 );

		var amount = DistanceCurve.EvaluateDelta( distanceDelta ) * GamePreferences.Screenshake;

		var noisex = Sandbox.Utility.Noise.Perlin( Time.Now * Rate, 0 ).Remap( 0, 1, -1, 1 );
		var noisey = Sandbox.Utility.Noise.Perlin( Time.Now * Rate + 830, 0 ).Remap( 0, 1, -1, 1 );
		var noisez = Sandbox.Utility.Noise.Perlin( Time.Now * Rate + 340, 0 ).Remap( 0, 1, -1, 1 );

		if ( TimeLimited )
		{
			var timeDelta = (EndTime.Relative / TimeLength).Remap( 1, 0 );

			if ( timeDelta >= 1 )
				return;

			amount *= TimeCurve.Evaluate( timeDelta );
		}

		camera.LocalRotation *= Rotation.From( noisex * Scale.pitch, noisey * Scale.yaw, noisez * Scale.roll ) * amount;
	}
}
```

### Подробный разбор

#### Атрибуты компонента

| Атрибут | Назначение |
|---------|-----------|
| `[Alias("EnvShake")]` | Альтернативное имя для поиска компонента (обратная совместимость) |
| `[Property]` | Свойство видно в инспекторе редактора s&box |
| `[FeatureEnabled("Timed")]` | Чекбокс, включающий/выключающий группу свойств |
| `[Feature("Timed")]` | Свойство видно только когда `TimeLimited = true` |
| `[Sync]` | Свойство синхронизируется по сети (сервер → клиент) |

#### Component.ITemporaryEffect

Интерфейс, сигнализирующий движку: «этот компонент — временный эффект». Свойство `IsActive` определяет, жив ли эффект. Когда `IsActive` становится `false`, движок может автоматически уничтожить объект.

#### Шум Перлина для тряски

```csharp
var noisex = Sandbox.Utility.Noise.Perlin( Time.Now * Rate, 0 ).Remap( 0, 1, -1, 1 );
```

**Шум Перлина** — это плавная случайная функция. В отличие от `Random.Float()`, который даёт резкие скачки, Perlin-шум создаёт **гладкие волны** — это выглядит как натуральная тряска.

- `Time.Now * Rate` — скорость колебаний. Чем выше `Rate`, тем быстрее тряска
- Разные смещения (`830`, `340`) — чтобы три оси тряслись независимо
- `.Remap( 0, 1, -1, 1 )` — преобразуем диапазон из [0,1] в [-1,1]

#### Кривые (Curve) — затухание

```csharp
[Property] public Curve DistanceCurve { get; set; } = new Curve( new( 0, 1 ), new( 1, 0 ) );
```

`Curve` — это кривая, задаваемая точками. `(0, 1)` → `(1, 0)` означает: на расстоянии 0 эффект = 100%, на максимальном расстоянии = 0%. В редакторе s&box эту кривую можно редактировать мышкой.

#### GamePreferences.Screenshake

Глобальная настройка: насколько игрок хочет тряску экрана. Если игроку тошнит от тряски — он может её уменьшить или выключить в настройках.

## Шаг 2.9: Звуки столкновений — `Code/Utility/SelfCollisionSound.cs`

### Что это и зачем

Когда физический объект (пропы, бочки, ящики) ударяется обо что-то, s&box по умолчанию проигрывает звуки столкновений для **обеих** поверхностей. Но иногда нужно, чтобы звучала только поверхность самого объекта (например, металлическая бочка должна звенеть, а не воспроизводить ещё и звук бетона, об который она ударилась).

### Полный код

Создай файл `Code/Utility/SelfCollisionSound.cs`:

```csharp
/// <summary>
/// Only play the collision sound from this object's surface - not both the impacting surface and this surface.
/// </summary>
public sealed class SelfCollisionSound : Component, Component.ICollisionListener
{
	protected override void OnStart()
	{
		// Turn off default collision sounds
		foreach ( var body in GetComponents<Rigidbody>().Select( x => x.PhysicsBody ) )
		{
			body.EnableCollisionSounds = false;
		}
	}
	void ICollisionListener.OnCollisionStart( Sandbox.Collision collision )
	{
		Play( collision.Self.Shape, collision.Self.Surface, collision.Contact.Point, MathF.Abs( collision.Contact.NormalSpeed ) );
	}

	public void Play( PhysicsShape shape, Surface surface, in Vector3 position, float speed )
	{
		if ( !shape.IsValid() ) return;
		if ( !shape.Body.IsValid() ) return;
		if ( speed < 50.0f ) return;
		if ( surface == null ) return;

		surface.PlayCollisionSound( position, speed );
	}
}
```

### Подробный разбор

#### Component.ICollisionListener

Это интерфейс обратного вызова движка. Когда физическое тело сталкивается с чем-то, движок вызывает `OnCollisionStart` на всех компонентах, реализующих `ICollisionListener`.

#### Отключение стандартных звуков

```csharp
foreach ( var body in GetComponents<Rigidbody>().Select( x => x.PhysicsBody ) )
{
    body.EnableCollisionSounds = false;
}
```

Сначала отключаем стандартное поведение, потом вручную проигрываем только звук «нашей» поверхности.

#### Фильтрация по скорости

```csharp
if ( speed < 50.0f ) return;
```

Если объект еле касается поверхности (скорость < 50 единиц) — звук не играет. Это предотвращает «дребезжание» от мелких вибраций.

#### Ключевое слово `in`

```csharp
public void Play( PhysicsShape shape, Surface surface, in Vector3 position, float speed )
```

`in` — модификатор, означающий «передать по ссылке, но без возможности изменения». Для `Vector3` (структура, 12 байт) это небольшая оптимизация — вместо копирования передаётся указатель.

## Шаг 2.11: Трассировщик пуль — `Code/Utility/Tracer.cs`

### Что это и зачем

Когда оружие стреляет, между дулом и точкой попадания рисуется **линия трассера** — светящийся след пули. `Tracer` — это `Renderer` (компонент отрисовки), который рисует анимированную линию, летящую от точки A к точке B.

### Полный код

Создай файл `Code/Utility/Tracer.cs`:

```csharp
﻿public sealed class Tracer : Renderer, Component.ExecuteInEditor, Component.ITemporaryEffect
{
	[Header( "Position" )]
	[Property, Feature( "Tracer" )] public WorldPoint EndPoint { get; set; }

	[Header( "Speed" )]
	[Property, Feature( "Tracer" )] public float DistancePerSecond { get; set; } = 1.0f;
	[Property, Feature( "Tracer" )] public float Length { get; set; } = 100.0f;
	[Property, Feature( "Tracer" )] public float StartDistance { get; set; } = 200.0f;

	[Header( "Rendering" )]
	[Property, Feature( "Tracer" )] public Gradient LineColor { get; set; } = new Gradient( new Gradient.ColorFrame( 0, Color.White ), new Gradient.ColorFrame( 1, Color.White.WithAlpha( 0 ) ) );
	[Property, Feature( "Tracer" )] public Curve LineWidth { get; set; } = new Curve( new Curve.Frame( 0, 2 ), new Curve.Frame( 1, 0 ) );
	[Property, Feature( "Tracer" )] public SceneLineObject.CapStyle StartCap { get; set; }
	[Property, Feature( "Tracer" )] public SceneLineObject.CapStyle EndCap { get; set; }

	[Group( "Rendering" )]
	[Property] public bool Opaque { get; set; } = true;

	[Group( "Rendering" )]
	[Property] public bool CastShadows { get; set; } = true;

	[Property, FeatureEnabled( "Light", Icon = "💡" )]
	public bool EnableLight { get; set; }

	[Property, Feature( "Light" )]
	public Gradient LightColor { get; set; } = new Gradient( new Gradient.ColorFrame( 0, Color.White ), new Gradient.ColorFrame( 1, Color.White ) );

	[Property, Feature( "Light" )]
	public Curve LightRadius { get; set; } = new Curve( new Curve.Frame( 0, 128 ), new Curve.Frame( 0.5f, 256 ), new Curve.Frame( 1, 128 ) );

	[Property, Feature( "Light" ), Range( 0, 1 )]
	public float LightPosition { get; set; } = 0;

	bool ITemporaryEffect.IsActive => !_finished;

	float _distance = 0.0f;
	bool _finished = false;

	SceneLineObject _so;
	SceneLight _light;
	protected override void OnEnabled()
	{
		_so = new SceneLineObject( Scene.SceneWorld );
		_so.Transform = Transform.World;

		_distance = StartDistance;
	}

	protected override void OnDisabled()
	{
		_so?.Delete();
		_so = null;

		_light?.Delete();
		_light = null;
	}

	protected override void OnUpdate()
	{
		var len = WorldPosition.Distance( EndPoint.Get() );

		_distance += DistancePerSecond * Time.Delta;

		if ( _distance > len + Length )
		{
			_finished = true;

			if ( Scene.IsEditor )
				_distance = 0;
		}
	}

	protected override void OnPreRender()
	{
		if ( !_so.IsValid() )
			return;

		var travel = EndPoint.Get() - WorldPosition;
		var maxlen = travel.Length;

		if ( _distance - Length > maxlen )
		{
			_so.RenderingEnabled = false;

			_light?.Delete();
			_light = null;
			return;
		}

		var midDistance = (_distance - Length * 0.5f).Clamp( 0, maxlen );
		var delta = midDistance.LerpInverse( 0, maxlen );

		if ( EnableLight )
		{
			_light ??= new SceneLight( Scene.SceneWorld );
			_light.Transform = Transform.World;
			_light.QuadraticAttenuation = 10;
			_light.LightColor = LightColor.Evaluate( delta );
			_light.ShadowsEnabled = false;
			_light.Radius = LightRadius.Evaluate( delta );
			_light.Position = WorldPosition + travel.Normal * (_distance - Length * LightPosition).Clamp( 0, maxlen - 5 );
		}
		else
		{
			_light?.Delete();
			_light = null;
		}

		_so.StartCap = StartCap;
		_so.EndCap = EndCap;
		_so.Opaque = Opaque;

		_so.RenderingEnabled = true;
		_so.Transform = WorldTransform;
		_so.Flags.CastShadows = CastShadows;
		_so.Attributes.Set( "BaseTexture", Texture.White );
		_so.Attributes.SetCombo( "D_BLEND", Opaque ? 0 : 1 );

		_so.StartLine();

		for ( float x = 0; x <= 1.0f; x += 0.1f )
		{
			var s = (_distance - Length * x).Clamp( 0, maxlen );
			var lineStart = (_distance - maxlen).Clamp( 0, Length ) / Length;

			_so.AddLinePoint( WorldPosition + travel.Normal * s, LineColor.Evaluate( x ), LineWidth.Evaluate( x ) );
		}

		{
			var s = (_distance - Length).Clamp( 0, maxlen );
			var lineStart = (_distance).Clamp( 0, Length ) / Length;
		}

		_so.EndLine();
	}
}

public struct WorldPoint
{
	[KeyProperty]
	public Vector3 Origin { get; set; }
	public GameObject Parent { get; set; }

	public Vector3 Get()
	{
		if ( Parent.IsValid() )
			return Parent.WorldTransform.PointToWorld( Origin );

		return Origin;
	}

	public static implicit operator WorldPoint( Vector3 v )
	{
		return new WorldPoint { Origin = v };
	}
}
```

### Подробный разбор

#### Renderer — визуальный компонент

`Renderer` — базовый класс для всех компонентов, которые рисуют что-то на экране. В отличие от `Component`, `Renderer` участвует в системе рендеринга движка.

#### Component.ExecuteInEditor

Этот интерфейс-маркер означает: «выполнять этот компонент даже в режиме редактора». Обычно компоненты работают только во время игры. Трассер нужен и в редакторе — для превью.

#### SceneLineObject — рисование линий

```csharp
_so = new SceneLineObject( Scene.SceneWorld );
_so.StartLine();
_so.AddLinePoint( position, color, width );
_so.EndLine();
```

Это низкоуровневый объект рендеринга — рисует линию в 3D-пространстве. Каждый кадр мы пересоздаём линию с обновлёнными позициями.

#### Анимация полёта

```csharp
_distance += DistancePerSecond * Time.Delta;
```

`_distance` — сколько трассер «пролетел». Каждый кадр увеличивается. Когда `_distance > len + Length` — трассер прошёл всю дистанцию и исчез.

- `StartDistance = 200` — трассер начинается не от дула, а на расстоянии 200 единиц (чтобы не перекрывать вью-модель)
- `Length = 100` — длина видимой части трассера

#### Gradient и Curve — визуальные параметры

- `Gradient LineColor` — цвет меняется вдоль линии (от яркого до прозрачного)
- `Curve LineWidth` — ширина меняется вдоль линии (от 2 до 0)

#### Динамическое освещение

Если `EnableLight = true`, вместе с трассером летит **источник света** (`SceneLight`). Это создаёт эффект светящейся пули, освещающей окружение.

```csharp
_light ??= new SceneLight( Scene.SceneWorld );
```

`??=` — оператор «присвоить, если null». Создаём свет только один раз.

#### WorldPoint — точка в мире с привязкой к объекту

```csharp
public struct WorldPoint
{
    public Vector3 Origin { get; set; }
    public GameObject Parent { get; set; }
}
```

`WorldPoint` — это позиция, которая может быть **привязана к объекту** (Parent). Если Parent задан, позиция вычисляется относительно него. Это нужно, если цель трассера — движущийся объект.

```csharp
public static implicit operator WorldPoint( Vector3 v )
```

**Неявное преобразование** — позволяет писать `tracer.EndPoint = new Vector3(100, 0, 0)` вместо `tracer.EndPoint = new WorldPoint { Origin = new Vector3(100, 0, 0) }`.

## Шаг 2.12: Система шума камеры — `Code/Utility/CameraNoise/`

### Что это и зачем

Это набор эффектов камеры: **удар** (punch), **отдача** (recoil), **тряска** (shake). Все они наследуются от `BaseCameraNoise` и управляются системой `CameraNoiseSystem`.

Когда игрок стреляет, создаётся `new Punch(...)` — камера дёргается вверх и плавно возвращается. Когда рядом взрыв — `new Shake(...)`. При стрельбе из автомата — `new Recoil(...)`.

### Файл 1: `Code/Utility/CameraNoise/CameraNoiseSystem.cs`

```csharp
﻿namespace Sandbox.CameraNoise;

public class CameraNoiseSystem : GameObjectSystem<CameraNoiseSystem>, ICameraSetup
{
	List<BaseCameraNoise> _all = new();

	public CameraNoiseSystem( Scene scene ) : base( scene )
	{
	}

	void ICameraSetup.PreSetup( Sandbox.CameraComponent cc )
	{
		foreach ( var effect in _all )
		{
			effect.Update();
		}

		_all.RemoveAll( x => x.IsDone );
	}

	void ICameraSetup.PostSetup( CameraComponent cc )
	{
		foreach ( var effect in _all )
		{
			effect.ModifyCamera( cc );
		}
	}

	public void Add( BaseCameraNoise noise )
	{
		_all.Add( noise );
	}
}

public abstract class BaseCameraNoise
{
	public float LifeTime { get; protected set; }
	public float CurrentTime { get; protected set; }
	public float Delta => CurrentTime.LerpInverse( 0, LifeTime, true );
	public float DeltaInverse => 1 - Delta;

	public BaseCameraNoise()
	{
		CameraNoiseSystem.Current.Add( this );
	}

	public virtual bool IsDone => CurrentTime > LifeTime;

	public virtual void Update()
	{
		CurrentTime += Time.Delta;
	}

	public virtual void ModifyCamera( CameraComponent cc ) { }
}
```

#### Подробный разбор CameraNoiseSystem

**Двухфазная обработка:**

1. `PreSetup` — обновляем таймеры всех эффектов и удаляем завершившиеся
2. `PostSetup` — применяем эффекты к камере

Зачем именно Post, а не Pre? Потому что камерные эффекты (тряска, отдача) должны применяться **после** позиционирования вью-модели — чтобы оружие в руках тряслось вместе с камерой.

**BaseCameraNoise — базовый класс:**

```csharp
public BaseCameraNoise()
{
    CameraNoiseSystem.Current.Add( this );
}
```

Конструктор **автоматически регистрирует** эффект в системе. Создал `new Punch(...)` — и он сразу работает. Не нужно вызывать никаких дополнительных методов.

- `Delta` — прогресс от 0 (начало) до 1 (конец)
- `DeltaInverse` — обратный прогресс от 1 до 0 (удобно для затухания)
- `IsDone` — когда `CurrentTime > LifeTime`, эффект завершён и будет удалён

### Файл 2: `Code/Utility/CameraNoise/Punch.cs`

```csharp
﻿namespace Sandbox.CameraNoise;

class Punch : BaseCameraNoise
{
	float deathTime;
	float lifeTime = 0.0f;

	public Punch( Vector3 target, float time, float frequency, float damp )
	{
		damping.Current = target * GamePreferences.Screenshake;
		damping.Target = 0;
		damping.Frequency = frequency;
		damping.Damping = damp;

		deathTime = time;
	}

	public override bool IsDone => deathTime <= 0;

	Vector3.SpringDamped damping;

	public override void Update()
	{
		deathTime -= Time.Delta;
		lifeTime += Time.Delta;

		damping.Update( Time.Delta );
	}

	public override void ModifyCamera( CameraComponent cc )
	{
		var amount = lifeTime.Remap( 0, 0.3f, 0, 1 );

		cc.WorldRotation *= new Angles( damping.Current * amount );
	}
}
```

#### Подробный разбор Punch

**Пружинное демпфирование (Spring-Damped):**

```csharp
Vector3.SpringDamped damping;
```

`Vector3.SpringDamped` — встроенный в s&box тип пружинной системы. Он симулирует поведение пружины:
- `Current` — текущее значение (начинается с `target`)
- `Target` — целевое значение (0 — камера возвращается в нормальное положение)
- `Frequency` — частота колебаний (чем выше, тем быстрее «пружина»)
- `Damping` — демпфирование (чем выше, тем быстрее затухают колебания)

Представь, что ты ударил по пружине: она отскакивает, качается и постепенно успокаивается. Именно так выглядит эффект «удара» по камере.

**Плавный ввод:**

```csharp
var amount = lifeTime.Remap( 0, 0.3f, 0, 1 );
```

Первые 0.3 секунды эффект плавно нарастает от 0 до 1. Это предотвращает резкий рывок в первый кадр.

### Файл 3: `Code/Utility/CameraNoise/Recoil.cs`

```csharp
﻿using Sandbox.Utility;

namespace Sandbox.CameraNoise;

/// <summary>
/// Creates a bunch of other common effects
/// </summary>
class Recoil : BaseCameraNoise
{
	public Recoil( float amount, float speed = 1 )
	{
		new RollShake() { Size = 0.5f * amount * GamePreferences.Screenshake, Waves = 3 * speed };
	}

	public override void ModifyCamera( CameraComponent cc )
	{
	}
}

/// <summary>
/// Shake the screen in a roll motion
/// </summary>
class RollShake : BaseCameraNoise
{
	public float Size { get; set; } = 3.0f;
	public float Waves { get; set; } = 3.0f;

	public RollShake()
	{
		LifeTime = 0.3f;
	}

	public override void ModifyCamera( CameraComponent cc )
	{
		var delta = Delta;
		var s = MathF.Sin( delta * MathF.PI * Waves * 2 );
		cc.WorldRotation *= new Angles( 0, 0, s * Size ) * Easing.EaseOut( DeltaInverse );
	}
}
```

#### Подробный разбор Recoil

**Recoil — фабрика эффектов:**

`Recoil` сам по себе ничего не делает (`ModifyCamera` пуст). Вместо этого в конструкторе он создаёт `RollShake` — визуальный эффект раскачивания. Это паттерн **фабрика** — один объект создаёт другие.

**RollShake — синусоидальная тряска:**

```csharp
var s = MathF.Sin( delta * MathF.PI * Waves * 2 );
cc.WorldRotation *= new Angles( 0, 0, s * Size ) * Easing.EaseOut( DeltaInverse );
```

- `MathF.Sin(...)` — синусоида создаёт волнообразное покачивание
- `Waves = 3` → 3 полных колебания за 0.3 секунды
- `new Angles( 0, 0, s * Size )` — вращение только по оси roll (наклон влево-вправо)
- `Easing.EaseOut( DeltaInverse )` — эффект затухает к концу (сначала сильно, потом слабо)

### Файл 4: `Code/Utility/CameraNoise/Shake.cs`

```csharp
﻿using Sandbox.Utility;

namespace Sandbox.CameraNoise;

class Shake : BaseCameraNoise
{
	float lifeTime;
	float deathTime;
	float amount = 0.0f;

	public Shake( float amount, float time )
	{
		this.amount = amount * GamePreferences.Screenshake;
		deathTime = time;
		lifeTime = time;
	}

	public override bool IsDone => deathTime <= 0;

	Vector3.SpringDamped damping;

	public override void Update()
	{
		deathTime -= Time.Delta;
		damping.Update( Time.Delta );
	}

	public override void ModifyCamera( CameraComponent cc )
	{
		var x = Noise.Perlin( Time.Now * 1000.0f, 2345 );
		var y = Noise.Perlin( Time.Now * 1000.0f, 21 );
		var z = Noise.Perlin( Time.Now * 1000.0f, 865 );

		var delta = MathX.Remap( deathTime, 0, lifeTime, 0, 1 );

		cc.WorldRotation *= new Angles( x, y, z ) * delta * amount;
	}
}
```

#### Подробный разбор Shake

В отличие от `Punch` (одиночный рывок), `Shake` — это **постоянная тряска** заданной длительности. Каждый кадр генерируется случайный поворот из шума Перлина, который плавно затухает (`delta` уменьшается от 1 до 0).

- `Time.Now * 1000.0f` — высокочастотный шум (быстрая тряска)
- Разные seed-ы (`2345`, `21`, `865`) — независимые оси
- `delta * amount` — линейное затухание

## Шаг 2.14: Система управления — `Code/Game/ControlSystem/`

### Что это и зачем

Когда игрок садится в машину, он управляет ею. Когда управляет турелью — поворачивает её. Система управления определяет, **кто** управляет **чем**, и рассылает команды управления.

### Файл 1: `Code/Game/ControlSystem/ClientInput.cs`

```csharp
﻿

using Sandbox.Utility;

public struct ClientInput
{
	readonly record struct State( Connection connection, PlayerController playerController );

	static State _currentState;

	static Connection Connection => _currentState.connection;

	public readonly bool IsEnabled => !string.IsNullOrWhiteSpace( Action );

	public string Action { get; set; }

	/// <summary>
	/// Returns an analog value between 0 and 1 representing how much the input is pressed
	/// </summary>
	public readonly float GetAnalog()
	{
		if ( !IsEnabled ) return 0;
		return Down() ? 1 : 0;
	}

	/// <summary>
	/// Returns true if button is currently held down
	/// </summary>
	public readonly bool Down()
	{
		if ( !IsEnabled ) return false;

		return Connection?.Down( Action ) ?? false;
	}

	/// <summary>
	/// Returns true if button was released
	/// </summary>
	public readonly bool Released()
	{
		if ( !IsEnabled ) return false;

		return Connection?.Released( Action ) ?? false;
	}

	/// <summary>
	/// Returns true if button was pressed
	/// </summary>
	public readonly bool Pressed()
	{
		if ( !IsEnabled ) return false;

		return Connection?.Pressed( Action ) ?? false;
	}

	internal static IDisposable PushScope( PlayerController player )
	{
		var previousState = _currentState;
		_currentState = new State( player?.Network?.Owner, player );

		return DisposeAction.Create( () => _currentState = previousState );
	}
}
```

#### Подробный разбор ClientInput

**Что такое `struct` vs `class`?**

`ClientInput` — это `struct` (структура), а не `class`. Структуры:
- Хранятся в стеке (быстрее)
- Копируются при присваивании (value type)
- Используются для маленьких, «лёгких» данных

**readonly record struct State:**

```csharp
readonly record struct State( Connection connection, PlayerController playerController );
```

Это **вложенная** запись (record). `record struct` — неизменяемая (readonly) структура с автоматически генерируемыми `Equals`, `GetHashCode`, `ToString`. Хранит текущего игрока и его сетевое подключение.

**PushScope — паттерн Scope/Disposable:**

```csharp
internal static IDisposable PushScope( PlayerController player )
{
    var previousState = _currentState;
    _currentState = new State( player?.Network?.Owner, player );
    return DisposeAction.Create( () => _currentState = previousState );
}
```

Это **паттерн scope** — временно устанавливаем контекст «кто сейчас управляет», а потом восстанавливаем предыдущий:

```csharp
using var scope = ClientInput.PushScope( player );
// Здесь ClientInput.Down("Jump") проверяет ввод конкретного player-а
// При выходе из using — восстанавливается предыдущий контекст
```

**Null-conditional и null-coalescing:**

```csharp
return Connection?.Down( Action ) ?? false;
```

- `Connection?` — если `Connection` = `null`, не вызываем `.Down()`, а возвращаем `null`
- `?? false` — если результат `null`, возвращаем `false`

### Файл 2: `Code/Game/ControlSystem/ControlSystem.cs`

```csharp
public class ControlSystem : GameObjectSystem<ControlSystem>
{
	// When each chair first became occupied. Used to sort seats so the earliest occupant is in charge
	private readonly Dictionary<BaseChair, RealTimeSince> _occupiedSince = new();

	public ControlSystem( Scene scene ) : base( scene )
	{
		Listen( Stage.StartFixedUpdate, 10, OnTick, "ControlSystem" );
	}

	void OnTick()
	{
		// TODO this should be more generic, some kind of interface?
		var driven = new HashSet<GameObject>();

		foreach ( var chair in GetSortedSeats() )
		{
			var builder = new LinkedGameObjectBuilder();
			builder.AddConnected( chair.GameObject );

			// Skip if a seat occupied earlier already claimed this
			if ( builder.Objects.Any( driven.Contains ) ) continue;
			driven.UnionWith( builder.Objects );

			RunControl( chair, builder );
		}
	}

	IEnumerable<BaseChair> GetSortedSeats()
	{
		foreach ( var chair in Scene.GetAll<BaseChair>() )
		{
			if ( !chair.IsValid() || !chair.IsOccupied )
				_occupiedSince.Remove( chair );
			else
				_occupiedSince.TryAdd( chair, 0 );
		}

		return Scene.GetAll<BaseChair>()
			.Where( c => c.IsValid() && c.IsOccupied )
			.OrderBy( c => (float)_occupiedSince.GetValueOrDefault( c, default ) );
	}

	void RunControl( BaseChair chair, LinkedGameObjectBuilder builder )
	{
		var player = chair.GetOccupant();
		if ( !player.IsValid() ) return;

		using var scope = ClientInput.PushScope( player );

		foreach ( var o in builder.Objects )
		{
			foreach ( var controllable in o.GetComponentsInChildren<IPlayerControllable>() )
			{
				controllable?.OnControl();
			}
		}
	}
}
```

#### Подробный разбор ControlSystem

**Listen — подписка на этап обновления:**

```csharp
Listen( Stage.StartFixedUpdate, 10, OnTick, "ControlSystem" );
```

Система подписывается на `StartFixedUpdate` — этап физического обновления. Число `10` — приоритет (порядок выполнения среди других систем). `OnTick` — метод, который будет вызываться каждый физический тик (по умолчанию 50 раз в секунду, как задано в `sandbox.sbproj`).

**Связанные объекты (LinkedGameObjectBuilder):**

Машина состоит из множества GameObject-ов: корпус, колёса, сиденья, турель. Все они **связаны** (constraints, joints). `LinkedGameObjectBuilder.AddConnected()` находит все связанные объекты — чтобы управление распространялось на всю «конструкцию».

**Приоритет по времени занятия:**

```csharp
.OrderBy( c => (float)_occupiedSince.GetValueOrDefault( c, default ) );
```

Если два игрока сидят в одной машине — управляет тот, кто сел **раньше**. `RealTimeSince` — автоматический таймер, который считает секунды с момента создания.

**HashSet для предотвращения двойного управления:**

```csharp
var driven = new HashSet<GameObject>();
if ( builder.Objects.Any( driven.Contains ) ) continue;
driven.UnionWith( builder.Objects );
```

Если конструкция уже управляется другим игроком — пропускаем.
