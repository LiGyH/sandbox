# 00.16 — Prefabs (префабы)

## Что мы делаем?

Разбираемся, что такое **префаб** — заготовка `GameObject`, которую можно переиспользовать. В Sandbox через префабы сделаны **все**: игрок, все виды оружия, пикапы, пропы, эффекты.

## Зачем нужен префаб?

Представь: ты настроил игрока — `GameObject` с компонентами `Player`, `CharacterController`, `ModelRenderer`, `PlayerInventory`, правильной моделью, правильной камерой. Теперь ты хочешь:

- спавнить нового игрока каждый раз, когда кто-то входит на сервер;
- использовать этого же игрока в другой сцене;
- если пофиксил баг в «шаблоне игрока» — чтобы он исправился и на карте А, и на карте Б, и везде, где игрок уже стоит.

Префаб решает это: **один шаблон → много копий, обновил шаблон → обновились все копии**.

## Создание префаба

В редакторе:
1. Настрой `GameObject` в сцене как надо (со всеми компонентами и настройками).
2. Правый клик на нём → **Convert to Prefab**.
3. Сохрани в папку проекта (`prefabs/player.prefab`).

Теперь это отдельный **файл-ассет** `.prefab`, и объект в сцене превращается в **экземпляр префаба** (подсвечивается синим в иерархии).

## Экземпляры префаба в сцене

`GameObject`-ы, которые являются экземплярами префаба, ведут себя особенно:
- **Подсвечены синим** в иерархии.
- **Нельзя** редактировать их структуру напрямую (добавлять/удалять детей) — это будет перезатёрто при обновлении префаба.
- **Можно переопределять свойства** — такие переопределения помечаются жёлтой точкой.
- Правый клик → **Unlink from Prefab** превратит копию в обычный набор `GameObject`-ов, отвязанных от шаблона.

## Спавн префаба в коде

Префаб подтягивается в код через `[Property] GameObject ...` — в инспекторе ты перетаскиваешь `.prefab`-файл:

```csharp
public sealed class MyGun : Component
{
    [Property] public GameObject BulletPrefab { get; set; }

    protected override void OnUpdate()
    {
        Assert.NotNull( BulletPrefab, "BulletPrefab не назначен!" );

        if ( Input.Pressed( "Attack1" ) )
        {
            // Клон в мировой точке
            GameObject bullet = BulletPrefab.Clone( WorldPosition );

            // Можно сразу настроить
            bullet.WorldRotation = WorldRotation;

            // Теперь пуля в сцене, с компонентом BulletComponent (как в префабе)
            var comp = bullet.GetComponent<BulletComponent>();
            comp.Velocity = Vector3.Forward * 1000;
        }
    }
}
```

### Варианты `Clone(...)`

```csharp
BulletPrefab.Clone();                      // в точке (0,0,0)
BulletPrefab.Clone( worldPos );
BulletPrefab.Clone( worldPos, worldRot );
BulletPrefab.Clone( new Transform( pos, rot, scale ) );
```

## Разорвать связь с префабом

Клонированный префаб **всё ещё связан** с исходным файлом. Если ты хочешь независимый `GameObject`, который не будет меняться при обновлении шаблона:

```csharp
bullet.BreakFromPrefab();
```

Используется редко, обычно связь с префабом полезна.

## Сетевые префабы

Для сетевых объектов префаб **обязателен** — клиент получает от сервера только «спавни префаб X», и по этому X он знает, какие компоненты и настройки накатить. Создание сетевого объекта:

```csharp
var enemy = EnemyPrefab.Clone( spawnPos );
enemy.NetworkSpawn();    // теперь он есть у всех клиентов
```

Подробнее: [00.21 — Создание сетевых объектов](00_21_Networked_Objects.md).

## Вложенные префабы

Префаб может **содержать другие префабы** как детей. Обновление вложенного префаба распространится и на сборный.

Пример: префаб `Car` содержит 4 экземпляра префаба `Wheel`. Если ты исправил `Wheel` — все 4 колеса в каждой машине обновятся.

## Префаб vs Scene

- **Scene** — это *один* уровень. Грузится одна за раз (не считая аддитивных).
- **Prefab** — это *один переиспользуемый объект*. В одной сцене могут быть сотни экземпляров одного префаба.

## Практические советы

- ✅ **Игрока делай префабом** — иначе спавнить его через `NetworkHelper` не получится.
- ✅ **Оружие — префабом** — чтобы у каждого игрока было своё, и дать можно было кодом.
- ✅ **Эффекты (взрывы, дым, искры) — префабом** — спавнишь по месту.
- ❌ **Статическую геометрию карты** префабом делать не надо — это просто `GameObject` в сцене.

## Результат

После этого этапа ты знаешь:

- ✅ Что такое префаб и зачем.
- ✅ Как создать префаб из объекта сцены.
- ✅ Как заспавнить префаб в коде через `[Property] GameObject` + `Clone()`.
- ✅ Что обновление файла префаба обновляет **все** его копии.
- ✅ Что сетевые объекты Sandbox спавнятся через `Clone()` + `NetworkSpawn()`.
