# 28.01 — Анимации: общая картина

## Что мы делаем?

Открываем большую тему — **как в s&box оживают модели**. До этого этапа мы относились к моделям как к «картинкам в мире»: поставил `SkinnedModelRenderer`, выбрал `.vmdl`, и оно стоит. На самом деле под капотом работает целый конвейер: **скелет → AnimGraph → параметры → позы → IK → морфы → события**. Эта фаза — про каждый из этих кирпичиков.

> 🧭 Эта Фаза 28 — теоретическо-практическая. Подэтапы 28.01–28.09 про анимации, 28.10–28.16 про шейдеры. Будут и теория, и разбор реального кода Sandbox.

## Из чего состоит анимация в s&box

```
┌──────────────────────────────────────────────────────────────────┐
│                          Модель (.vmdl)                          │
│   меш + скелет (bones) + морфы + список анимаций (.vanim)        │
└──────────────────────────────────────────────────────────────────┘
                               │
                               ▼
┌──────────────────────────────────────────────────────────────────┐
│                        AnimGraph (.vanmgrph)                     │
│   стейты, переходы, BlendSpace, параметры, IK-чейны, теги        │
└──────────────────────────────────────────────────────────────────┘
                               │
                               ▼
┌──────────────────────────────────────────────────────────────────┐
│                       SkinnedModelRenderer                       │
│   компонент в сцене: рендерит модель + крутит AnimGraph каждый   │
│   кадр; имеет API Set/Get параметров, SetIk/ClearIk, события.    │
└──────────────────────────────────────────────────────────────────┘
                               │
                               ▼
┌──────────────────────────────────────────────────────────────────┐
│                       Кадр на экране                             │
│   итоговая поза скелета + морфы + IK → меш деформирован → пиксели│
└──────────────────────────────────────────────────────────────────┘
```

## Ключевые понятия

| Понятие | Что это |
|---|---|
| **`.vmdl`** | Модель: меш, скелет, морфы, список клипов и (опц.) ссылка на AnimGraph. |
| **`.vanim`** | Один анимационный клип (запись поз скелета по времени). |
| **`.vanmgrph`** | AnimGraph — «программа» из стейтов и переходов, выбирающая клипы по параметрам. |
| **`ModelRenderer`** | Компонент, рисующий статическую модель (без скелета). |
| **`SkinnedModelRenderer`** | Компонент со скелетом: гоняет AnimGraph, поддерживает IK, теги, события. |
| **Параметры графа** | bool/int/float/Vector3/Rotation, которые ты пишешь из C# — `renderer.Set("speed", 200f)`. |
| **IK** | Inverse Kinematics — «дотянись рукой до этой точки». |
| **Морфы (blendshapes)** | Пер-вершинная деформация (мимика, форма лица). |
| **Anim events / tags** | Метки на таймлайне клипа: «сейчас левая нога коснулась пола», «сейчас выброс гильзы». |

## Где это в коде Sandbox

- **Игрок** — `Code/Player/Player.cs` создаёт `SkinnedModelRenderer` для тела и рэгдолла. Параметры графа гоняет `PlayerController` + `CitizenAnimationHelper` (см. 28.04).
- **Hold-поза игрока** — `Code/Player/PlayerInventory.cs:479` пишет `holdtype` в граф Citizen, когда игрок берёт оружие.
- **NPC, который держит пропу** — `Code/Npcs/Layers/AnimationLayer.Hold.cs` (разбор в 28.05).
- **View-модель оружия** — `Code/Game/Weapon/WeaponModel/ViewModel.cs` (разбор в 28.06): `move_x`, `aim_pitch`, `b_attack`, `b_reload`, `move_bob` и т.д.
- **Noclip-поза** — `Code/Player/NoclipMoveMode.cs:21` ставит `b_noclip` и обнуляет `duck`.

## Что важно понять прямо сейчас

1. **`SkinnedModelRenderer` — главное действующее лицо.** Без него анимаций нет. Если у объекта только `ModelRenderer`, скелет и AnimGraph не работают.
2. **Из C# ты не «проигрываешь анимации» вручную** — ты **пишешь параметры** в граф, а граф уже сам выбирает, какой клип сейчас играть.
3. **Параметры графа — это «контракт»** между художником-аниматором (он собирает граф в редакторе) и программистом (ты пишешь в граф из C#). Имена строк (`"holdtype"`, `"move_x"`) фиксируются один раз и потом не меняются.
4. **Для Citizen-персонажа есть готовый помощник** — `CitizenAnimationHelper`. Он скрывает «магические строки» за нормальным C#-API. См. 28.04.

## Результат

После этого этапа ты знаешь:

- ✅ Что такое `.vmdl` / `.vanim` / `.vanmgrph` и как они связаны.
- ✅ Что `SkinnedModelRenderer` — это «движок анимаций» внутри сцены.
- ✅ Что из C# ты двигаешь анимации **через параметры графа**, а не через клипы напрямую.
- ✅ Где в коде Sandbox это уже используется.
