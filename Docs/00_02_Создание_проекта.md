# 00.02 — Создание проекта

## Что мы делаем?

Создаём **пустой игровой проект** в s&box, который потом шаг за шагом превратим в полную копию Sandbox.

## Шаг 1: Открой редактор

1. Запусти **s&box** через Steam
2. Перейди в раздел **Create** (Создать)

## Шаг 2: Создай новый проект

1. Нажми **«New Project»** (Новый проект)
2. Выбери шаблон **«Empty»** (Пустой) или **«Game»**
3. Заполни поля:
   - **Name** (Название): `Sandbox`
   - **Organization**: твоё имя или `facepunch`
   - **Ident** (Идентификатор): `sandbox`
4. Нажми **«Create»**

## Шаг 3: Что создал редактор?

После создания появится папка проекта с такой структурой:

```
Sandbox/
├── Code/              ← Папка для твоего кода (C#)
├── Assets/            ← Папка для ресурсов (модели, звуки, текстуры)
├── ProjectSettings/   ← Настройки проекта
├── sandbox.sbproj     ← Главный файл проекта
└── ...
```

> **Как это работает внутри движка:**  
> Файл `.sbproj` — это **манифест проекта** в формате JSON.  
> Движок читает его при загрузке, чтобы понять:
> - Какой тип проекта (game, addon, library)
> - Сколько игроков поддерживается
> - Какая сцена загружается при старте
> - Какие ресурсы включать в сборку
>
> В исходниках движка за это отвечает система в `engine/Sandbox.AppSystem/` —  
> она парсит `.sbproj` и создаёт экземпляр игры.  
> См. https://github.com/LiGyH/sbox-public — папка `engine/`.

## Шаг 4: Открой проект в редакторе

1. В списке проектов найди свой **Sandbox**
2. Двойным кликом открой его
3. Ты увидишь **редактор сцен** — пока он пустой, это нормально

## Что значит каждая папка?

| Папка | Для чего |
|-------|----------|
| `Code/` | Сюда мы будем класть файлы с кодом `.cs`, `.razor`, `.scss` |
| `Assets/` | Модели `.vmdl`, звуки `.sound`, текстуры, материалы |
| `ProjectSettings/` | Настройки физики, ввода, сети, столкновений, звука |

## Результат

После этого этапа у тебя:
- ✅ Создан пустой проект **Sandbox**
- ✅ Ты видишь структуру папок
- ✅ Проект открывается в редакторе
