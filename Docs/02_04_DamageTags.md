# 02.04 — Теги урона (DamageTags) 🏷️

## Что мы делаем?

Этот файл создаёт **теги урона** — строковые константы, описывающие тип нанесённого урона. Когда игрок получает урон, вместе с числом (сколько HP снять) передаётся набор тегов: «это был хедшот», «это был взрыв», «это было падение».

Зачем? Потому что разные типы урона обрабатываются по-разному:
- **Хедшот** — двойной урон и особая иконка в ленте убийств.
- **Взрыв** — при смерти тело разлетается на куски (gib).
- **Падение** — специальная анимация и звук.
- **Урон самому себе** — обычно уменьшен, но иногда нет.

## Как это работает внутри движка?

### Статический класс с константами

`static class` — класс, у которого нельзя создать экземпляр. Он служит «контейнером» для констант. Это стандартный подход в C# для группировки связанных значений.

```csharp
// Использование
if ( args.Tags.Has( DamageTags.Headshot ) )
{
    damage *= 2.0f; // двойной урон за хедшот
}
```

### Почему строки, а не enum?

В s&box теги урона передаются через `TagSet` — коллекцию строк. Один удар может иметь **несколько** тегов одновременно: например, хедшот из ракетницы — это и `"head"`, и `"explosion"`. С `enum` это было бы сложнее (пришлось бы использовать флаги).

Константы (`const string`) компилируются прямо в код — никаких аллокаций в рантайме. По производительности это равно использованию `enum`.

### Связь с DamageParams

В файле `PlayerEvent.cs` (02.03) мы видели структуру `DamageParams`:

```csharp
public struct DamageParams
{
    public TagSet Tags { get; set; }  // ← сюда попадают теги урона
    // ...
}
```

Система урона создаёт `DamageParams`, добавляет теги и рассылает событие `OnDamage`. Компоненты проверяют теги и реагируют.

## Создай файл

Путь: `Code/Player/DamageTags.cs`

```csharp


public static class DamageTags
{
	public const string Headshot = "head";
	public const string Crush = "crush";
	public const string Explosion = "explosion";
	public const string Shock = "shock";
	public const string Fall = "fall";
	public const string GibAlways = "gib_always";
	public const string FullSelfDamage = "full_self_damage";
}
```

> ⚠️ Обратите внимание: файл начинается с двух пустых строк — это соответствует оригиналу в репозитории. Сохраняйте форматирование строка в строку.

## Разбор кода

### Объявление класса

```csharp
public static class DamageTags
```
- `public` — доступен из любого файла.
- `static` — нельзя создать экземпляр. Все члены — статические.
- `DamageTags` — понятное имя: «теги урона».

### Тег Headshot

```csharp
public const string Headshot = "head";
```
**Попадание в голову.** Обычно означает удвоенный урон. В ленте убийств отображается специальная иконка. Система проверяет, в какой хитбокс попала пуля, и добавляет этот тег, если это голова.

### Тег Crush

```csharp
public const string Crush = "crush";
```
**Раздавливание.** Урон от физических объектов: упавшая стена, машина, дверь. При смерти от раздавливания тело разлетается на части (gib-эффект).

### Тег Explosion

```csharp
public const string Explosion = "explosion";
```
**Взрыв.** Урон от гранат, ракет, взрывных бочек. При смерти от взрыва тело гарантированно разлетается на куски. В ленте убийств показывается иконка взрыва.

### Тег Shock

```csharp
public const string Shock = "shock";
```
**Электрический удар.** Урон от электрических ловушек или оружия. Может воспроизводить особую анимацию (судороги) и звуковой эффект.

### Тег Fall

```csharp
public const string Fall = "fall";
```
**Падение.** Урон от падения с высоты. Рассчитывается на основе `distance` из события `OnLand` (02.03). Если игрок умирает от падения, в ленте убийств отображается «разбился насмерть» без указания убийцы.

### Тег GibAlways

```csharp
public const string GibAlways = "gib_always";
```
**Принудительное расчленение.** При смерти с этим тегом тело **всегда** разлетается на куски, независимо от количества урона. Используется для мощного оружия: если ракета попадает прямо в игрока, нет смысла показывать обычную анимацию смерти.

### Тег FullSelfDamage

```csharp
public const string FullSelfDamage = "full_self_damage";
```
**Полный урон самому себе.** Обычно в шутерах урон самому себе уменьшен (например, своя граната наносит 50% урона). Но если стоит этот тег — урон применяется полностью, без уменьшения. Пример: стоять на собственной мине.

### Сводная таблица

| Константа | Строковое значение | Эффект в игре |
|---|---|---|
| `Headshot` | `"head"` | Удвоенный урон, иконка хедшота |
| `Crush` | `"crush"` | Gib-эффект при смерти |
| `Explosion` | `"explosion"` | Gib-эффект, иконка взрыва |
| `Shock` | `"shock"` | Электрическая анимация |
| `Fall` | `"fall"` | Урон от падения, «разбился» в ленте |
| `GibAlways` | `"gib_always"` | Принудительное расчленение |
| `FullSelfDamage` | `"full_self_damage"` | Нет уменьшения урона себе |

### Пример использования

```csharp
// При создании урона от взрыва
var damage = new PlayerDamageParams
{
    Damage = 100f,
    Tags = new TagSet( DamageTags.Explosion, DamageTags.GibAlways )
};

// При обработке урона
if ( args.Tags.Has( DamageTags.Headshot ) )
    finalDamage *= 2.0f;

if ( args.Tags.Has( DamageTags.FullSelfDamage ) )
    selfDamageMultiplier = 1.0f; // не уменьшать
```

## Результат

После создания этого файла:
- Система урона может различать типы повреждений через строковые теги.
- Теги используются в `DamageParams.Tags` (из 02.03) для передачи информации о типе урона.
- Визуальные эффекты (gib, анимации) и геймплейная логика (множители урона) зависят от этих тегов.
- Константы обеспечивают единообразие: нет риска опечатки в строке `"head"` — вместо этого используется `DamageTags.Headshot`.
