# 00.05 — Настройки столкновений (Collision)

## Что мы делаем?

Настраиваем **правила столкновений** — какие объекты сталкиваются друг с другом, а какие проходят насквозь.

## Зачем это нужно?

Представь: пуля не должна сталкиваться с водой (она пролетает сквозь), но должна попадать в стены и игроков.  
А триггер-зона (невидимая область, которая что-то делает когда ты в неё заходишь) — не должна останавливать тебя.

## Что означают типы столкновений?

| Тип | Объяснение |
|-----|------------|
| **Collide** | Объекты **сталкиваются** и останавливают друг друга |
| **Trigger** | Объекты **проходят сквозь**, но движок **уведомляет** код (можно отслеживать) |
| **Ignore** | Объекты **полностью игнорируют** друг друга |
| **Unset** | Не задано — используется поведение по умолчанию |

## Что означают слои?

| Слой | Что это |
|------|---------|
| `solid` | Твёрдые объекты (стены, пол, пропы) |
| `trigger` | Невидимые зоны (триггеры телепортации, урона) |
| `ladder` | Лестницы |
| `water` | Вода |
| `ragdoll` | Тряпичные куклы (трупы) |
| `player` | Игрок |
| `effect` | Визуальные эффекты (искры, следы) |
| `bullet` | Пули |
| `bolt` | Болты/стрелы (тяжёлые снаряды) |
| `projectile` | Снаряды (ракеты, гранаты) |
| `sound` | Звуковые зоны |
| `noclip` | Режим полёта (всё проходит сквозь) |
| `constraint` | Физические связи (верёвки, пружины) |

> **Как это работает внутри движка:**  
> Класс `CollisionRules` хранит **матрицу столкновений**.  
> Каждому объекту в игре присваивается **слой** (layer).  
> Когда физический движок проверяет два объекта, он смотрит в эту матрицу:  
> «Что делать когда `solid` встречает `bullet`?» → `Collide`.
>
> Секция `Pairs` — это конкретные правила для пар слоёв.  
> Если пара не указана, используется значение из `Defaults`.
>
> Обрати внимание: `noclip` игнорирует **всё** — это режим свободного полёта,  
> когда игрок-администратор проходит сквозь стены.

- ✅ Пули попадают в стены и рэгдоллы
- ✅ Noclip проходит сквозь всё
