# 26.06 — Звук: SoundEvent и Sound.Play

## Что мы делаем?

Разбираем встроенную звуковую систему s&box: как **проиграть звук**, как сделать его **3D** (затухает с расстоянием), как настроить **громкость**, как создать **SoundEvent**-ассет с вариантами и микшером.

## Два уровня API

| Уровень | Что это |
|---|---|
| **Низкий** — `Sound.Play( "путь/к/звуку.sound", позиция )` | прямой вызов из кода |
| **Высокий** — `SoundEvent`-ассет | настраиваемый ресурс с вариациями, питчем, рандомом, микшером |

Хорошее правило: **в коде не пиши пути к `.wav`/`.mp3`**. Создавай `SoundEvent` (`.sound`-файл) и ссылайся на него.

## Базовый вызов

```csharp
// Проиграть в позиции игрока (3D)
Sound.Play( "weapons/pistol_shoot.sound", WorldPosition );

// Проиграть «в голове» слушателя (2D, без затухания)
Sound.Play( "ui/click.sound" );
```

Перегрузка с `Vector3` делает звук **позиционным** — он будет тише сбоку, тише вдалеке, перейдёт в стерео-канал в зависимости от ориентации камеры.

## Что такое `SoundEvent`?

`SoundEvent` — это `GameResource`-ассет (см. [00.29](00_29_GameResource.md)). В нём настраивается:

| Поле | Что задаёт |
|---|---|
| `Sounds` | список `.wav`/`.mp3`-файлов (один из них выбирается случайно) |
| `Volume` | громкость (мин/макс — выбирается рандомно в диапазоне) |
| `Pitch` | питч (мин/макс — для разнообразия выстрелов) |
| `DistanceMin` / `DistanceMax` | с какой дистанции начинает затухать и где затухает в ноль |
| `Mixer` | в какой микшер уходит ([00.07](00_07_Настройки_звука.md)) |
| `Spatial` | 2D или 3D |

Это вытаскивает «случайность и баланс» из кода — звукорежиссёр (или ты сам в редакторе) правит ассет, перезагрузка не нужна.

## Привязка к движущемуся объекту

Если выстрел идёт из ствола движущегося оружия, звук должен **двигаться вместе**. Используй `WorldPosition` на каждый кадр или `SoundHandle.Position`:

```csharp
var handle = Sound.Play( "weapons/minigun_loop.sound", WorldPosition );
// в OnUpdate:
handle.Position = WorldPosition;
```

`SoundHandle` — структура, которой можно остановить звук:

```csharp
handle.Stop();         // мгновенно
handle.Stop( 0.5f );   // плавно за полсекунды
```

## Цикличные звуки

Зацикливание (loop) задаётся **внутри `SoundEvent`** (флаг `Looping`), не в коде. Хочешь крутить мотор машины — создай `engine_loop.sound` с галкой Looping и просто запусти его:

```csharp
_engineHandle = Sound.Play( "vehicles/engine_loop.sound", WorldPosition );
// при выключении мотора:
_engineHandle.Stop( 0.3f );
```

## Микшеры (Mixer)

Микшер — это «канал»: SFX, Music, Voice, UI. Игрок может их регулировать в настройках. В каждом `SoundEvent` есть поле `Mixer` — выбери нужный. Подробнее в [00.07 — Настройки звука](00_07_Настройки_звука.md).

## Доступ к слушателю

«Слушатель» — это виртуальные «уши» в мире, относительно них считается панорама и громкость. По умолчанию это камера, но можно перенести (например, в голову игрока в режиме от первого лица):

```csharp
SoundListener.Position = Player.Head.WorldPosition;
SoundListener.Rotation = Player.Head.WorldRotation;
```

## Голосовой чат и липсинк

В s&box есть встроенный **voice chat** (см. [03.16 — SandboxVoice](03_16_SandboxVoice.md)) и **lip-sync** для моделей. Они построены поверх той же системы — но это отдельная большая тема, в этом этапе мы её не раскрываем.

## Звук удара и физический материал

Когда два тела сталкиваются, движок **сам** проигрывает звук, основанный на их `Surface` (см. [26.01](26_01_Physics_Bodies.md), [01.12](01_12_SelfCollisionSound.md)). Тебе не нужно вручную играть `Sound.Play` на каждое касание — поля `ImpactSound` достаточно.

## Что важно запомнить

- **Не хардкодь пути к `.wav`** — оборачивай в `SoundEvent` (`.sound`).
- `Sound.Play( name, position )` — позиционный 3D, без `position` — 2D.
- Возвращённый `SoundHandle` позволяет остановить или передвинуть звук.
- Цикличность, громкость, питч, рандомизация — настраиваются в `SoundEvent`-ассете.
- Каждый `SoundEvent` идёт через **микшер** — игрок крутит громкость отдельных категорий.
- Для физических звуков пользуйся `Surface.ImpactSound`, не вызывай `Sound.Play` руками.
