# 00.20 — Сеть: хост, клиент, соединение

## Что мы делаем?

Начинаем блок про **сетевую модель s&box**. В этом файле — общая картина: кто такие «хост», «клиент», «прокси», «соединение». Без этих понятий дальше будет каша.

## Главный принцип

> **В s&box сеть — peer-to-peer с авторитетным хостом.**
> Один игрок поднимает лобби — он **хост** (сервер). Остальные присоединяются — они **клиенты**. Хост владеет «общим состоянием мира», но **не** каждый объект.

Это не выделенный сервер (хотя их тоже можно делать — см. Facepunch docs про Dedicated Servers). Это похоже на мультиплеер в Minecraft: кто-то первый зашёл, он и хост.

## Понятия

### `Connection`

Каждый игрок (включая самого хоста) представлен объектом `Connection`. У него есть:

- `DisplayName` — ник.
- `SteamId` — Steam ID.
- `IsHost` — правда ли это хост.
- `ConnectionId` — уникальный номер сессии.

Список соединений:

```csharp
foreach ( var conn in Connection.All )
    Log.Info( conn.DisplayName );

// Текущее «моё» соединение
Connection me = Connection.Local;

// Хост сессии
Connection host = Connection.Host;
```

### «Хост» vs «Клиент» — в коде

Когда ты пишешь код, один и тот же `.cs` файл исполняется **и** на хосте, **и** на клиентах. Проверки, чтобы разграничить поведение:

```csharp
if ( Networking.IsHost )
{
    // только на хосте (сервер-авторитетные вещи: урон, смерти)
}

if ( Networking.IsClient )
{
    // только на клиентах (эффекты, UI)
}
```

Важно: **хост — это тоже клиент**. Он держит своё окно и камеру. Проверка `IsHost` ≠ «графики нет» — она просто означает «у меня в руках сетевая истина».

### «Proxy» — копия чужого объекта у меня

Когда игрок А зашёл в лобби игрока Б, у Б в сцене появляется `GameObject` игрока А. Этот `GameObject` — **прокси**. Его движение контролирует **не** Б, а клиент А: он шлёт свою позицию, Б просто отображает.

Главная проверка в коде:

```csharp
protected override void OnUpdate()
{
    if ( IsProxy )
        return;   // это не мой объект, я не им управляю

    // дальше — код, который выполняет только «владелец»
    ReadInput();
    Move();
}
```

Это свойство доступно из любого компонента — оно проверяет, является ли `GameObject` прокси для **тебя**.

## Создание лобби

```csharp
Networking.CreateLobby( new LobbyConfig
{
    MaxPlayers = 8,
    Privacy    = LobbyPrivacy.Public,
    Name       = "Моё лобби"
} );
```

После этого ты становишься хостом сессии.

## Поиск и вход в лобби

```csharp
// Список публичных лобби
var lobbies = await Networking.QueryLobbies();

// Подключиться к одному из них
Networking.Connect( lobbies[0].LobbyId );
```

Лобби в s&box работают через Steam, так что не нужно возиться с IP-адресами.

## Как сделать сетевую сцену

Есть три части:

1. **`GameObject` должен быть сетевым** — либо заранее настроен в редакторе (галочка **Network Object**), либо создан кодом и опубликован через `NetworkSpawn()`.
2. **В сцене должен быть компонент, который создаёт сессию** — это `NetworkHelper` (встроенный). Он сам поднимает сервер при старте сцены, если включён `StartServer`.
3. **Игрок должен быть префабом**, и в `NetworkHelper` указан `PlayerPrefab` — иначе непонятно, кого спавнить новым соединениям.

Подробнее: [00.21 — Сетевые объекты и NetworkHelper](00_21_Networked_Objects.md).

## Отслеживание соединений

Чтобы узнать, когда кто-то подключился/отключился, сделай компонент с интерфейсом `Component.INetworkListener`:

```csharp
public sealed class MySessionListener : Component, Component.INetworkListener
{
    void INetworkListener.OnActive( Connection conn )
    {
        Log.Info( $"Подключился: {conn.DisplayName}" );
    }

    void INetworkListener.OnDisconnected( Connection conn )
    {
        Log.Info( $"Отключился: {conn.DisplayName}" );
    }
}
```

Вызывается **только на хосте**, потому что только он знает обо всех соединениях.

## Потоки данных — что идёт по сети?

Коротко, по сети гоняются 4 вида штук:

| Что | Чем задаётся | Подробнее |
|---|---|---|
| Спавн/удаление сетевых объектов | `NetworkSpawn()`, `Destroy()` | [00.21](00_21_Networked_Objects.md) |
| Передача владения | `Network.TakeOwnership()`, `.DropOwnership()` | [00.22](00_22_Ownership.md) |
| Удалённые вызовы методов | `[Rpc.Broadcast]`, `[Rpc.Owner]`, `[Rpc.Host]` | [00.23](00_23_Rpc_Messages.md) |
| Синхронизация свойств | `[Sync]` | [00.24](00_24_Sync_Properties.md) |

Это и есть **вся сеть** в s&box. Простая и удобная модель.

## Что НЕ по сети

- ❌ Визуальные эффекты по умолчанию **не** синхронизируются — каждый клиент проигрывает их у себя по команде сервера.
- ❌ Звуки по умолчанию играются локально — их нужно запускать на всех через `[Rpc.Broadcast]`.
- ❌ `Input` читается только на локальном клиенте.
