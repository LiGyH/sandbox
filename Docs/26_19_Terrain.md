# 26.19 — Terrain: рельеф

## Что мы делаем?

`Terrain` — встроенный в s&box компонент **высотной карты (heightmap)** с поддержкой нескольких текстурных слоёв. Это правильный инструмент для **открытых пространств**: лес, поле, горы. Делать ландшафт из ящиков и треугольников в Hammer — мучительно и неэффективно; Terrain устраняет проблему.

## Из чего состоит Terrain

| Понятие | Что это |
|---|---|
| **Heightmap** | 2D-сетка значений «высоты» (например, 1024×1024 точек) |
| **Resolution** | плотность сетки |
| **Tile/Chunk** | блок рельефа, рендерящийся отдельно для оптимизации |
| **Layer** | текстура (трава, камень, песок, дорога), наносимая кистью |
| **Splatmap** | маска, где какой слой виден |
| **Material** | физический материал каждого слоя (трение, звук шагов) |

## Создание terrain

В редакторе сцены:

1. Добавь GameObject и компонент `Terrain` (точное имя может меняться от версии — `TerrainComponent`/`HeightField`).
2. Задай размеры (ширина × длина в units, обычно 4096–16384).
3. Задай разрешение heightmap (256×256 для лоу-полигонального, 1024×1024 для детализированного).
4. Открой инструменты редактирования:
   - **Sculpt** — поднимать/опускать кистью.
   - **Smooth** — сглаживать резкие переходы.
   - **Flatten** — сравнять до заданной высоты.
   - **Paint** — рисовать слой текстуры.

## Слои и текстуры

Каждый слой — это:

- цветовая текстура (`albedo`);
- нормали (`normal`);
- параметры (`roughness`, `metallic`, `ao`);
- tiling (как часто повторяется на квадратный метр).

Слои **смешиваются по splatmap**'у. Под кистью «Paint» ты подмешиваешь нужный слой в нужном месте. Можно делать плавные переходы (трава → песок у воды).

## Физика и навмеш

Terrain участвует в физике автоматически: пуля попадает, игрок ходит. Звук шагов берётся из `Surface` слоя, на котором стоит игрок.

Для NavMesh ([26.17](26_17_NavMesh.md)) terrain тоже учитывается, если генерация навмеша включает «динамическую геометрию». Слишком крутые склоны (определяется параметром `Max slope` навмеша) автоматически становятся непроходимыми.

## Растительность (Foliage)

Часто к terrain'у пристёгивается система **разбрасывания растительности**: трава, кусты, мелкие камни инстансируются по правилам:

- плотность (на квадратный метр);
- зависит от слоя (трава растёт только на «grass»-слое);
- зависит от уклона (камни не лежат на крутом склоне);
- LOD-уровни и отсечение по дистанции.

В Sandbox-стиле игре трава нужна редко; но если делаешь outdoor-режимы — это будет одна из самых заметных деталей.

## Производительность

Terrain — потенциально один из самых **дорогих** объектов. Чек-лист:

- **Не задирай разрешение.** 1024×1024 для 8000×8000 unit-карты обычно достаточно.
- **Используй чанки** (если есть настройка) — невидимые куски не рендерятся.
- **LOD-биас** для дальних чанков — снижает геометрию вдалеке.
- **Слоёв ≤ 6.** Каждый дополнительный слой — лишний sample в шейдере.
- **Текстуры в правильном tiling.** 1×1м для травы — ок, 0.1×0.1м — потеря производительности.

## Сетевая сторона

Heightmap и splatmap **не нужно сетить** в обычной игре — это статическая часть карты. Если делаешь **разрушаемый ландшафт** (кратеры от взрывов), это требует своей системы: храни «дельту» heightmap'а, синхронизируй её через Sync/RPC, применяй на всех клиентах.

## Когда **НЕ** terrain

- Внутренние помещения, коридоры, многоэтажки → **Hammer/Mesh**.
- Маленькие острова под несколько пропов → проще `BlockMesh` или модель.
- Сильно вертикальные структуры (отвесные скалы, пещеры) → terrain не умеет «нависающие» куски — это однозначная heightmap.

## Что важно запомнить

- `Terrain` хранит высоту каждой точки 2D-сетки.
- Несколько **слоёв текстур** смешиваются через splatmap, рисуемый кистью.
- Каждый слой имеет свой `Surface` — отсюда трение и звук шагов.
- Terrain работает с физикой и навмешем «бесплатно».
- Растительность инстансируется по правилам слоя/уклона.
- Не задирай разрешение и количество слоёв — производительность падает быстро.
- Динамическая модификация (кратеры) требует своей сетевой схемы.
