# 03.03 — Патроны игрока (Player.Ammo) 🔫

## Что мы делаем?

Создаём **файл-заглушку** для системы патронов. В текущей версии Sandbox патроны управляются на уровне оружия (`BaseWeapon.Ammo`), а не на уровне игрока. Файл `Player.Ammo.cs` существует как зарезервированное место — если в будущем понадобится глобальный пул патронов, код будет здесь.

## Почему файл пустой?

В ранних версиях Sandbox патроны хранились у игрока (как в Half-Life: «у игрока 50 пуль, 3 ракеты»). Сейчас каждое оружие хранит свои патроны самостоятельно (`ClipContents`, `ReserveAmmo`). Файл оставлен для обратной совместимости.

## Создай файл

Путь: `Code/Player/Player.Ammo.cs`

```csharp
public sealed partial class Player
{
}
```

## Объяснение

- `public sealed partial class Player` — продолжение того же класса `Player`
- `sealed` — нельзя наследовать (для производительности и безопасности)
- `partial` — можно разделить по файлам
- Тело пустое — никакой логики

## Зачем хранить пустой файл?

1. **Резервация** — если добавить сюда `AmmoType.Pistol`, `AmmoType.Shotgun` и т.д., не придётся создавать новый файл
2. **Консистентность** — все partial-файлы Player следуют одному паттерну именования
3. **Git-история** — файл уже в репозитории, значит будущие изменения будут видны в diff
