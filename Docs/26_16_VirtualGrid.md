# 26.16 — VirtualGrid: большие списки в Razor

## Что мы делаем?

Когда меню спавна или каталог моделей содержит **тысячи** элементов, обычный `<div>`-список положит всё на лопатки: панели создаются физически, верстаются, рендерятся. **`VirtualGrid`** — Razor-компонент, который рендерит **только видимые** элементы и пересоздаёт их при скролле. Незаменимая вещь для каталогов и инвентарей.

## Базовое использование

```razor
<root>
    <VirtualGrid Items=@Items ItemSize=@(new Vector2(120, 140))>
        <Item Context="item">
            @if ( item is Package pkg )
            {
                <SpawnButton Icon=@pkg.Thumb Action=@(() => Spawn(pkg)) />
            }
        </Item>
    </VirtualGrid>
</root>

@code {
    public IEnumerable<Package> Items { get; set; }
}
```

Что произошло:

- `Items` — любое `IEnumerable<T>` (массив, список, LINQ-выражение).
- `ItemSize` — `Vector2` ширина×высота желаемой ячейки. Реальная ширина подгоняется под контейнер с сохранением соотношения.
- `<Item Context="item">` — шаблон одной ячейки. `item` — это конкретный элемент данных.

## Как это работает

`VirtualGrid` смотрит на свой размер, делит его на размер ячейки, понимает «вижу 6×4 = 24 ячейки». Рендерит только их. При скролле:

1. Удаляет панели, ушедшие за верх.
2. Создаёт панели для тех, что приехали снизу.
3. Поведение для пользователя — обычный плавный список.

Памяти уходит на 24 панели, а не на 100 000.

## Стиль и размер

VirtualGrid обязан иметь **известный размер** в стилях, иначе он не будет работать:

```scss
VirtualGrid {
    width: 100%;
    height: 100%;
    gap: 8px;
}
```

`gap` — расстояние между ячейками. Все остальные стили (фон, рамка) работают как обычно.

## Несколько типов в одной сетке

Можно мешать типы — внутри `<Item>` проверяй через `is`:

```razor
<Item Context="item">
    @if ( item is Package pkg )      { <PackageCard Pkg=@pkg /> }
    else if ( item is Spawnlist sl ) { <SpawnlistCard List=@sl /> }
    else                             { <UnknownCard /> }
</Item>
```

Чтобы сетка пересоздавалась, когда изменился `Items`, **меняй ссылку** (новый массив/список), а не модифицируй существующий — Razor сравнивает по ссылке, чтобы понять «надо ли пересчитать».

## Подсказки

| Проблема | Решение |
|---|---|
| Сетка пустая | Проверь, что VirtualGrid имеет `width/height` в стилях |
| Ячейки разной высоты | Так нельзя. Все ячейки должны быть одного размера. |
| Не обновляется при изменении `Items` | Присвой новый массив (`Items = filtered.ToArray()`), не модифицируй на месте |
| Ячейки слишком маленькие | Увеличь `ItemSize` (он подгоняется, но соотношение сохраняется) |
| Скролл прыгает | Не меняй `Items` каждый кадр — кешируй |

## Сравнение с обычным списком

| | `<div> @foreach` | `<VirtualGrid>` |
|---|---|---|
| Удобство | максимум | чуть сложнее |
| 10 элементов | оба ок | не нужно |
| 100 элементов | оба ок | можно |
| 1000 элементов | начинает тормозить | работает гладко |
| 100000 элементов | падает | работает гладко |

Правило: **<100 элементов — обычный список; ≥1000 — VirtualGrid**. В диапазоне 100–1000 — по вкусу.

## Соединение с фильтром поиска

Главный сценарий — каталог с поиском:

```razor
<input value=@SearchText oninput=@OnSearch />

<VirtualGrid Items=@Filtered ItemSize=@(new Vector2(120, 140))>
    <Item Context="item">
        <PackageCard Pkg=@((Package)item) />
    </Item>
</VirtualGrid>

@code {
    string SearchText = "";
    public Package[] All { get; set; }
    public Package[] Filtered =>
        string.IsNullOrEmpty( SearchText )
            ? All
            : All.Where( p => p.Title.Contains( SearchText, StringComparison.OrdinalIgnoreCase ) ).ToArray();

    void OnSearch( string text ) { SearchText = text; StateHasChanged(); }
}
```

Каждое нажатие клавиши пересчитывает `Filtered` и пересобирает видимые ячейки. Для 100k элементов **`Where + ToArray`** на каждом нажатии тяжеловат — добавь дебаунс или индексируй заранее.

## Что важно запомнить

- `VirtualGrid` рендерит **только видимое** — для больших списков обязателен.
- Задай ему размер в стилях (`width: 100%; height: 100%`).
- `ItemSize` — желаемое соотношение ячейки, реальный размер подгонится под контейнер.
- Все ячейки должны быть **одного размера**.
- Чтобы пересчитать, присвой `Items` **новый массив** (новая ссылка).
- Соединяй с поиском, но не пересчитывай фильтр чаще, чем нужно.
