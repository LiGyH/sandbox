# Этап 0: Подготовка — Знакомство с s&box

## Что такое s&box?

**s&box** (произносится «сэндбокс») — это игровой движок от компании Facepunch (создатели Garry's Mod и Rust). Он построен на базе движка Source 2 от Valve.

Если ты знаком с Garry's Mod — s&box это его духовный наследник, только современный, с поддержкой C# вместо Lua.

## Что нам понадобится?

1. **Steam** — s&box распространяется через Steam
2. **s&box** — сам движок (купи или получи доступ через Steam)
3. **Базовое понимание файловой структуры** — уметь открывать папки, создавать файлы

## Основные понятия (словарь новичка)

Прежде чем мы начнём писать код, давай разберёмся с терминами. Каждый из них тебе встретится в руководстве.

### 🎮 Движок (Engine)

**Движок** — это программа, которая делает всю «тяжёлую работу»: рисует графику, считает физику, играет звуки. Ты пишешь код, а движок его выполняет.

> **Аналогия**: Движок — это как автомобиль. Ты крутишь руль и жмёшь педали (пишешь код), а мотор делает всё остальное.

Исходный код движка s&box доступен здесь: https://github.com/LiGyH/sbox-public

### 🧱 GameObject (Игровой объект)

Всё, что ты видишь в игре — это **GameObject**: игрок, стена, оружие, свет, камера. Каждый GameObject — это «контейнер», к которому прикрепляются компоненты.

**В движке** (`sbox-public`): GameObject — это основной класс, который управляет деревом объектов в сцене. Он содержит позицию (`Position`), поворот (`Rotation`), масштаб (`Scale`), и список дочерних объектов (`Children`).

```
GameObject "Игрок"
├── Позиция: (100, 0, 50)
├── Поворот: (0, 90, 0)
├── Компоненты:
│   ├── PlayerController  ← управляет движением
│   ├── Player            ← наш код (здоровье, смерть)
│   └── PlayerInventory   ← инвентарь
└── Дочерние объекты:
    ├── "Модель тела"     ← 3D модель персонажа
    └── "Камера"          ← откуда мы смотрим
```

### 🔧 Component (Компонент)

**Компонент** — это кусок логики, прикреплённый к GameObject. Вместо того чтобы делать один огромный класс «Игрок», мы делаем несколько маленьких компонентов:

- `Player` — здоровье, смерть
- `PlayerInventory` — инвентарь
- `PlayerFlashlight` — фонарик
- `PlayerFallDamage` — урон от падения

Каждый компонент отвечает за одну конкретную вещь. Это называется **компонентная архитектура**.

**В движке**: Компоненты наследуются от класса `Component`. Движок вызывает у них специальные методы:
- `OnEnabled()` — когда компонент включается
- `OnDisabled()` — когда выключается
- `OnUpdate()` — каждый кадр (60 раз в секунду)
- `OnFixedUpdate()` — с фиксированной частотой (для физики)
- `OnDestroy()` — когда объект удаляется

```csharp
// Пример простейшего компонента
public class МойКомпонент : Component
{
    protected override void OnUpdate()
    {
        // Этот код выполняется каждый кадр
    }
}
```

### 🎬 Scene (Сцена)

**Сцена** — это «уровень» или «мир». Она содержит все GameObject'ы. Когда ты загружаешь карту — ты загружаешь сцену.

В нашем проекте главная сцена: `Assets/scenes/sandbox.scene`

В ней по умолчанию есть только один объект — `MapLoader`, который загружает карту:

```json
{
  "Name": "MapLoader",
  "Components": [
    {
      "__type": "Sandbox.MapInstance",
      "MapName": "facepunch.flatgrass",
      "UseMapFromLaunch": true
    }
  ]
}
```

### 📦 Prefab (Шаблон)

**Prefab** (префаб) — это сохранённый шаблон GameObject со всеми его компонентами и настройками. Вместо того чтобы каждый раз настраивать игрока руками, мы создаём шаблон один раз и потом «штампуем» копии.

В нашем проекте шаблон игрока: `Assets/prefabs/engine/player.prefab`

### 🌐 Networking (Сеть)

s&box — **сетевой** движок. Это значит, что каждое действие в игре должно работать и в мультиплеере. Основные понятия:

- **Host (Хост)** — главный компьютер (сервер)
- **Client (Клиент)** — подключённый игрок
- **[Sync]** — данные, которые автоматически синхронизируются между хостом и клиентами
- **[Rpc.Host]** — метод, который выполняется на хосте (сервере)
- **[Rpc.Broadcast]** — метод, который выполняется у всех игроков одновременно

```csharp
// Здоровье синхронизируется автоматически: хост меняет — все видят
[Sync] public float Health { get; set; } = 100f;

// Этот метод может вызвать клиент, но выполнится он на хосте
[Rpc.Host]
public void TakeDamage( float amount )
{
    Health -= amount;
}
```

**В движке** (`sbox-public`): Сетевая система использует `NetworkMode` для определения, как объект синхронизируется. `NetworkMode.Object` означает, что объект является корнем сетевой синхронизации.

### 📝 Property (Свойство)

Когда ты пишешь `[Property]` перед переменной в компоненте — она появляется в редакторе s&box. Ты можешь менять её значение мышкой, без написания кода.

```csharp
[Property] public float MaxHealth { get; set; } = 100f;
//          ↑ это видно в редакторе, можно менять руками
```

### 🏷️ Tags (Теги)

**Теги** — это метки на объектах. Они помогают определять, что перед нами: `"player"`, `"solid"`, `"trigger"`, `"ragdoll"`. Физическая система использует теги для столкновений.

## Как работает s&box изнутри?

### Игровой цикл (Game Loop)

Каждый кадр движок делает следующее:

```
1. Обработка ввода (клавиатура, мышь)
   ↓
2. FixedUpdate (физика, 50 раз/сек)
   ↓
3. Update (логика, каждый кадр)
   ↓
4. Синхронизация по сети
   ↓
5. Рендер (отрисовка кадра)
   ↓
6. Повтор
```

### GameObjectSystem

Это **глобальная система**, которая существует в единственном экземпляре на всю сцену. Наш `GameManager` — это именно такая система. Она управляет всей игрой целиком.

```csharp
// Так создаётся глобальная система
public class GameManager : GameObjectSystem<GameManager>
{
    public GameManager( Scene scene ) : base( scene ) { }
}
```

**В движке**: `GameObjectSystem<T>` — шаблон (generics), который гарантирует, что в сцене будет только один экземпляр `T`. Доступ к нему: `GameManager.Current`.

## Структура файлов проекта s&box

```
МойПроект/
├── МойПроект.sbproj    ← Файл проекта (JSON). s&box читает его первым
├── ProjectSettings/    ← Настройки проекта
│   ├── Input.config    ← Привязки клавиш
│   ├── Physics.config  ← Настройки физики
│   ├── Collision.config← Правила столкновений
│   ├── Networking.config← Настройки сети
│   └── Mixer.config    ← Настройки звука
├── Assets/             ← Все ресурсы (модели, текстуры, звуки, сцены)
│   ├── scenes/         ← Файлы сцен (.scene)
│   └── prefabs/        ← Шаблоны объектов (.prefab)
├── Code/               ← Весь код на C# 
└── Editor/             ← Код расширений редактора
```

### Файл проекта (.sbproj)

Это JSON-файл, который говорит движку s&box:
- Как называется проект (`Title`)
- Какого он типа (`Type: "game"`)
- Какая сцена запускается при старте (`StartupScene`)
- Сколько максимум игроков (`MaxPlayers`)
- Какая карта по умолчанию (`MapList`)

## Что такое C#?

**C#** (произносится «си-шарп») — это язык программирования. Он используется в s&box для написания всей логики игры.

Основы, которые тебе нужно знать:

```csharp
// Это комментарий — движок его игнорирует

// Переменная — хранит одно значение
float здоровье = 100f;     // число с дробной частью
int патроны = 30;           // целое число
bool живой = true;          // да или нет
string имя = "Игрок";      // текст

// Функция (метод) — выполняет действие
void Прыгнуть()
{
    // код прыжка
}

// Класс — объединяет переменные и функции
public class Player : Component
{
    public float Health { get; set; } = 100f;
    
    protected override void OnUpdate()
    {
        // выполняется каждый кадр
    }
}
```

### Ключевые слова, которые встретятся

| Слово | Что значит |
|-------|-----------|
| `public` | Доступно отовсюду |
| `private` | Доступно только внутри класса |
| `protected` | Доступно внутри класса и его наследников |
| `override` | Перезаписываем метод из родительского класса |
| `virtual` | Метод можно перезаписать в наследнике |
| `static` | Принадлежит классу, а не экземпляру |
| `partial` | Класс разделён на несколько файлов |
| `sealed` | Класс нельзя наследовать |
| `interface` | «Контракт» — набор методов, которые класс обязан реализовать |
