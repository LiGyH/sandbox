# 26.24 — Сервисы: Achievements, Leaderboards, Stats

## Что мы делаем?

Завершающий этап Фазы 26 — обзор **встроенных бекенд-сервисов** s&box: достижений, статистики, лидербордов и токенов авторизации. Они закрывают практически всю «пользовательскую прогрессию» без своего сервера. В текущем руководстве у нас уже есть [04.02 — GameManager.Achievements](04_02_GameManager_Achievements.md), но более широкая картина не освещена — её и закрываем.

## Achievements (достижения)

Достижения настраиваются на странице игры в asset.party, в коде только **разблокируются**:

```csharp
Sandbox.Services.Achievements.Unlock( "first_blood" );
```

Виды достижений:

| Тип | Как разблокируется |
|---|---|
| **Manual** | вручную, методом `Unlock( id )` |
| **Stat** | автоматически, когда соответствующая статистика дошла до порога |

### Stat-based пример

Достижение «100 props спавнено»:

1. На asset.party создаёшь достижение, тип `Stat`, target stat `props_spawned`, max value `100`.
2. В коде на каждый спавн:

   ```csharp
   Sandbox.Services.Stats.Increment( "props_spawned", 1 );
   ```

Движок сам разблокирует достижение, когда статистика дойдёт до 100. Ты не вызываешь `Unlock`.

### Список достижений

```csharp
foreach ( var a in Sandbox.Services.Achievements.All )
    Log.Info( $"{a.Name} — {a.IsUnlocked}" );
```

### Score

У каждого достижения свой **score** (5–100), сумма по всем достижениям игры — не больше 1000. Это аналог геймерскорта — общий рейтинг игрока на платформе.

### Когда вызывать

- На **хосте**. Клиент не знает достоверно об «убийствах», «спавнах» и т.д.
- **Один раз** на событие. Не клади `Unlock` в `OnUpdate`.
- Достижения должны быть значимы для игрока — не плоди «зашёл в меню».

## Stats (статистика)

`Stats` — числа, привязанные к игроку и к игре:

```csharp
Sandbox.Services.Stats.Increment( "props_spawned", 1 );
Sandbox.Services.Stats.Set( "max_score_in_round", 9999 );
Sandbox.Services.Stats.SetGlobal( "leaderboard_score", 12345 );
```

| Метод | Что делает |
|---|---|
| `Increment( name, value )` | прибавить к статистике |
| `Set( name, value )` | установить значение (больше — записывается, меньше — нет, обычно) |
| `SetGlobal( name, value )` | для глобальных лидербордов |

Статы используются:

- как условия для достижений;
- как источник данных лидербордов;
- для аналитики (сколько раз игрок открывал тулган).

### Чтение

```csharp
double v = Sandbox.Services.Stats.LocalPlayer.Get( "props_spawned" ).Value;
```

(Имя API в твоей версии может отличаться — сверься с актуальной документацией.)

## Leaderboards (лидерборды)

Лидерборды строятся **поверх стат**: на странице игры создаёшь лидерборд «Best round score» с привязкой к статистике `max_score_in_round` и направлением сортировки. Дальше статистика кладётся как обычно — лидерборд сам обновляется.

### Чтение топа

```csharp
var lb = await Sandbox.Services.Leaderboards.GetAsync( "best_round_score" );
foreach ( var entry in lb.Entries )
    Log.Info( $"{entry.Rank}. {entry.DisplayName}: {entry.Value}" );
```

### Web-API

Лидерборды доступны и **извне игры** через REST:

```
https://services.facepunch.com/sbox/leaderboard/list?package=...&board=best_round_score
```

Удобно для собственного сайта, Discord-бота, веб-панели.

## Auth Tokens (токены авторизации)

Когда хочешь вызывать **свой** бекенд (HTTP/[26.12](26_12_Http_Requests.md), WebSocket/[26.13](26_13_WebSockets.md)) и быть уверенным, что запрос идёт от настоящего игрока в твоей игре, проси у движка **подписанный токен**:

```csharp
string token = await Sandbox.Services.Auth.GetToken( "MyServerName" );
```

Сервер потом:

1. Получает токен в заголовке.
2. Делает HTTP-запрос к Facepunch (`/sbox/auth/verify?token=...`), получает в ответ Steam ID игрока.
3. Доверяет: «да, это действительно игрок X».

Преимущества:

- Не нужны свои логины/пароли.
- Привязка к Steam-аккаунту.
- Защита от подделки запросов.

## Сторона, на которой это вызывать

| Действие | Где |
|---|---|
| `Achievements.Unlock` | хост (или dedicated-сервер) — он знает реальное событие |
| `Stats.Increment` | хост; если статистика только-локальная (для своего HUD) — можно клиент |
| `Stats.Set` (global) | только хост |
| `Leaderboards.GetAsync` | хост; результат раздать клиентам через `[Sync]` |
| `Auth.GetToken` | каждый клиент сам себе |

## Тестирование

В режиме разработки достижения и статистика не отправляются на «продакшен» — сервис работает в **песочнице** твоего dev-аккаунта. Когда публикуешь игру — переключается на реальные данные. Не пиши тесты руками `Unlock("test_achievement")` миллион раз — есть лимиты.

## Что важно запомнить

- **Achievements** разблокируются `Unlock( id )` или автоматически через **Stat**-привязку.
- **Stats** — числовые накопители; основа достижений и лидербордов.
- **Leaderboards** строятся поверх стат, читаются как из игры, так и через web-API.
- **Auth Tokens** дают подписанный «сертификат», что запрос идёт от настоящего игрока.
- Все вызовы делает **хост**, не каждый клиент.
- В деве работает **песочница**, не настоящие данные.
