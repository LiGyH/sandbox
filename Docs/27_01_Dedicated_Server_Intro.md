# 27.01 — Dedicated Server: что это и зачем нужен

## Что мы делаем?

Открываем **Фазу 27** — большой блок про **Dedicated Server (выделенный сервер)** в s&box. До этого момента всё, что мы видели, работало в режиме **Listen Server** (хост — это один из игроков, см. [00.20](00_20_Networking_Basics.md)). Сейчас разбираемся, что меняется, когда сервер — отдельный процесс на отдельной машине, и почему без этого нельзя сделать ни боевой шутер на 64 человека, ни RP-сервер на 1000+, ни тем более MMO.

> 📚 Официальные документы: [Facepunch/sbox-docs → networking/dedicated-servers](https://github.com/Facepunch/sbox-docs/tree/master/docs/networking/dedicated-servers). На них опирается вся Фаза 27.

## Listen Server vs Dedicated Server

В s&box есть **две** топологии:

| Признак | Listen Server (хост-игрок) | Dedicated Server |
|---|---|---|
| Сервер — это | один из игроков (`Networking.IsHost`) | отдельный процесс `sbox-server.exe` |
| Кто видит сцену? | хост-игрок видит и играет | сервер только считает, никого не рисует |
| Кто авторитет? | хост | сервер |
| Когда хост вышел | сессия закрывается | сервер живёт сам по себе |
| Анти-чит | минимальный (хост сам чит) | сервер хранит правду |
| Производительность | ограничена ПК хоста | ограничена железом сервера |
| Подходит для | Sandbox, deathmatch на 8–16 | конкурентные игры, RP, MMO |

**Главное отличие:** на dedicated-сервере **нет графики, нет UI, нет камеры, нет локального игрока**. Это «голая» сцена, которая крутит только серверную сторону логики.

## Почему хост-игрок не подходит для серьёзных проектов

1. **Хост-уход → сессии конец.** В [00.08](00_08_Настройки_сети.md) у нас стоит `DestroyLobbyWhenHostLeaves: true`. На большом сервере это смерть.
2. **Чит-уязвимость.** Хост запускает свой клиент → у него на ПК и сетевая истина, и движок. Его легко подменить.
3. **Производительность.** Хост одновременно: считает физику, отправляет апдейты 64 игрокам **и** рендерит свой кадр в 144 FPS. CPU и пинг — узкое горлышко.
4. **NAT и доступность.** Лобби работает через Steam-сеть; для постоянного публичного сервера нужен открытый порт и фиксированный IP.
5. **Серверный код.** Чувствительный код (API-ключи, БД, читы-детект) **нельзя** хранить в публикуемой игре — он попадёт всем клиентам. Только dedicated даёт `#if SERVER` (см. [27.04](27_04_Local_Project_Serverside_Code.md)).

## Когда уже точно нужен Dedicated Server

- ✅ Любой соревновательный шутер от 16 человек.
- ✅ Battle Royale (64–256 игроков).
- ✅ RP/MMO/симулятор «жизни» (200–1000+).
- ✅ MOBA (10 игроков, но требования к честности — выше крыши).
- ✅ Любая игра с экономикой, инвентарём за деньги, торговлей.
- ✅ Турнирный режим, где результат важнее, чем удобство.

И когда **не** нужен:

- ❌ Co-op до 4 человек.
- ❌ Sandbox-режим «зайти к другу построить машинку».
- ❌ Прототип на стадии «работает ли вообще?».

## Как s&box понимает «я — сервер»

В коде есть три ключевые проверки:

```csharp
Networking.IsHost      // я держу сетевую истину (true и на listen-хосте, и на dedicated)
Networking.IsClient    // я — клиент (true на любом, кроме dedicated, потому что dedicated не клиент себе же)
Game.IsServer          // я — выделенный сервер (нет локального игрока, нет рендера)
```

> На **listen-хосте**: `IsHost == true`, `IsClient == true`, `Game.IsServer == false`.
>
> На **dedicated**: `IsHost == true`, `IsClient == false`, `Game.IsServer == true`.

Это значит: код, который проверяет `Networking.IsHost`, **продолжит работать** и на dedicated. А вот всё, что трогает камеру, ввод и UI — нужно отгораживать `if ( Game.IsServer ) return;`.

Препроцессорный символ **`#if SERVER`** доступен **только** при сборке локального проекта, запущенного через `+game path/to/.sbproj` (см. [27.04](27_04_Local_Project_Serverside_Code.md)).

## Пара важных предупреждений

- **Dedicated работает только с *опубликованными* играми из мастерской ИЛИ с локальным проектом.** Просто «дать сборку другу-серверу» нельзя.
- **Опубликованная игра не несёт `#if SERVER`-кода вообще** — Facepunch вырезает его при публикации. Серверный код существует только если ты сам поднимаешь сервер с локальным `.sbproj` (см. [27.04](27_04_Local_Project_Serverside_Code.md)). Это сделано специально: «истинно серверная» логика остаётся у владельца сервера.
- **Сеть s&box — Steam-датаграммы.** Не «голый» UDP, поэтому DDoS и спуфинг IP сильно сложнее, но и обычные «открой порт и забудь» правила работают иначе. Подробнее — в [27.10](27_10_Hosting_Operations.md).

## Что важно запомнить

- Dedicated Server — отдельный процесс `sbox-server.exe`, без графики и локального игрока.
- Listen-хост — это «один из игроков», dedicated — «честный судья».
- Серверный (скрытый от клиентов) код возможен **только** на dedicated с локальным проектом.
- `Game.IsServer` отличает dedicated от listen-хоста; `Networking.IsHost` — общий «я авторитет».
- Без dedicated не делается ни 64+ игрок, ни анти-чит, ни RP/MMO.
