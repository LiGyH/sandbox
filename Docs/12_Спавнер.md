# Этап 12: Система спавна

## Цель этого этапа

Создать систему спавна пропов (моделей), сущностей и дупликатов. Это механика из меню спавна — выбираешь объект, и он появляется перед тобой.

## Архитектура спавна

```
Игрок нажимает "Заспавнить бочку" в меню
    ↓
GameManager.Spawn( "prop:models/barrel.vmdl" )  [Rpc.Broadcast]
    ↓ (только на хосте)
ISpawner spawner = new PropSpawner( path )
    ↓
spawner.Spawn( transform, player ) → GameObject[]
    ↓
Объект появляется в мире
```

### Интерфейс ISpawner:

```csharp
// Все спавнеры реализуют этот интерфейс
public interface ISpawner
{
    string DisplayName { get; }
    Task<bool> Loading { get; }
    Task<List<GameObject>> Spawn( Transform transform, Player player );
}
```

## Шаг 12.1: Интерфейс ISpawner

Создай файл `Code/Spawner/ISpawner.cs`:

```csharp
/// <summary>
/// Интерфейс для всех спавнеров. Каждый тип контента
/// (проп, сущность, дупликат) имеет свой спавнер.
/// </summary>
public interface ISpawner
{
    /// <summary>
    /// Имя для отображения (логи, UI).
    /// </summary>
    string DisplayName { get; }

    /// <summary>
    /// Task, который завершается когда ресурс загружен.
    /// Нужно, потому что модели могут загружаться из облака.
    /// </summary>
    Task<bool> Loading { get; }

    /// <summary>
    /// Создаёт объекты в мире.
    /// </summary>
    /// <param name="transform">Где появится объект</param>
    /// <param name="player">Кто спавнит (для отмены)</param>
    /// <returns>Список созданных объектов</returns>
    Task<List<GameObject>> Spawn( Transform transform, Player player );
}
```

## Шаг 12.2: PropSpawner — спавн пропов

Создай файл `Code/Spawner/PropSpawner.cs`:

```csharp
/// <summary>
/// Спавнит пропы (3D модели с физикой).
/// Проп = модель + Rigidbody + Collider.
/// </summary>
public class PropSpawner : ISpawner
{
    private string _modelPath;

    public PropSpawner( string modelPath )
    {
        _modelPath = modelPath;
    }

    public string DisplayName => _modelPath;
    public Task<bool> Loading => Task.FromResult( true );

    public Task<List<GameObject>> Spawn( Transform transform, Player player )
    {
        // Создаём новый объект
        var go = new GameObject( true, "Spawned Prop" );
        go.WorldTransform = transform;

        // Добавляем 3D модель
        var renderer = go.AddComponent<ModelRenderer>();
        renderer.Model = Model.Load( _modelPath );

        // Добавляем физику — пусть падает и сталкивается
        var body = go.AddComponent<Rigidbody>();
        
        // Добавляем коллайдер из модели
        var collider = go.AddComponent<ModelCollider>();
        collider.Model = renderer.Model;

        // Добавляем компонент Prop — он обрабатывает разрушение
        go.AddComponent<Prop>();

        // Делаем сетевым
        go.NetworkSpawn();

        // Устанавливаем владельца (для системы отмены)
        Ownable.Set( go, player.Network.Owner );

        return Task.FromResult( new List<GameObject> { go } );
    }
}
```

### Что такое Prop?

`Prop` — встроенный компонент движка. Он добавляет:
- Разрушение при уроне (модель разлетается на части)
- Звуки столкновений
- Информацию о здоровье модели

### Как происходит спавн:

```
1. Создаётся пустой GameObject
2. Добавляется ModelRenderer (видимая модель)
3. Добавляется Rigidbody (физика)
4. Добавляется ModelCollider (столкновения)
5. NetworkSpawn() → все клиенты видят объект
6. Ownable.Set() → запоминаем кто создал (для отмены)
```

## Шаг 12.3: EntitySpawner — спавн сущностей

Сущности — это объекты с поведением (источники света, эмиттеры, динамит).

Создай файл `Code/Spawner/EntitySpawner.cs`:

```csharp
/// <summary>
/// Спавнит сущности (entities) — объекты с логикой.
/// Отличие от пропа: сущность — это ПРЕФАБ с компонентами.
/// </summary>
public class EntitySpawner : ISpawner
{
    private string _prefabPath;

    public EntitySpawner( string prefabPath )
    {
        _prefabPath = prefabPath;
    }

    public string DisplayName => _prefabPath;
    public Task<bool> Loading => Task.FromResult( true );

    public Task<List<GameObject>> Spawn( Transform transform, Player player )
    {
        // Загружаем префаб
        var prefab = GameObject.GetPrefab( _prefabPath );
        if ( prefab is null )
        {
            Log.Warning( $"Entity prefab not found: {_prefabPath}" );
            return Task.FromResult( new List<GameObject>() );
        }

        // Клонируем префаб
        var go = prefab.Clone( new CloneConfig
        {
            Transform = transform,
            StartEnabled = true
        });

        // Делаем сетевым
        go.NetworkSpawn();
        Ownable.Set( go, player.Network.Owner );

        return Task.FromResult( new List<GameObject> { go } );
    }
}
```

## Шаг 12.4: SpawnerWeapon — оружие-спавнер

В панели быстрого доступа (hotbar) можно назначить конкретную модель на слот. При нажатии ЛКМ она спавнится.

Создай файл `Code/Spawner/SpawnerWeapon.cs`:

```csharp
/// <summary>
/// Оружие, которое спавнит конкретный объект при нажатии ЛКМ.
/// Используется в панели быстрого доступа.
/// </summary>
public class SpawnerWeapon : BaseCarryable
{
    private ISpawner _spawner;

    /// <summary>
    /// Данные спавнера (путь к модели или сущности).
    /// </summary>
    [Sync] public string SpawnerData { get; set; }

    public void SetSpawner( ISpawner spawner )
    {
        _spawner = spawner;
    }

    public void RestoreSpawnerData( string data )
    {
        SpawnerData = data;
        // Восстанавливаем спавнер из данных
    }

    public override void OnPlayerUpdate( Player player )
    {
        if ( Input.Pressed( "attack1" ) )
        {
            SpawnObject();
        }
    }

    [Rpc.Host]
    private async void SpawnObject()
    {
        if ( _spawner == null ) return;

        var player = Owner;
        if ( !player.IsValid() ) return;

        var eyes = player.EyeTransform;
        var tr = Scene.Trace
            .Ray( eyes.Position, eyes.Position + eyes.Forward * 200 )
            .IgnoreGameObjectHierarchy( player.GameObject )
            .Run();

        var transform = new Transform( tr.EndPosition );

        var objects = await _spawner.Spawn( transform, player );
        
        if ( objects is { Count: > 0 } )
        {
            var undo = player.Undo.Create();
            undo.Name = $"Spawn {_spawner.DisplayName}";
            foreach ( var go in objects )
                undo.Add( go );
        }
    }
}
```

## Шаг 12.5: Идентификатор спавна

Спавн использует строковый идентификатор формата `"тип:путь"`:

```
"prop:models/citizen_props/barrel.vmdl"     → PropSpawner
"entity:prefabs/light_point.prefab"         → EntitySpawner
"mount:garrysmod/props_c17/chair.vmdl"      → MountSpawner (из монтированных игр)
"dupe:12345"                                → DuplicatorSpawner
```

GameManager парсит этот идентификатор и создаёт нужный спавнер:

```csharp
ISpawner spawner = type switch
{
    "prop" => new PropSpawner( path ),
    "entity" => new EntitySpawner( path ),
    "mount" => new MountSpawner( path ),
    "dupe" => new DuplicatorSpawner( data ),
    _ => null
};
```

## ✅ Проверка

1. Из меню спавна выбери модель → она появляется перед тобой
2. Нажми Z → последний заспавненный объект удаляется (Undo)
3. Объекты имеют физику — падают, сталкиваются
