# 00.09 — Основы движка: GameObject и Transform

## Что мы делаем?

Прежде чем писать код Sandbox, нужно понять **фундамент движка** — что такое `GameObject`. Всё, что существует в мире игры (игрок, пуля, дерево, лампа, кнопка меню), — это `GameObject`. Без этого понимания дальше двигаться нет смысла: в Фазе 1 ты начнёшь писать код, и он на каждом шагу опирается именно на `GameObject`.

> 🧭 Этот этап — **теоретический**. Ты не пишешь файлы в проект, а читаешь и запоминаешь. Если что-то не понял — перечитай, попробуй в редакторе. В следующих файлах мы вернёмся к практике.

## Что такое GameObject?

`GameObject` — это **пустой контейнер**, «коробка», у которой есть:

1. **Transform** — где он находится в мире (позиция, поворот, масштаб).
2. **Имя и идентификатор** (`Name`, `Id` — это `Guid`).
3. **Флаг «включен/выключен»** (`Enabled`).
4. **Родитель и дети** — иерархия.
5. **Теги** (`Tags`) — произвольные метки-строки.
6. **Компоненты** — именно они делают `GameObject` чем-то осмысленным.

Сам по себе `GameObject` **ничего не делает**. Он стоит в мире как невидимая точка. Чтобы он начал что-то делать — рендериться, сталкиваться, двигаться, стрелять, — к нему нужно прикрепить **компоненты** (см. [00.12 — Component](00_12_Component.md)).

## Transform — положение в мире

`Transform` — это тройка значений: **позиция + поворот + масштаб**.

| Свойство | Что это |
|---|---|
| `WorldPosition` | Позиция в мировых координатах (`Vector3`). |
| `LocalPosition` | Позиция относительно родителя. |
| `WorldRotation` | Поворот в мире (`Rotation`). |
| `LocalRotation` | Поворот относительно родителя. |
| `WorldScale` / `LocalScale` | Масштаб (`Vector3`). |
| `WorldTransform` | Всё сразу одним объектом `Transform`. |

Пример кода из редактора или компонента:

```csharp
// Поставить объект в точку (100, 100, 100) в мире
GameObject.WorldPosition = new Vector3( 100, 100, 100 );

// Поставить относительно родителя
GameObject.LocalPosition = new Vector3( 0, 0, 64 );

// Всё сразу: позиция + угол + масштаб
GameObject.WorldTransform = new Transform(
    Vector3.Zero,
    new Angles( 0, 90, 0 ),
    1.0f );
```

> **Важно:** если у `GameObject` есть **родитель**, его `WorldPosition` вычисляется **автоматически** из позиции родителя и `LocalPosition`. Когда родитель двигается — дети двигаются вместе с ним.

## Жизненный цикл GameObject

```
┌───────────────────────────────────────────────┐
│  new GameObject()   ← создаётся               │
│         │                                     │
│         ▼                                     │
│  AddComponent<T>()  ← на него навешивают      │
│         │             компоненты              │
│         ▼                                     │
│  Enabled = true     ← работает в сцене        │
│         │                                     │
│         ▼                                     │
│  Destroy()          ← удаление                │
└───────────────────────────────────────────────┘
```

Создать `GameObject` в коде:

```csharp
var go = new GameObject();
go.Name = "Моя пуля";
go.WorldPosition = new Vector3( 0, 0, 100 );
```

Уничтожить:

```csharp
go.Destroy();          // удалить GameObject целиком
```

## Почему это важно именно для Sandbox?

В коде Sandbox ты на каждом шагу увидишь такие вещи:

```csharp
GameObject bullet = BulletPrefab.Clone( WorldPosition );
bullet.WorldRotation = eyeRotation;
```

Или:

```csharp
if ( !GameObject.IsValid() ) return;
```

Без понимания, что это «коробка с Transform и компонентами», код будет выглядеть магией. Теперь — не будет.

## Результат

После этого этапа ты знаешь:

- ✅ `GameObject` — это пустой контейнер в мире.
- ✅ У него есть `Transform` (позиция/поворот/масштаб), имя, теги, родитель, дети и компоненты.
- ✅ Сам он ничего не делает — всё делают компоненты.
- ✅ Как обращаться к его позиции в мире и относительно родителя.
