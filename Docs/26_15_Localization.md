# 26.15 — Локализация UI (`#token`, словари)

## Что мы делаем?

Если ты планируешь, чтобы в твою игру играли не только русскоязычные пользователи (а Sandbox — игра международная), любая видимая на экране строка должна идти **через систему локализации**. Этот этап показывает, как s&box сопоставляет токены и переводы.

## Принцип

Везде, где пишешь строку для игрока, **не пиши её прямо** — пиши **токен** вида `#category.key`. Движок сам подставит перевод по выбранному языку.

```html
<label>#menu.play</label>
```

```csharp
panel.Text = "#hud.health";
```

Если перевод не найден, движок просто выводит токен как есть — это видно сразу при тестировании.

## Файлы переводов

Все переводы лежат в `Localization/{language}/{prefix}.json` твоего проекта. Например:

```
Localization/
  en/sandbox.json
  ru/sandbox.json
  de/sandbox.json
```

Содержимое — обычный JSON-словарь:

```json
// Localization/ru/sandbox.json
{
    "menu.play": "Играть",
    "menu.options": "Настройки",
    "menu.quit": "Выйти",
    "hud.health": "Здоровье",
    "weapon.colt1911": "Кольт 1911",
    "spawnmenu.props": "Модели",
    "spawnmenu.tools": "Инструменты"
}
```

```json
// Localization/en/sandbox.json
{
    "menu.play": "Play",
    "menu.options": "Options",
    "menu.quit": "Quit",
    "hud.health": "Health",
    "weapon.colt1911": "Colt 1911",
    "spawnmenu.props": "Models",
    "spawnmenu.tools": "Tools"
}
```

## Где использовать токены

### В Razor-разметке

```html
<root>
    <button onclick=@OnPlay>#menu.play</button>
    <label>#hud.health: @CurrentHealth</label>
</root>
```

### В стилях/атрибутах

```html
<input placeholder="#chat.typehere" />
```

### В коде

```csharp
notice.Text = "#notice.matchstart";
```

### В конкатенациях

```csharp
panel.Text = $"{Player.Name}: #weapon.colt1911";   // токен подставится
```

s&box заменяет `#token` **в момент рендера** строки в UI, поэтому конкатенация работает.

## Fallback на английский

Если перевод отсутствует в текущем языке, движок ищет тот же токен в `en/`. Поэтому **`Localization/en/...` должен быть всегда полным** — это твоя «база истины».

## Что НЕ переводить

- Имена игроков (это пользовательский ввод).
- Названия кнопок ввода (`Mouse1`, `R`, `Space`).
- Внутренние идентификаторы и теги.
- Текст в логах разработчика (`Log.Info` — для разработчика, не для игрока).

## Параметризация (плейсхолдеры)

s&box поддерживает простой `string.Format`-style:

```json
{
    "kill.feed": "{0} убил {1} из {2}"
}
```

```csharp
notice.Text = string.Format( Localizer.Get( "kill.feed" ),
    attacker.Name, victim.Name, weapon.Name );
```

(API имени метода может отличаться от версии — фактический `Localizer.Get` или `__("...")` смотри в актуальном API). Главное правило — **не склеивай локализуемые слова** (порядок слов в языках разный):

```
"Убит из {0}"      ✅
"Убит из " + name  ❌  // в немецком слово в конце
```

## Проверка покрытия

Простой приём: при сборке бери `en/*.json` и `ru/*.json`, сравни ключи. Любой ключ, которого нет в обоих, — кандидат на fallback. Хорошо иметь маленький скрипт в CI.

## Что важно запомнить

- Любая видимая игроку строка → начинается с `#`.
- Файлы переводов — `Localization/{lang}/*.json`, простой словарь токен→перевод.
- При отсутствии перевода — **fallback на `en/`**, поэтому английский набор обязателен.
- Имена пользователей и идентификаторы **не локализуй**.
- Пиши строки с `{0}/{1}` плейсхолдерами; не клей переводимые слова.
