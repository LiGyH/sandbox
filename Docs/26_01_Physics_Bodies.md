# 26.01 — Физика: тела, формы и Rigidbody

## Что мы делаем?

Этап **«Углублённые основы движка»** дополняет [Фазу 0.5](README.md) темами из [официальной документации Facepunch](https://github.com/Facepunch/sbox-docs/), которые в основном курсе не были раскрыты. Мы начинаем с **физики**: что такое физическое тело, какие бывают коллайдеры, как заставить объект падать, скользить, отскакивать.

> 🧭 Этот файл — **теоретический**. Никакого нового кода в проект не добавляется. Цель — дать словарь, к которому тебя позже отсылают другие этапы (например, спавнер пропов, PhysGun, гибы игрока).

## Что такое физика в s&box?

`s&box` использует физический движок **Source 2 (Rubikon)**. Всё, что должно сталкиваться, падать, толкаться или висеть на верёвке, состоит из трёх вещей:

| Понятие | Класс/компонент | За что отвечает |
|---|---|---|
| **Тело** (rigid body) | `Rigidbody`-компонент | масса, скорость, ускорение, гравитация |
| **Форма** (collider) | `BoxCollider`, `SphereCollider`, `CapsuleCollider`, `MeshCollider`, `ModelCollider` | геометрия столкновений |
| **Связь** (constraint) | `FixedJoint`, `HingeJoint`, `SliderJoint`, `BallJoint`, `SpringJoint` | соединяет два тела друг с другом |

`Rigidbody` без `Collider` будет «прозрачным» — у него нет формы для столкновений. `Collider` без `Rigidbody` — это **статика** (стена, пол): объект сталкивается, но сам не двигается.

## Типы движения тела

Свойство `Rigidbody.MotionEnabled` (и связанный `RigidbodyFlags`) определяет, как тело себя ведёт:

| Тип | Поведение |
|---|---|
| **Dynamic** | физика управляет позицией: гравитация, удары, силы |
| **Static** | объект неподвижен, оптимизирован для столкновений |
| **Keyframed** (kinematic) | позицией управляешь ты (через `WorldPosition`), но столкновения работают |

Игрок и НПС обычно — **keyframed**: их двигает контроллер, а не гравитация. Бочка, ящик, выброшенное оружие — **dynamic**. Карта (BSP) и мапные пропы — **static**.

## Основные свойства Rigidbody

```csharp
Rigidbody rb = GameObject.Components.Get<Rigidbody>();

rb.MassOverride       = 50.0f;             // масса в килограммах
rb.LinearDamping      = 0.1f;              // сопротивление линейному движению
rb.AngularDamping     = 1.0f;              // сопротивление вращению
rb.Gravity            = true;              // действует ли гравитация
rb.Velocity           = Vector3.Up * 500;  // мгновенная скорость
rb.AngularVelocity    = new Vector3( 0, 0, 5 );

rb.ApplyImpulse( Vector3.Up * 1000 );      // добавить импульс к центру масс
rb.ApplyForce( Vector3.Forward * 50 );     // добавить силу (на этот тик)
rb.ApplyTorque( new Vector3( 0, 0, 100 ) );
```

> **Импульс vs сила:** импульс мгновенно меняет скорость (как удар битой), сила действует постепенно (как ветер).

## Физические материалы (Surface)

Каждый коллайдер ссылается на `Surface` — **физический материал**, описывающий *трение*, *упругость*, *плотность*, *звук удара*. В проекте ты увидишь файлы `*.surface`:

| Поле | Что задаёт |
|---|---|
| `Friction` | насколько объект скользит (0 — лёд, 1 — резина) |
| `Elasticity` | насколько отскакивает (0 — глина, 1 — мяч) |
| `Density` | плотность (масса = объём × плотность) |
| `ImpactSound` | звук, играемый при ударе |

В [01.12 — SelfCollisionSound](01_12_SelfCollisionSound.md) мы уже ссылались на эти данные — теперь ты знаешь, откуда они берутся.

## Связь с настройками проекта

Слои столкновений (Collision Matrix), которые мы настраивали в [00.05](00_05_Настройки_столкновений.md), применяются именно к этим телам. Когда два коллайдера хотят столкнуться, движок смотрит в матрицу: «тег `bullet` сталкивается с тегом `world`?» — если нет, проход насквозь.

## Что важно запомнить

- **Тело + форма + материал** — три обязательных слагаемых физического объекта.
- **Static** — для геометрии карты, **Dynamic** — для физических предметов, **Keyframed** — для игрока/НПС.
- Свойства массы, гравитации, скорости находятся на `Rigidbody`-компоненте.
- Звук удара и трение задаются `Surface`-ассетом, не кодом.
- Матрица столкновений из [00.05](00_05_Настройки_столкновений.md) фильтрует, какие коллайдеры друг с другом могут пересекаться.
