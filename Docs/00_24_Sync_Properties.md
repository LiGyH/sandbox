# 00.24 — Sync Properties (синхронизация свойств)

## Что мы делаем?

Разбираем атрибут `[Sync]` — как заставить **значение переменной автоматически отправляться всем клиентам**, когда его меняет владелец. Это способ хранить **состояние** по сети (в отличие от RPC, которые шлют **события**).

## Простой пример

```csharp
public sealed class Player : Component
{
    [Sync] public int Kills { get; set; }
    [Sync] public bool IsAlive { get; set; } = true;
    [Sync] public string Nickname { get; set; } = "Player";
}
```

- Владелец (локальный игрок) меняет `Kills = 5` — **все клиенты видят 5**.
- Прокси **не могут** писать в это свойство: они только читают.

## Какие типы поддерживаются

| Поддерживается | Нет |
|---|---|
| `int`, `float`, `bool`, `byte`, `long` | Пользовательские ссылочные классы |
| `string` | `List<T>`, `Dictionary` (напрямую) |
| `Vector3`, `Vector2`, `Rotation`, `Angles`, `Color` | Массивы (используй `NetList`) |
| Структуры (`struct`) | Делегаты, события |
| `GameObject`, `Component`, `GameResource` (через сетевые ID) | |
| Перечисления (`enum`) | |

Для коллекций есть специальные классы:

```csharp
[Sync] public NetList<int> Scores { get; set; } = new();
[Sync] public NetDictionary<string, int> Counters { get; set; } = new();
```

`NetList<T>` и `NetDictionary<K,V>` работают как обычные `List`/`Dictionary` (индексы, `Add`, `Remove`), но их изменения синхронизируются.

> ⚠️ **Важно:** `NetList` и `NetDictionary` **не** работают вместе с `[Property]` — не будут редактироваться в инспекторе.

## Детект изменений — `[Change]`

Хочешь сделать что-то, когда значение поменялось? Добавь `[Change("МетодОбратногоВызова")]`:

```csharp
public sealed class Player : Component
{
    [Sync, Change( "OnHealthChanged" )]
    public int Health { get; set; } = 100;

    void OnHealthChanged( int oldValue, int newValue )
    {
        if ( oldValue > 0 && newValue <= 0 )
            OnDeath();

        if ( newValue < oldValue )
            PlayHurtSound();
    }
}
```

Метод вызывается **у всех**, кто получил новое значение (владелец и прокси).

> ⚠️ `[Change]` **не** сработает при изменении содержимого `NetList` или `NetDictionary` — только при присваивании самого свойства.

## Флаги — `SyncFlags`

```csharp
// Хост владеет этим значением (не владелец объекта)
[Sync( SyncFlags.FromHost )] public int SessionTime { get; set; }

// Значение интерполируется (плавно меняется у прокси за несколько тиков)
[Sync( SyncFlags.Interpolate )] public Vector3 TargetPos { get; set; }

// Query-режим (см. ниже)
[Sync( SyncFlags.Query )] public Vector3 Velocity { get; set; }
```

| Флаг | Что делает |
|---|---|
| `SyncFlags.FromHost` | Владельцем значения становится хост (не владелец объекта). Пример: глобальный счёт матча. |
| `SyncFlags.Interpolate` | Значение интерполируется для других клиентов в течение нескольких тиков. Для плавных позиций/поворотов. |
| `SyncFlags.Query` | См. ниже. |

## Query Mode — ручная пометка «грязного» значения

По умолчанию движок автоматически определяет, что свойство поменялось (через кодогенерацию сеттера). Но если ты пишешь `Velocity.x = 5` — сеттера **нет**, меняешь поле структуры напрямую, движок не узнает.

`SyncFlags.Query` заставляет движок каждый тик **сам опрашивать** значение и сравнивать с предыдущим. Дороже, но ловит такие ситуации.

## Как комбинировать с `[Property]`

Часто хочется и показать в инспекторе, и синхронизировать:

```csharp
[Property, Sync] public float MoveSpeed { get; set; } = 200f;
[Property, Sync] public int MaxHealth { get; set; } = 100;
```

Дизайнер выставил значение в префабе, оно применилось, потом синхронизируется.

## `[Sync]` vs `[Rpc]`

| Задача | Используй |
|---|---|
| «Все должны знать, какое у игрока HP» | `[Sync]` |
| «Сейчас у всех должен сыграть звук выстрела» | `[Rpc.Broadcast]` |
| «У всех должен быть список моих убийств» | `[Sync] NetList<string>` |
| «Пуля попала в стену — эффекты у всех» | `[Rpc.Broadcast]` |
| «Игрок подбежал к подключившемуся позже — новый клиент должен узнать HP» | `[Sync]` (иначе новый клиент не узнает) |

Правило:
- **«Постоянное значение»** → `[Sync]`.
- **«Однократное событие»** → `[Rpc]`.

## Частые ошибки

- ❌ **Писать в `[Sync]` на прокси.** Значение перезатрётся следующим снапшотом от владельца.
- ❌ **Ожидать, что `[Change]` сработает при мутации коллекции.** Не сработает.
- ❌ **Пытаться `[Sync]` тяжёлые объекты** (текстуры, моделей целиком). Только базовые типы + ссылки на сетевые объекты.
- ❌ **Забывать `= new()` для `NetList`.** Иначе к нулевому полю нельзя обратиться.
