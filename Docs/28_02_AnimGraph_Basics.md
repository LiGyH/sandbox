# 28.02 — AnimGraph: стейты, переходы, BlendSpace

## Что мы делаем?

Разбираемся с **AnimGraph** — главным инструментом анимаций в s&box. AnimGraph — это файл `.vanmgrph`, который ты собираешь визуально в редакторе. Он отвечает на вопрос: «по текущим параметрам — какую позу скелета нарисовать прямо сейчас?»

> 🧭 Этот этап — теоретический: ты не пишешь код, а учишься читать граф в редакторе. Сам граф для Citizen уже готов в s&box — мы им пользуемся, но важно понимать устройство, чтобы не бояться менять.

## Создание .vanmgrph

1. В **Asset Browser** правый клик → **Create → Animation Graph** → имя файла.
2. Перетаскиваешь на него `.vmdl`-модель в свойствах графа (граф знает, чей у него скелет).
3. На пустом графе будет один корневой узел — **Final Pose**. Всё, что приходит в этот узел, и есть итоговая поза кадра.
4. В `.vmdl` указываешь свежесозданный AnimGraph — теперь любой `SkinnedModelRenderer` с этой моделью автоматически его крутит.

## Узлы графа

Узлы — это «коробочки», которые принимают на вход параметры/позы и выдают позу. Самые ходовые:

| Узел | Что делает |
|---|---|
| **Animation Clip** | Просто играет один `.vanim`. Кирпичик в основании. |
| **Selector** | Выбирает один из входов по `int`-параметру. Например, по `holdtype` (0=ничего, 1=пистолет, 2=винтовка). |
| **BlendList** | Смешивает несколько входов с весами. Для слоёв (стрелять + бежать). |
| **BlendSpace 1D** | Смешивает клипы по одной оси: например, `walk_back ↔ idle ↔ walk_forward` по `move_x`. |
| **BlendSpace 2D** | Смешивает клипы по сетке `(x, y)`: 8 направлений ходьбы по `move_x` и `move_y`. |
| **State Machine** | Машина состояний: «idle → run → jump → land». Переходы — по условиям/параметрам. |
| **Additive** | Накладывает разницу одной анимации поверх другой (целиться, не ломая ходьбу). |
| **Layer / Mask** | Применяет результат только к части скелета (например, только руки). |
| **IK Chain** | Двигает кость к мировой точке (см. 28.07). |

## State Machine — самое полезное

Машина состояний выглядит как граф «кружочков и стрелочек»:

```
┌──────┐  speed > 50     ┌──────┐  b_jump     ┌──────┐
│ idle │ ───────────────▶│ run  │ ──────────▶ │ jump │
└──────┘                 └──────┘             └──────┘
   ▲                        ▲ │                  │
   │ speed ≤ 50             │ │ b_grounded       │ b_grounded
   └────────────────────────┘ └──────────────────┘
```

- **Узлы** — это позы (могут быть одиночным клипом или целым подграфом).
- **Стрелки (transitions)** — переход с условием. Условия — выражения над параметрами (`speed > 50 && b_grounded`).
- **Время перехода** настраивается — можно «дотащить» 0.2 секунды плавного блендинга, можно прыгнуть мгновенно.
- **Tag-условия:** «перейти, только когда текущий клип проиграл секцию `can_exit`».

## BlendSpace 1D / 2D — для движения

**1D** — массив клипов на одной оси:

```
move_x:  -200    -100     0     100     200
clip:    walk_b  walk_b  idle  walk_f  run_f
```

Граф интерполирует между соседними клипами по значению `move_x`. Для остановки скорости тебе достаточно писать один float — поза сама развернётся в нужную ходьбу.

**2D** — массив клипов на плоскости `(x, y)`. Удобно для 8 направлений ходьбы (вперёд, назад, влево, вправо и диагонали) — у Citizen внутри именно так.

## Additive — для прицеливания

Идея: «оставь ходьбу как есть, но добавь сверху наклон корпуса вверх/вниз по `aim_pitch`».

- Берётся базовая поза (T-pose или нейтральный кадр).
- Из «целящейся» анимации **вычитается** база — получается «дельта поворота корпуса».
- Эта дельта **прибавляется** к ходьбе.
- Итог: персонаж бежит и одновременно целится туда, куда смотрит камера.

В Citizen это используется для `aim_pitch` / `aim_yaw` ViewModel’а (см. 28.06).

## Layer / Mask — для рук отдельно

Часто нужно: «ноги пусть идут как идут, но руки — в позе «держу пистолет»». Делается **слоем с маской**:

1. Берётся слой «база» (полная поза тела).
2. Поверх — слой «верхняя половина» (применяется только к костям выше `spine_2`).
3. Маска — флаг «применять / не применять» для каждой кости.

Поэтому в коде ты часто видишь:
```csharp
_renderer.Set( "holdtype", (int)CitizenAnimationHelper.HoldTypes.HoldItem );
_renderer.Set( "holdtype_pose", _holdPose );
```
— это включает **слой рук**, а ноги при этом продолжают анимацию ходьбы из базового слоя.

## Подсказки на практике

- **Не плоди десятки параметров.** Чем меньше «ручек», тем стабильнее граф.
- **Имена параметров фиксируй один раз.** Переименование = поиск по всему `Code` и правка строк.
- **Пользуйся Citizen-графом.** Для людей в Sandbox он уже всё умеет: ходьба, прицел, hold, duck, ragdoll, fall.
- **Свой граф пиши только под нестандартный скелет** (робот, насекомое, дверь со сложной механикой).
