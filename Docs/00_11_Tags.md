# 00.11 — Теги (Tags)

## Что мы делаем?

Разбираем **теги** — самый простой способ пометить `GameObject` строкой. Теги — это «ярлыки», по которым объекты группируются: «игрок», «враг», «пикап», «нельзя трогать физгуном» и т.д.

## Зачем нужны теги?

Движок и твой код **постоянно задают вопросы** типа:
- Это враг?
- Это пуля своя или чужая?
- Этот объект должен сталкиваться с другим?
- Должна ли камера его рендерить?
- Можно ли его поднять физганом?

Можно было бы проверять класс компонента (`if ( go.GetComponent<Enemy>() != null )`), но это медленно и жёстко привязывает тебя к типам. Теги — это быстрая, свободная строка-метка.

## Базовое API

```csharp
// Добавить тег
GameObject.Tags.Add( "enemy" );

// Проверить
if ( GameObject.Tags.Has( "enemy" ) )
{
    GameObject.Destroy();
}

// Удалить
GameObject.Tags.Remove( "enemy" );

// Поставить/снять одной строкой
GameObject.Tags.Set( "enemy", isEnemy );
```

## Где теги используются в s&box

### 1. Физика и столкновения

В настройках физики (см. [00.05 — Настройки столкновений](00_05_Настройки_столкновений.md)) ты указываешь **какие теги с какими сталкиваются**. Например: теги `player` и `trigger` не сталкиваются физически, но триггеры видят касание.

### 2. Трейсы (лучи)

Когда оружие делает трейс, он может **игнорировать** объекты по тегу:

```csharp
var tr = Scene.Trace
    .Ray( eyePos, eyePos + forward * 1000 )
    .WithoutTags( "player", "trigger" )    // игнорировать
    .Run();
```

Или наоборот — попасть **только** в нужные:

```csharp
var tr = Scene.Trace
    .Ray( from, to )
    .WithTag( "vehicle" )
    .Run();
```

### 3. Камера

У `CameraComponent` можно указать, какие теги рендерить и какие нет. В Sandbox это используется для: не показывать свою ViewModel в третьем лице, скрывать дебаг-геометрию, и т.д.

### 4. Твой код

Ты можешь придумать любые свои теги: `"pickup"`, `"nocollide_player"`, `"saveexclude"`, `"npc_hostile"` — что угодно.

## Наследование тегов

**Важно:** теги **наследуются** от родителя к детям.

```
Car (теги: "vehicle", "physics")
└── Wheel   ← автоматически тоже имеет "vehicle" и "physics"
```

Если у ребёнка есть свой тег `"wheel"`, но родитель имеет `"vehicle"`, у ребёнка `Tags.Has( "vehicle" )` вернёт `true`. **Убрать унаследованный тег у ребёнка нельзя** — можно только снять его с родителя.

## Как задать теги в редакторе

1. Выдели `GameObject` в сцене.
2. В инспекторе справа найди поле **Tags**.
3. Нажми **+**, введи строку тега, Enter.

Теги, добавленные в редакторе, попадают в сцену и будут работать при запуске.

## Пример из Sandbox

В Sandbox физган не должен хватать игроков. Физган делает трейс и фильтрует:

```csharp
var tr = Scene.Trace
    .Ray( eye, eye + dir * range )
    .WithoutTags( "player", "debris" )
    .Run();
```

А у пикапов на карте стоит тег `"pickup"`, и код игрока при касании ищет именно этот тег:

```csharp
if ( other.Tags.Has( "pickup" ) )
    TryPickup( other );
```

## Частые ошибки

- ❌ Писать теги **с большой буквы** в одном месте и с маленькой в другом — движок различает регистр. Придерживайся одного стиля (в Sandbox — `lowercase`).
- ❌ Пытаться снять тег с ребёнка, который унаследовал его от родителя.
- ❌ Использовать длинные строки как теги — теги это метки, не описания.
