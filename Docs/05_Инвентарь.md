# Этап 5: Инвентарь и предметы

## Цель этого этапа

Создать систему инвентаря:
- 6 слотов для оружия/инструментов
- Переключение между слотами (клавиши 1-6 или колёсико мыши)
- Подбор и выбрасывание предметов
- Автоматический выбор лучшего оружия
- Сохранение и загрузка лоадаута (набора оружия)

После этого этапа у игрока будет инвентарь, готовый принимать оружие.

## Как устроен инвентарь

Инвентарь — это компонент `PlayerInventory`, прикреплённый к тому же GameObject, что и `Player`. Оружие хранится как **дочерние объекты** игрока:

```
GameObject "Игрок"
├── Player
├── PlayerInventory          ← управляет списком оружия
├── PlayerController
└── Children (дочерние объекты):
    ├── "Physgun"            ← слот 0, ВКЛЮЧЁН (активное оружие)
    │   └── Physgun (BaseCarryable)
    ├── "Toolgun"            ← слот 1, ВЫКЛЮЧЕН
    │   └── Toolgun (BaseCarryable)
    └── "M4A1"               ← слот 2, ВЫКЛЮЧЕН
        └── M4a1Weapon (BaseCarryable)
```

### Ключевые принципы:

1. **Одно оружие активно** — остальные выключены (`Enabled = false`)
2. **Оружие = дочерний GameObject** игрока
3. **Каждое оружие знает свой слот** (`InventorySlot`)
4. **Переключение** = выключить текущее, включить новое

## Шаг 5.1: Создаём PlayerInventory

Создай файл `Code/Player/PlayerInventory.cs`:

```csharp
/// <summary>
/// Инвентарь игрока. Управляет списком оружия и инструментов.
/// Оружие хранится как дочерние объекты GameObject'а игрока.
/// </summary>
public sealed class PlayerInventory : Component, IPlayerEvent
{
    /// <summary>
    /// Максимальное количество слотов.
    /// </summary>
    [Property] public int MaxSlots { get; set; } = 6;

    /// <summary>
    /// Ссылка на компонент Player (обязательный).
    /// </summary>
    [RequireComponent] public Player Player { get; set; }

    /// <summary>
    /// Все оружия в инвентаре, отсортированные по слоту.
    /// GetComponentsInChildren ищет BaseCarryable во ВСЕХ дочерних объектах.
    /// </summary>
    public List<BaseCarryable> Weapons => GetComponentsInChildren<BaseCarryable>( true )
        .OrderBy( x => x.InventorySlot )
        .ToList();

    /// <summary>
    /// Текущее активное оружие.
    /// [Sync] — синхронизируется от хоста.
    /// [Change] — при изменении вызывается OnActiveWeaponChanged.
    /// </summary>
    [Sync( SyncFlags.FromHost ), Change] 
    public BaseCarryable ActiveWeapon { get; private set; }

    /// <summary>
    /// Вызывается автоматически когда ActiveWeapon меняется.
    /// Выключает старое оружие, включает новое.
    /// </summary>
    public void OnActiveWeaponChanged( BaseCarryable oldWeapon, BaseCarryable newWeapon )
    {
        // Выключаем старое оружие
        if ( oldWeapon.IsValid() )
            oldWeapon.GameObject.Enabled = false;

        // Включаем новое оружие
        if ( newWeapon.IsValid() )
            newWeapon.GameObject.Enabled = true;
    }

    // ==============================
    // Поиск и слоты
    // ==============================

    /// <summary>
    /// Возвращает оружие в указанном слоте, или null если пусто.
    /// </summary>
    public BaseCarryable GetSlot( int slot )
    {
        if ( slot < 0 || slot >= MaxSlots ) return null;
        return GetComponentsInChildren<BaseCarryable>( true )
            .FirstOrDefault( x => x.InventorySlot == slot );
    }

    /// <summary>
    /// Находит первый пустой слот. Возвращает -1 если всё занято.
    /// </summary>
    public int FindEmptySlot()
    {
        // Собираем все занятые слоты
        var occupied = GetComponentsInChildren<BaseCarryable>( true )
            .Where( x => x.InventorySlot >= 0 )
            .Select( x => x.InventorySlot )
            .ToHashSet();

        // Ищем первый свободный
        for ( int i = 0; i < MaxSlots; i++ )
        {
            if ( !occupied.Contains( i ) )
                return i;
        }

        return -1; // Всё занято
    }

    /// <summary>
    /// Возвращает оружие определённого типа.
    /// </summary>
    public T GetWeapon<T>() where T : BaseCarryable
    {
        return Weapons.OfType<T>().FirstOrDefault();
    }

    /// <summary>
    /// Есть ли оружие определённого типа в инвентаре?
    /// </summary>
    public bool HasWeapon<T>() where T : BaseCarryable
    {
        return GetWeapon<T>().IsValid();
    }

    /// <summary>
    /// Возвращает лучшее оружие (по значению Value).
    /// </summary>
    public BaseCarryable GetBestWeapon()
    {
        return Weapons.OrderByDescending( x => x.Value ).FirstOrDefault();
    }

    // ==============================
    // Подбор предметов
    // ==============================

    /// <summary>
    /// Выдаёт оружие по умолчанию новому игроку.
    /// </summary>
    public void GiveDefaultWeapons()
    {
        Pickup( "weapons/physgun/physgun.prefab", false );
        Pickup( "weapons/toolgun/toolgun.prefab", false );
    }

    /// <summary>
    /// Подобрать оружие из префаба (путь к файлу).
    /// Автоматически находит свободный слот.
    /// </summary>
    public bool Pickup( string prefabName, bool notice = true )
    {
        if ( !Networking.IsHost ) return false;

        var slot = FindEmptySlot();
        if ( slot < 0 ) return false;

        return Pickup( prefabName, slot, notice );
    }

    /// <summary>
    /// Подобрать оружие в конкретный слот.
    /// </summary>
    public bool Pickup( string prefabName, int targetSlot, bool notice = true )
    {
        if ( !Networking.IsHost ) return false;

        // Загружаем префаб оружия
        var prefab = GameObject.GetPrefab( prefabName );
        if ( prefab is null )
        {
            Log.Warning( $"Prefab not found: {prefabName}" );
            return false;
        }

        if ( targetSlot < 0 || targetSlot >= MaxSlots ) return false;

        // Проверяем — слот занят?
        var occupant = GetSlot( targetSlot );
        if ( occupant.IsValid() ) return false;

        // Клонируем оружие как дочерний объект игрока
        var clone = prefab.Clone( new CloneConfig 
        { 
            Parent = GameObject,        // Родитель = игрок
            StartEnabled = false        // Начинаем выключенным
        });
        
        // Делаем сетевым с владельцем игрока
        clone.NetworkSpawn( false, Network.Owner );

        // Настраиваем слот
        var weapon = clone.GetComponent<BaseCarryable>( true );
        weapon.InventorySlot = targetSlot;

        // Уведомляем об подборе
        IPlayerEvent.PostToGameObject( Player.GameObject, e => e.OnPickup( weapon ) );

        return true;
    }

    // ==============================
    // Переключение оружия
    // ==============================

    /// <summary>
    /// Переключить на указанное оружие.
    /// </summary>
    public void SwitchWeapon( BaseCarryable weapon, bool allowHolster = false )
    {
        // Клиент отправляет запрос хосту
        if ( !Networking.IsHost )
        {
            HostSwitchWeapon( weapon, allowHolster );
            return;
        }

        // Если уже держим это оружие
        if ( weapon == ActiveWeapon )
        {
            if ( allowHolster )
                ActiveWeapon = null; // Убрать оружие
            return;
        }

        ActiveWeapon = weapon;
    }

    /// <summary>
    /// RPC для переключения оружия — клиент → хост.
    /// </summary>
    [Rpc.Host]
    private void HostSwitchWeapon( BaseCarryable weapon, bool allowHolster = false )
    {
        SwitchWeapon( weapon, allowHolster );
    }

    // ==============================
    // Обработка ввода
    // ==============================

    /// <summary>
    /// Вызывается каждый кадр из Player.OnControl().
    /// Обрабатывает нажатия клавиш инвентаря.
    /// </summary>
    public void OnControl()
    {
        // Выбросить активное оружие (G)
        if ( Input.Pressed( "drop" ) )
        {
            if ( ActiveWeapon.IsValid() )
                Drop( ActiveWeapon );
            return;
        }

        // Обновляем активное оружие
        if ( ActiveWeapon.IsValid() && !ActiveWeapon.IsProxy )
            ActiveWeapon.OnPlayerUpdate( Player );
    }

    // ==============================
    // Выбрасывание
    // ==============================

    /// <summary>
    /// Выбросить оружие. Создаёт физический объект перед игроком.
    /// </summary>
    public bool Drop( BaseCarryable weapon )
    {
        if ( !Networking.IsHost ) return false;
        if ( !weapon.IsValid() ) return false;

        // Позиция и скорость броска
        var dropPosition = Player.EyeTransform.Position + Player.EyeTransform.Forward * 48f;
        var dropVelocity = Player.EyeTransform.Forward * 200f + Vector3.Up * 100f;

        // Если это активное оружие — сначала убираем
        if ( ActiveWeapon == weapon )
            SwitchWeapon( null, true );

        // Удаляем оружие
        weapon.DestroyGameObject();

        // Переключаемся на лучшее оставшееся
        var best = GetBestWeapon();
        if ( best.IsValid() )
            SwitchWeapon( best );

        return true;
    }

    // ==============================
    // Обновление каждый кадр
    // ==============================

    protected override void OnUpdate()
    {
        var renderer = Player?.Controller?.Renderer;

        if ( ActiveWeapon.IsValid() )
        {
            // Обновляем оружие каждый кадр
            ActiveWeapon.OnFrameUpdate( Player );
        }
    }

    // ==============================
    // События
    // ==============================

    /// <summary>
    /// Когда игрок заспавнился — выдаём оружие по умолчанию.
    /// </summary>
    void IPlayerEvent.OnSpawned()
    {
        GiveDefaultWeapons();
    }
}
```

### Разбор ключевых моментов:

#### Как работает переключение оружия по сети:

```
Клиент нажимает "2":
1. OnControl() → обнаруживает нажатие Slot2
2. SwitchWeapon( GetSlot(1) )
3. Клиент → отправляет HostSwitchWeapon через [Rpc.Host]
4. Хост получает запрос → ActiveWeapon = новое оружие
5. [Sync] + [Change] → новое значение отправляется ВСЕМ
6. У каждого клиента вызывается OnActiveWeaponChanged()
7. Старое оружие выключается, новое включается
```

#### Почему клиент не может менять ActiveWeapon напрямую?

`ActiveWeapon` помечен `[Sync( SyncFlags.FromHost )]` — менять может **только хост**. Это защита от читов: клиент не может «включить» оружие, которого у него нет.

#### `GetComponentsInChildren<BaseCarryable>( true )`

Эта функция ищет **все компоненты** типа `BaseCarryable` (и его наследников) среди **дочерних объектов**. Параметр `true` означает «включая выключенные объекты».

```
GameObject "Игрок"          ← PlayerInventory здесь
├── "Physgun"               ← дочерний объект
│   └── Physgun : BaseCarryable  ← будет найден!
└── "Toolgun"               ← дочерний объект (выключен)
    └── Toolgun : BaseCarryable  ← тоже будет найден (true)
```

## Шаг 5.2: Обновляем Player — интеграция инвентаря

Добавь в `Player.OnControl()`:

```csharp
private void OnControl()
{
    Controller.UseInputControls = true;

    // Обработка ввода инвентаря
    GetComponent<PlayerInventory>()?.OnControl();

    // Переключение слотов по номеру (1-6)
    for ( int i = 0; i < 6; i++ )
    {
        if ( Input.Pressed( $"Slot{i + 1}" ) )
        {
            var inventory = GetComponent<PlayerInventory>();
            var weapon = inventory?.GetSlot( i );
            if ( weapon.IsValid() )
                inventory.SwitchWeapon( weapon );
        }
    }
}
```

## Шаг 5.4: Сохранение лоадаута (упрощённое)

В полной версии `PlayerInventory` сохраняет набор оружия игрока в `LocalData` (локальное хранилище). При следующем спавне оружие восстанавливается.

Формат хранения — JSON-массив:

```json
[
    {"PrefabPath": "weapons/physgun/physgun.prefab", "Slot": 0},
    {"PrefabPath": "weapons/toolgun/toolgun.prefab", "Slot": 1},
    {"PrefabPath": "weapons/m4a1/m4a1.prefab", "Slot": 2}
]
```

> 💡 Сохранение лоадаута мы полностью реализуем в Этапе 16 (Сохранение и загрузка). Пока достаточно базовой системы.
