# 00.22 — Ownership (владение сетевым объектом)

## Что мы делаем?

Разбираем **владение** — ключевое понятие сетевой модели. Владелец объекта — это тот, кто **симулирует его положение и состояние**. Все остальные видят прокси. Правильная работа с ownership — это 80% правильного мультиплеера.

## Основной принцип

> **У сетевого объекта всегда ровно один владелец** (или его нет — тогда владелец = хост).
> Владелец двигает его, меняет `[Sync]`, может запускать RPC с флагом `OwnerOnly`.
> Все остальные видят **прокси**, ничего не решают, только рендерят.

## Кто владеет объектом по умолчанию?

| Откуда появился объект | Владелец |
|---|---|
| Стоял в сцене изначально (Network Object) | **Никто** — симулирует хост |
| Кто-то сделал `go.NetworkSpawn()` | **Этот кто-то** |
| Кто-то сделал `go.NetworkSpawn( conn )` | Указанный `conn` |

## Проверки в коде

```csharp
if ( IsProxy )
{
    // я НЕ владелец — этим занимается кто-то другой
    return;
}

// я владелец — двигаю, стреляю, читаю инпут
```

Более точные проверки:

```csharp
Network.IsOwner         // я владелец этого объекта
Network.OwnerId         // Guid владельца
Network.Owner           // объект Connection владельца (или null)
Network.HasOwner        // есть ли вообще владелец
```

## Смена владельца — Owner Transfer

По умолчанию **только хост** может менять владельца. Это можно поменять:

```csharp
go.Network.SetOwnerTransfer( OwnerTransfer.Takeover );  // любой может забрать
go.Network.SetOwnerTransfer( OwnerTransfer.Request );   // нужно согласие хоста
go.Network.SetOwnerTransfer( OwnerTransfer.Fixed );     // только хост (по умолчанию)
```

Для Sandbox типично ставят `Takeover` на пропы — чтобы любой игрок, взяв физганом, мог симулировать его у себя (плавное движение без лага).

## Забрать владение

Если `OwnerTransfer.Takeover`:

```csharp
go.Network.TakeOwnership();
```

С этого момента ты владелец, ты двигаешь, все остальные — прокси.

## Отдать владение

```csharp
go.Network.DropOwnership();
```

Теперь владельца нет, симулирует хост.

## Типичный паттерн — Pickup

Игрок подбирает пропу, владение этой пропой переходит к нему:

```csharp
void TryPickup()
{
    var tr = Scene.Trace
        .Ray( eyePos, eyePos + look * 100 )
        .WithoutTags( "player" )
        .Run();

    if ( !tr.Hit ) return;
    if ( tr.GameObject is not { } go ) return;
    if ( !go.Tags.Has( "pickup" ) ) return;

    // "Теперь ты моя" — становлюсь симулятором
    go.Network.TakeOwnership();

    Carrying = go;
}
```

И при выбрасывании:

```csharp
void Drop()
{
    Carrying?.Network.DropOwnership();
    Carrying = null;
}
```

## Почему это вообще нужно?

Представь: игрок А держит физганом кирпич. Если кирпичом владеет хост, движения игрока А идут:

```
A двигает мышь → A шлёт хосту инпут → хост двигает кирпич → хост шлёт А новую позицию
```

Это **2× пинг** задержки для игрока А. Кирпич будет лагать.

Если владение на А:

```
A двигает мышь → A сам двигает кирпич → A шлёт позицию всем остальным
```

Для А — мгновенно. Остальные видят минимум лага, плюс умная интерполяция.

Поэтому:
- Игрок — владеет **своим** игровым объектом.
- Машина — владеет **тот, кто за рулём**.
- Прочее — владеет хост, либо тот, кто взаимодействовал последним.

## Disconnection — что если владелец вышел?

По умолчанию — его объекты удалятся. Настраивается через **Orphaned Mode**:

```csharp
go.Network.SetOrphanedMode( NetworkOrphaned.Host );     // хост забирает
go.Network.SetOrphanedMode( NetworkOrphaned.Random );   // случайный клиент
go.Network.SetOrphanedMode( NetworkOrphaned.ClearOwner );
go.Network.SetOrphanedMode( NetworkOrphaned.Destroy );  // по умолчанию
```

**Только сам владелец** может менять Orphaned Mode (другие не имеют права).

## Частые ошибки

- ❌ **Двигать прокси-объект** — движения не увидят другие.
- ❌ **Писать в `[Sync]` будучи прокси** — значение перезатрётся следующим снапшотом от владельца.
- ❌ **Не проверять `IsProxy`** в `OnFixedUpdate` с движением — получишь «борьбу за позицию», когда все клиенты двигают один объект.
- ❌ **Забыть `SetOwnerTransfer`** на пропах, которые игроки должны брать — у них не получится забрать владение.

## Карта решений

| Задача | Действие |
|---|---|
| Игрок управляет своим персонажем | `NetworkSpawn( conn )` отдаёт владение сразу |
| Игрок взял пропу физганом | `Network.TakeOwnership()` (с `OwnerTransfer.Takeover`) |
| Игрок отпустил пропу | `Network.DropOwnership()` |
| Игрок сел в машину | Машина меняет владельца на игрока |
| Игрок вышел из игры, но его машина пусть остаётся | `NetworkOrphaned.Host` |
