# 00.08 — Настройки сети (Networking)

## Что мы делаем?

Настраиваем **сетевую часть** — как сервер и клиенты обмениваются данными.

## Зачем это нужно?

Sandbox — мультиплеер-игра. Когда ты играешь с друзьями:
- Один компьютер — **сервер** (хост), он считает всю физику и логику
- Остальные — **клиенты**, они отправляют свои действия и получают обновления

## Шаг 1: Создай файл

Путь: `ProjectSettings/Networking.config`

```json
{
  "DestroyLobbyWhenHostLeaves": true,
  "AutoSwitchToBestHost": false,
  "ClientsCanSpawnObjects": false,
  "ClientsCanRefreshObjects": false,
  "UpdateRate": 30,
  "__guid": "bf156628-1b7e-4af7-a101-1d08e8ca36c5",
  "__schema": "configdata",
  "__type": "NetworkingSettings",
  "__version": 1
}
```

## Что означает каждая строка?

| Поле | Значение | Объяснение |
|------|----------|------------|
| `DestroyLobbyWhenHostLeaves` | `true` | Когда хост уходит — лобби закрывается |
| `AutoSwitchToBestHost` | `false` | Не переключать хоста автоматически |
| `ClientsCanSpawnObjects` | `false` | Клиенты **не могут** создавать объекты напрямую |
| `ClientsCanRefreshObjects` | `false` | Клиенты **не могут** обновлять объекты напрямую |
| `UpdateRate` | `30` | 30 сетевых обновлений в секунду |

> **Как это работает внутри движка:**  
> Система сети в `engine/Sandbox.Engine/` использует **авторитарную модель**:
> - **Сервер решает всё** — позиции, здоровье, физика
> - **Клиент только запрашивает** — «я хочу идти вперёд», «я хочу стрелять»
>
> `ClientsCanSpawnObjects: false` — это **безопасность**.  
> Если бы клиенты могли создавать объекты, читеры могли бы спавнить что угодно.  
> Вместо этого клиент отправляет **запрос** серверу, и сервер решает — разрешить или нет.
>
> `UpdateRate: 30` означает, что сервер отправляет клиентам обновления 30 раз в секунду.  
> Это меньше чем `TickRate: 50` (физика), потому что сеть — это узкое место.  
> Движок **интерполирует** (плавно переходит) между обновлениями, чтобы движение выглядело плавно.

## Разница между TickRate и UpdateRate

```
Физика (50 Hz):    ● ● ● ● ● ● ● ● ● ●  (50 расчётов в секунду)
Сеть (30 Hz):      ● ·  ● · ● ·  ● · ● ·  (30 отправок в секунду)
```

Физика считается чаще, но отправляется клиентам реже — чтобы не засорять канал.

- ✅ Авторитарная модель (сервер решает)
- ✅ 30 обновлений в секунду по сети
