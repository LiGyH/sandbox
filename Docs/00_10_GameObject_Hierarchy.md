# 00.10 — Иерархия GameObject: родитель и дети

## Что мы делаем?

Разбираем **иерархию** — как `GameObject` могут быть вложены друг в друга, и почему это важно. Без этого непонятно, как к игроку «прикрепляется» оружие, как у машины вращаются колёса и почему `ViewModel` оружия двигается вместе с камерой.

## Зачем вообще нужна иерархия?

Представь, что ты сделал модель машины. У машины есть 4 колеса. Если каждое колесо — отдельный `GameObject` в мире, то когда машина поедет, колёса **останутся на месте**, потому что их никто не двигает.

Решение: сделай машину **родителем** колёс. Тогда:

- **Двигаешь машину** → колёса двигаются вместе с ней автоматически.
- **Вращаешь колесо** → вращается только оно, машина стоит.

```
Car (GameObject)       ← родитель, WorldPosition = (100, 0, 0)
├── WheelFL             ← ребёнок, LocalPosition = (+50, -30, 0) → мир = (150, -30, 0)
├── WheelFR             ← ребёнок, LocalPosition = (+50, +30, 0) → мир = (150, +30, 0)
├── WheelRL             ← ребёнок, LocalPosition = (-50, -30, 0) → мир = (50, -30, 0)
└── WheelRR             ← ребёнок, LocalPosition = (-50, +30, 0) → мир = (50, +30, 0)
```

Когда `Car.WorldPosition` меняется, **мировые позиции детей пересчитываются автоматически** из их `LocalPosition` + позиции родителя.

## Как установить родителя в коде

```csharp
// Сделать weapon дочерним к player
weapon.SetParent( player );

// Или сразу при создании
var bullet = new GameObject();
bullet.SetParent( gun );
bullet.LocalPosition = Vector3.Zero; // ровно в центре пушки
```

Отвязать от родителя:

```csharp
weapon.SetParent( null );   // теперь weapon — корневой объект сцены
```

## Обращение к детям и родителю

```csharp
// Перечислить всех детей
foreach ( var child in GameObject.Children )
{
    Log.Info( child.Name );
}

// Получить родителя
GameObject parent = GameObject.Parent;

// Найти конкретного ребёнка по имени
var muzzle = GameObject.Children.FirstOrDefault( x => x.Name == "Muzzle" );
```

## Local vs World — не путай!

Самая частая ошибка новичков — путать `Local*` и `World*`:

| Свойство | Относительно чего | Когда использовать |
|---|---|---|
| `WorldPosition` | Всего мира (0,0,0 — центр карты) | Когда надо телепортировать в абсолютную точку карты |
| `LocalPosition` | Своего родителя | Когда надо прицепить объект к другому со смещением |

Пример из оружия Sandbox — muzzle flash должен появляться **на кончике ствола**, который прикреплён к модели:

```csharp
// ❌ НЕПРАВИЛЬНО — мировая точка (0,0,0), а не ствол
flashEffect.WorldPosition = Vector3.Zero;

// ✅ ПРАВИЛЬНО — прикрепить к стволу и сдвинуть локально
flashEffect.SetParent( muzzleBone );
flashEffect.LocalPosition = Vector3.Zero;
```

## Удаление с детьми

Когда ты удаляешь `GameObject` с `Destroy()`, **все его дети удаляются автоматически**. Это удобно: удалил машину — колёса тоже исчезли.

## Наследование тегов

Теги (см. [00.11](00_11_Tags.md)) **наследуются** от родителя к детям. Если у машины есть тег `vehicle`, у всех её колёс автоматически тоже есть этот тег.

## Результат

После этого этапа ты знаешь:

- ✅ Как сделать один `GameObject` ребёнком другого (`SetParent`).
- ✅ Чем `LocalPosition` отличается от `WorldPosition`.
- ✅ Что дети двигаются вместе с родителем автоматически.
- ✅ Что `Destroy()` у родителя удаляет и всех детей.
- ✅ Как перечислить детей и получить родителя.
