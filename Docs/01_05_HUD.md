# 01.05 — Масштабирование интерфейса (HUD)

## Что мы делаем?

Этот файл — крошечный, но важный. Он предоставляет **коэффициент масштабирования** для пользовательского интерфейса (HUD). Суть проста: все элементы HUD проектируются под разрешение 1080p (1920×1080). Если у игрока другое разрешение экрана, этот коэффициент пересчитывает размеры элементов пропорционально.

Представьте: вы нарисовали полоску здоровья шириной 200 пикселей на мониторе 1080p. Если у игрока монитор 4K (2160p), без масштабирования полоска будет крошечной. Коэффициент `Hud.Scale` решает эту проблему.

## Как это работает внутри движка?

- `Screen.Height` — текущая высота экрана в пикселях (предоставляется движком).
- Деление `Screen.Height / 1080.0f` даёт множитель:
  - На 1080p: `1080 / 1080 = 1.0` (без изменений).
  - На 720p: `720 / 1080 ≈ 0.67` (элементы уменьшаются).
  - На 4K: `2160 / 1080 = 2.0` (элементы увеличиваются вдвое).

Любой элемент UI умножает свой размер на `Hud.Scale`, и он автоматически подстраивается под экран.

## Создай файл

Путь: `Code/Utility/HUD.cs`

```csharp
namespace Sandbox;

internal static class Hud
{
	public static float Scale => Screen.Height / 1080.0f;
}
```

## Разбор кода

### Пространство имён

```csharp
namespace Sandbox;
```
Класс находится в пространстве `Sandbox`, доступен внутри проекта.

### Объявление класса

```csharp
internal static class Hud
```
- `internal` — класс доступен только внутри текущей сборки (проекта). Внешние модули его не видят.
- `static` — экземпляр создавать не нужно. Обращаемся напрямую: `Hud.Scale`.

### Свойство Scale

```csharp
public static float Scale => Screen.Height / 1080.0f;
```
- `=>` — expression-bodied свойство (вычисляется при каждом обращении).
- `Screen.Height` — текущая высота экрана.
- `1080.0f` — эталонная высота. Суффикс `f` означает `float`.
- Результат: число с плавающей точкой — множитель масштаба.

### Пример использования

```csharp
// Размер шрифта для 1080p — 24 пикселя
float fontSize = 24.0f * Hud.Scale;
// На 4K: 24 * 2.0 = 48 пикселей
// На 720p: 24 * 0.67 = ~16 пикселей
```

## Результат

После создания этого файла:
- Проект имеет единый коэффициент масштабирования UI.
- Все элементы HUD могут использовать `Hud.Scale` для адаптации под любое разрешение.
- Интерфейс выглядит одинаково на мониторах 720p, 1080p, 1440p и 4K.
